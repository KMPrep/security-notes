Attack Structure

Category: Offensive Security, Threat Modelling, Detection Engineering

Overview

Understanding attack structure — the ordered phases an attacker moves through from first contact to final objective — is foundational for both offensive and defensive security engineering. It lets defenders ask: "At which phase does our current detection coverage fail? Where are our blind spots?"

Two frameworks dominate:

  • Cyber Kill Chain (Lockheed Martin, 2011): Linear 7-phase model; useful for APT-style targeted attacks
  • MITRE ATT&CK (see also Threat Modelling): Tactic-technique matrix; more granular; reflects real-world attacker behaviour

Cyber Kill Chain

The Kill Chain models a targeted intrusion as seven sequential phases. An attacker must complete each phase to reach the next. Defenders can break the chain at any phase to stop the attack.

flowchart LR
    R[Reconnaissance] --> W[Weaponisation]
    W --> D[Delivery]
    D --> E[Exploitation]
    E --> I[Installation]
    I --> C[Command and Control]
    C --> A[Actions on Objectives]
Phase Attacker Goal Attacker Actions Defender Countermeasures
Reconnaissance Gather intelligence on target OSINT, Google dorking, LinkedIn scraping, Shodan, port scanning, phishing simulations Monitor for scanning; track exposed info; red team your own attack surface
Weaponisation Create the attack tool Build or buy malware; embed in document; pair with exploit Threat intel; track attacker TTPs; harden endpoints against known exploit kits
Delivery Get the weapon to the target Phishing email, watering hole, USB drop, supply chain compromise Email filtering, URL rewriting, user awareness training, USB policy
Exploitation Trigger the vulnerability Browser exploit, macro execution, social engineering the user Patch management, sandboxing, application allowlisting, disable Office macros
Installation Establish persistence Drop malware, create backdoor user, modify run keys, install service EDR, FIM, Sysmon, least privilege
Command and Control (C2) Maintain attacker access Beacon to C2 server via HTTP/S, DNS, ICMP, custom protocol DNS monitoring, TLS inspection, anomaly detection, network segmentation
Actions on Objectives Achieve the mission Exfiltrate data, encrypt files, sabotage systems, move laterally DLP, honey tokens, data classification, CASB, rate limiting

Kill Chain limitation: It is linear. Real attacks skip phases, return to earlier phases, or run phases in parallel. Modern attackers (especially cloud-native attacks) may go directly from initial access to impact without a separate persistence phase.

MITRE ATT&CK Phases in Attack Narrative

The ATT&CK matrix adds granularity that the Kill Chain lacks. Below is a representative targeted attack mapped to ATT&CK tactics with example techniques and concrete sample tradecraft at each step.

Phase 1 — Reconnaissance (TA0043)

Attacker goal: Learn enough about the target to craft a credible attack.

Technique Example Sample attack technique / tradecraft
T1598 — Phishing for Information Attacker sends a survey pretending to be an IT support team Fake helpdesk form captures org chart, VPN vendor, and MFA method
T1595 — Active Scanning Nmap scan of IP range to find exposed services Nmap -sV -Pn; HTTP title grabs; TLS cert enumeration
T1591 — Gather Victim Org Info LinkedIn scraping to identify employees and their roles LinkedIn, company careers page, press releases, GitHub org mining
T1589 — Gather Victim Identity Info HaveIBeenPwned; previous breach data to find reused passwords Breach corpus correlation; username pattern discovery; email reuse

Defender visibility: OSINT about your own attack surface. Shodan/Censys scans of your IP range. Red team exercise.

Applying Attack Structure to Defense

For each phase, ask:

  1. What logs would this generate? (Detection visibility)
  2. What controls prevent this phase? (Prevention)
  3. How long would this phase take to detect with current tooling? (Dwell time)
  4. If this phase is missed, which subsequent phase can we still catch them? (Defense in depth)
Phase Primary Detection Source Primary Prevention Control
Reconnaissance Perimeter scan logs; threat intel Attack surface reduction; OSINT hygiene
Initial Access Email gateway; WAF; EDR Phishing training; patching; MFA
Execution Sysmon; EDR; PowerShell logs Application allowlisting; disable macros
Persistence Sysmon; registry monitoring; FIM Least privilege; endpoint hardening
Lateral Movement Windows event logs; network flows Network segmentation; MFA on all internal services
Exfiltration DLP; network monitoring Data classification; CASB; egress filtering

Practice Questions

  • Walk me through a realistic phishing-to-data-exfiltration attack using the Kill Chain or ATT&CK framework.
  • At which phase in the Kill Chain is it most cost-effective to stop an attacker? Why?
  • How would you detect Pass-the-Hash (T1550.002) in Windows event logs?
  • An attacker is using LOLBins to avoid triggering AV. Name three LOLBin techniques and how you would detect each.
  • A user receives a spearphishing email. They click the link, their browser is exploited, and a Cobalt Strike beacon is installed. Map each step to an ATT&CK technique and state what detection you would add.
  • What is domain fronting and how is it used in C2? How would you detect or block it?
  • How does the supply chain compromise vector (T1195) differ defensively from a phishing attack?
