Data Structures & Algorithms to Master

Scope: Consolidated study list derived from three problem sets in this repo:

  • code/log_parsing/ — 25 senior security-engineer log-parsing questions
  • code/practice/medium/ — 120 medium detection/parsing/secure-code questions
  • code/practice/hard/ — 60 hard streaming / graph / concurrency / parser-security questions

How to use: Master Tier 1 + Tier 2 to clear essentially all medium and log-parsing
questions. Tiers 4–6 are the high-signal differentiators for the 60 hard questions and
senior Google/Meta/AWS security-engineering loops.

Every solution must state time and space complexity (explicit "GOLDEN RULE" in the
log-parsing set) and run end-to-end with embedded sample data.

Tier 3 — Algorithm techniques

  • Two-pointer / sliding-window join — correlating two log streams within a time window (hard Q11; log_parsing Q20; medium Q62).
  • Binary search — longest-prefix match, time-bounded lookups in sorted logs, bisect on timestamps (hard Q25).
  • Greedy interval merging — rate limiting, rule coalescing (hard Q54; medium Q73–Q75).
  • String matching — Aho-Corasick (multi-IOC), KMP/regex, domain-suffix, ReDoS-safe scanning (hard Q29–Q31).
  • Edit distance / homograph detection — Levenshtein + Unicode confusables (hard Q28).
  • State-machine parsing — multi-format syslog normalizer, email headers, key=value (log_parsing Q23; hard Q21; medium Q51–Q60).

Tier 4 — Streaming & probabilistic (the "hard" differentiator)

  • Count-Min Sketch — heavy hitters with bounded memory (hard Q02).
  • HyperLogLog / cardinality estimation — distinct counts at scale (hard Q01–Q03 context).
  • Reservoir sampling — uniform sampling over unbounded streams.
  • Lossy counting (Manku–Motwani) — frequency over threshold phi (hard Q02).
  • Bloom filter — fast probabilistic membership for blocklists/IOCs.

Tier 5 — Math / statistics for detection

  • Shannon entropy — DNS tunneling, DGA domains, encoded payloads (log_parsing Q4, Q11; medium Q29, Q31).
  • Mean / stddev / jitter coefficient — C2 beaconing periodicity (stddev/mean) (hard Q04; log_parsing Q3; medium Q27).
  • Inter-arrival / time-series analysis — beaconing, low-and-slow scans (hard Q04, Q05).
  • Haversine / geo-velocity — impossible-travel detection (medium Q9; log_parsing Q10).

Tier 6 — Concurrency & systems (hard set, Q53–Q60)

  • Locks, mutexes, race conditions — token revocation race, concurrent dedup (hard Q53, Q55).
  • Token bucket / sliding-window rate limiter — thread-safe (hard Q54; medium Q73–Q75).
  • Distributed lock (lease/TTL, fencing tokens) (hard Q58).
  • Atomic operations & idempotency — alert suppression (hard Q10, Q55).

Security-specific primitives (appear throughout, not classic DS/algo)

  • Constant-time comparison (hmac.compare_digest) — HMAC/webhook/JWT verification (hard Q45, Q46; medium Q114).
  • CIDR / IP math (ipaddress module, integer conversion) — subnet grouping, SSRF blocklists (hard Q22–Q25; medium Q67–Q70).
  • Safe parsing patterns — zip-bomb/decompression bounds, path-traversal normalization, magic-number validation (hard Q35–Q39, Q51).
