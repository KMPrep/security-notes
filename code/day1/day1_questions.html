25 Coding practice questions for Google Security Engineer - Day 1

These questions focus on Python fluency: conditionals, loops, lists, dicts, sets, string methods, slicing, list comprehensions, and functions. They're designed to be solvable in 10-15 minutes each, with a security engineering twist where applicable. Practice writing pseudocode first, then code, and test with examples.

String Manipulation

  1. Validate Email Format: Write a function to check if a string is a valid email (basic: contains '@' and '.').
  2. Count Special Characters: Given a string, count how many non-alphanumeric characters it contains.
  3. Reverse String Words: Reverse the order of words in a sentence without reversing the words themselves.
  4. Extract Domain from URL: From a URL string, extract the domain (e.g., "https://www.example.com/path" -> "www.example.com").
  5. Anagram Check: Check if two strings are anagrams (ignore case and spaces).

List and Array Operations

  1. Remove Duplicates: Remove duplicates from a list while preserving order.
  2. Find Missing Number: In a list of numbers from 1 to n with one missing, find it.
  3. Merge Two Sorted Lists: Merge two sorted lists into one sorted list.
  4. Rotate List: Rotate a list to the left by k positions.
  5. Find Intersection: Find common elements between two lists (preserve order from first).

Dictionary and Set Usage

  1. Word Frequency: Count the frequency of each word in a sentence.
  2. Group by Key: Given a list of tuples (key, value), group values by key.
  3. Unique Elements: Use a set to find unique elements in a list.
  4. Phone Book Lookup: Implement a simple phone book using a dict (add, lookup).
  5. Set Operations: Find the union, intersection, and difference of two sets.

Loops and Conditionals

  1. FizzBuzz: Print numbers 1 to 100, "Fizz" for multiples of 3, "Buzz" for 5, "FizzBuzz" for both.
  2. Prime Checker: Check if a number is prime.
  3. Sum of Evens: Sum all even numbers in a list.
  4. Max in List: Find the maximum value in a list without using built-in max().
  5. Palindrome List: Check if a list reads the same forwards and backwards.

List Comprehensions and Functions

  1. Square Numbers: Use list comprehension to square all numbers in a list.
  2. Filter Strings: Filter a list of strings to keep only those starting with a vowel.
  3. Function Composition: Write a function that takes another function and applies it twice.
  4. Default Arguments: Write a function with default arguments for logging levels.
  5. Security-Flavored: Parse Log Levels: Given a list of log lines, count occurrences of "INFO", "WARN", "ERROR".

Tips

  • Focus on clean, readable code.
  • Test edge cases: empty inputs, single elements, large inputs.
  • Time yourself: Aim for 10-15 minutes per question.
  • For security context: Think about input validation, avoiding common pitfalls like SQL injection (though not applicable here).
