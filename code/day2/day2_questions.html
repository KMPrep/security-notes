Day 2: Arrays, Strings, Hash Maps

Google Security Engineer practice questions

Focus: Arrays/lists, dictionaries, membership tests, counting, prefix-style thinking.
Complexity: Medium. These problems require you to combine data structures deliberately.
Time target: 15–25 minutes per problem.

Security framing: Many of these map directly to log analysis, threat detection, access
auditing, and event correlation — common in Google Security Engineering roles.

Arrays and Lists

Q1. Find All Duplicate IPs
Given a list of IP addresses from a firewall log, return all IPs that appear more than once.

  • Input: ["10.0.0.1", "192.168.1.1", "10.0.0.1", "10.0.0.2", "192.168.1.1"]
  • Output: ["10.0.0.1", "192.168.1.1"]
  • Hint: Use a dict to count, then filter.
  • Complexity goal: O(n) time, O(n) space.

Q2. Merge Sorted Event Timestamps
Given two sorted lists of Unix timestamps from two log sources, merge them into one sorted list.

  • Input: [1000, 1005, 1010], [1002, 1007, 1020]
  • Output: [1000, 1002, 1005, 1007, 1010, 1020]
  • Constraint: Do not use sorted() on the combined list. Use two-pointer merge.
  • Complexity goal: O(n + m) time.

Q3. Longest Consecutive Sequence of Failed Logins
Given a list of login results ("OK" or "FAIL"), find the longest consecutive run of "FAIL".

  • Input: ["OK", "FAIL", "FAIL", "FAIL", "OK", "FAIL", "FAIL"]
  • Output: 3
  • Security context: Detects brute-force login bursts.

Q4. Rotate Access Log Window
Given a list of log lines and an integer k, rotate the list left by k positions.

  • Input: ["a","b","c","d","e"], k=2
  • Output: ["c","d","e","a","b"]
  • Write two versions: one with slicing, one without.

Q5. Sliding Window: Peak Traffic Detection
Given a list of request counts per second and window size k, find the max total requests in any window of size k.

  • Input: [100, 200, 150, 400, 300, 250], k=3
  • Output: 850 (400+300+250)
  • Complexity goal: O(n) using sliding window, not O(n*k).

Strings and Parsing

Q6. First Non-Repeated Error Code
Given a string of error codes separated by spaces, find the first error code that appears only once.

  • Input: "E404 E200 E404 E500 E200 E403"
  • Output: "E500"
  • Hint: Use a dict to count, then scan again for the first with count 1.

Q7. Valid Bracket Depth in Config String
Given a config string with nested brackets, return the maximum nesting depth reached.

  • Input: "{host: {ip: {range: 10}}}"
  • Output: 3
  • Edge case: Unmatched brackets should return -1.

Q8. Caesar Cipher Detect and Decode
An attacker encoded a string with a Caesar cipher (shifted by an unknown amount 1–25).
Given the ciphertext and a known plaintext word that must appear in the decoded string, find the shift.

  • Input: ciphertext="khoor zruog", known="hello"
  • Output: shift=3, decoded="hello world"

Q9. Count Unique Substrings of Length K
Given a string of user session tokens and length k, return how many distinct substrings of length k exist.

  • Input: "abcabc", k=3
  • Output: 3 ("abc", "bca", "cab")
  • Complexity goal: O(n) using a set.

Q10. Normalize Log Paths
Given a list of file paths that may contain . and .., normalize each path.

  • Input: ["/var/log/../log/./syslog", "/etc/hosts/../hosts"]
  • Output: ["/var/log/syslog", "/etc/hosts"]
  • Hint: Use a stack.

Hash Maps (Dicts)

Q11. Two Sum — Threat Pair Detection
Two threat scores in a list sum to a critical threshold. Return their indices.

  • Input: scores=[2, 7, 11, 15], target=9
  • Output: [0, 1] (scores 2 and 7)
  • Complexity goal: O(n) using a dict, not O(n²) brute force.

Q12. Group Log Lines by Severity
Given a list of log lines each starting with a severity level, group them by severity.

  • Input: ["ERROR disk full", "INFO user login", "ERROR timeout", "WARN high mem"]
  • Output: {"ERROR": ["disk full", "timeout"], "INFO": ["user login"], "WARN": ["high mem"]}

Q13. Longest Substring Without Repeating Characters
Classic sliding window. Security context: scan a token stream for repeated auth patterns.

  • Input: "abcabcbb"
  • Output: 3 ("abc")
  • Complexity goal: O(n) with a dict tracking last seen index.

Q14. Anagram Detection in Alert Names
Given a list of alert names, group those that are anagrams of each other.

  • Input: ["listen","silent","enlist","google","gooles","inlets"]
  • Output: [["listen","silent","enlist","inlets"], ["google","gooles"]]
  • Hint: Sort each word as the key.

Q15. Count Users with Multiple Failed Attempts
From a list of (username, result) tuples, return users with 3 or more consecutive failures.

  • Input: [("alice","FAIL"),("alice","FAIL"),("alice","FAIL"),("bob","FAIL"),("bob","OK")]
  • Output: ["alice"]

Prefix Thinking

Q16. Prefix Sum for Suspicious Activity Windows
Given a list of daily security incident counts and a range [i, j], return the sum of incidents in that range in O(1) after O(n) preprocessing.

  • Input: counts=[1, 3, 2, 5, 4], query [1, 3]
  • Output: 10 (3+2+5)
  • Build the prefix sum array first.

Q17. Longest Common Prefix in Hostname List
Given a list of hostnames, find the longest common prefix.

  • Input: ["auth-server-01", "auth-server-02", "auth-db-01"]
  • Output: "auth-"

Q18. Zero-Sum Subarray in Anomaly Scores
Given a list of anomaly delta scores (can be negative), find if any subarray sums to zero.

  • Input: [4, -1, -3, 1, -2, 2]
  • Output: True (subarray [-1, -3, 1, -2, 2] sums to -3... recalculate; find one that does)
  • Hint: Use prefix sums and a set. If same prefix sum appears twice, there's a zero-sum subarray.

Mixed / Applied

Q19. IP Access Frequency Report
Given a list of raw log lines in the format "IP METHOD PATH STATUS", build a report:
count total requests per IP, and flag any IP with more than 5 requests.

  • Keep it to string split + dict aggregation, no regex needed yet.

Q20. Session Token Validation
Given a list of session tokens (alphanumeric, length 32), identify any that:

  • Are shorter or longer than 32 chars
  • Contain non-alphanumeric chars
  • Are duplicates
    Return a dict with keys "invalid", "duplicates".

Q21. Rate-Limit Checker
Given a list of (user_id, timestamp) pairs sorted by timestamp, detect any user who made
more than 5 requests within any 60-second window.

  • Return a list of offending user IDs.

Q22. Reconstruct Access Order
You have a dict mapping each file to the file it was accessed from (like a linked list).
Reconstruct the full access chain from a given start file.

  • Input: path={"b":"c", "a":"b", "c":"d"}, start="a"
  • Output: ["a", "b", "c", "d"]

Q23. Find Symmetric Pairs in Firewall Rules
Given a list of (src, dst) firewall rule tuples, find all symmetric pairs where both
(A, B) and (B, A) exist.

  • Input: [(1,2),(3,4),(2,1),(5,6),(4,3)]
  • Output: [(1,2),(3,4)]

Q24. Count Distinct Alert Types in Rolling Window
Given a list of alert type strings and window size k, return the number of distinct alert types
in each window of size k.

  • Input: ["XSS","SQLi","XSS","CSRF","SQLi"], k=3
  • Output: [2, 3, 2, 2]

Q25. Minimum Window Containing All Severity Levels
Given a log line list, find the smallest contiguous subarray that contains at least one of each:
"INFO", "WARN", "ERROR".

  • Input: ["INFO","XSS","WARN","INFO","ERROR","WARN"]
  • Output: ["WARN","INFO","ERROR"] (indices 3–4... find actual minimum)
  • Complexity goal: O(n) sliding window with dict tracking current window coverage.

Study Notes for Day 2

  • Reach for a dict any time you need counting, grouping, or fast lookup.
  • When you see "first occurrence" or "unique", think ordered dict or two-pass with dict.
  • Sliding window = two pointers + dict/set tracking the current window.
  • Prefix sums turn range queries from O(n) to O(1) after O(n) setup.
  • Complexity: Write it for every solution. Most of these should be O(n) or O(n log n).
