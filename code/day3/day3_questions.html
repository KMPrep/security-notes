Day 3: Stacks, Recursion Basics, Regex

Google Security Engineer practice questions

Focus: Stack usage, push/pop patterns, parentheses matching, DFS-style thinking, regex.
Complexity: Medium-hard. Problems require knowing when to reach for a stack vs. a loop.
Time target: 20–30 minutes per problem.

Security framing: Stacks model call chains, nested configs, policy evaluation trees, and
path traversal attacks. Regex is core to log detection rules, SIEM signatures, and input
validation. These show up constantly in security tooling work at Google.

Stacks

Q1. Balanced Brackets in Firewall Rule Config
Given a string of firewall config containing {}, [], (), determine if all brackets are balanced.

  • Input: "{allow: [tcp(80), tcp(443)]}" → True
  • Input: "{allow: [tcp(80)]}" missing close → False
  • Edge cases: empty string, only one bracket, nested mismatches.

Q2. Simplify Filesystem Path (Directory Traversal Detection)
Given an absolute path that may contain . and .., return the simplified canonical path.

  • Input: "/var/log/../log/./syslog" → "/var/log/syslog"
  • Input: "/etc/passwd/../../etc/shadow" → "/etc/shadow"
  • Security context: This is the foundation of path traversal (directory traversal) attack detection.

Q3. Evaluate Security Policy Expression
Given a postfix (Reverse Polish Notation) boolean expression using AND, OR, NOT,
and boolean literals T/F, evaluate it.

  • Input: ["T", "F", "AND"] → False
  • Input: ["T", "F", "OR", "T", "AND"] → True
  • Use a stack. Push literals, pop for operators.

Q4. Decode Compressed Log Token
Given an encoded string where k[pattern] means repeat pattern k times, decode it.

  • Input: "3[ab]2[cd]" → "ababababcdcd"
  • Input: "2[a3[b]]" → "abbbabbb" (nested)
  • Use a stack to handle nesting.

Q5. Next Greater Timestamp
Given a list of Unix timestamps, for each timestamp find the next timestamp in the list
that is strictly greater. Return -1 if none exists.

  • Input: [1000, 900, 1100, 1050, 1200]
  • Output: [1100, 1100, 1200, 1200, -1]
  • Use a monotonic stack. Security context: detect the next spike after a quiet period.

Q6. Largest Alert Window (Largest Rectangle in Histogram)
Given a list of alert counts per hour as bar heights, find the largest contiguous block
(max width × min height) that fits entirely within all bars in that range.

  • Input: [2, 1, 5, 6, 2, 3]
  • Output: 10 (bars 5 and 6, height 5, width 2)
  • Use a stack-based approach. Complexity goal: O(n).

Q7. Zombie Process Chain
Given a stack-based call tree represented as a string of push (P) and pop (K) operations, simulate the stack and return the sequence of items removed (killed).

  • Input: calls=["PA","PB","PC","KC","PD","KD","KB","KA"]
  • Output: killed in order: ["C","D","B","A"]

Recursion Basics

Q8. Flatten Nested Permission Structure
Given a nested list (lists can contain ints or other lists), flatten it to a single list.

  • Input: [1, [2, [3, 4]], 5]
  • Output: [1, 2, 3, 4, 5]
  • Write a recursive version. Know when to use recursion vs. a stack.

Q9. Count Nodes in a Policy Tree
A policy tree is represented as a dict: {name: [child1, child2, ...]}.
Recursively count total nodes in the tree from a given root.

  • Input: {"root": ["a","b"], "a": ["c"], "b": [], "c": []}, root="root"
  • Output: 4

Q10. Generate All Subsets of Permission Flags
Given a list of permission flags like ["read","write","exec"], generate all possible subsets.

  • Output: [[], ["read"], ["write"], ["exec"], ["read","write"], ...] (8 total)
  • Use recursion (choose/don't-choose pattern). Security: models least-privilege enumeration.

Q11. Power Set of Alert Tags
Same structure as Q10 but return only subsets of size <= 2 (practical for alert combinations).

  • Input: ["XSS","SQLi","RCE","LFI"]
  • Output: all subsets of size 0, 1, or 2.

Regex

Q12. Extract All IPv4 Addresses from Log Blob
Given a multi-line log string, extract all valid IPv4 addresses (each octet 0–255).

  • Pattern must reject: 999.1.1.1, 1.2.3, 1.2.3.4.5
  • Return deduplicated list preserving first-seen order.

Q13. Extract Timestamps from Mixed Logs
Given log lines in various formats, extract all timestamps matching YYYY-MM-DD HH:MM:SS.

  • Input: multi-line string with timestamps embedded in sentences
  • Output: list of matched timestamps

Q14. Validate Email-Like Strings
Write a regex to validate strings of the form user@domain.tld where:

  • user is alphanumeric plus ._-
  • domain is alphanumeric plus -
  • tld is 2–6 alpha chars
  • Return True/False for each input.

Q15. Detect Suspicious URL Patterns
Given a list of URL strings, use regex to flag any that contain:

  • SQL injection patterns: ' OR, %27, --
  • Path traversal: ../, ..%2F
  • Double encoding: %25
    Return a list of (url, reason) tuples for flagged URLs.

Q16. Extract HTTP Log Fields
Given a line in Combined Log Format:

192.168.1.1 - frank [10/Oct/2023:13:55:36 -0700] "GET /index.html HTTP/1.1" 200 2326

Write a single regex to extract: IP, user, timestamp, method, path, status code, bytes.

Q17. Match Base64-Looking Strings
In a log blob, identify tokens that look like Base64-encoded strings (length >= 20,
chars [A-Za-z0-9+/=]). Flag them for further analysis.

  • Return a list of matched tokens.

Q18. Redact PII from Logs
Given a log string, use regex to redact:

  • Email addresses → [EMAIL]
  • IPv4 addresses → [IP]
  • Credit card patterns (\d{4}-\d{4}-\d{4}-\d{4}) → [CC]
    Return the redacted string.

Mixed / Applied

Q19. Policy String Parser
Given a policy string like "DENY ip=10.0.0.1 port=443 proto=TCP", parse it into a dict:
{"action":"DENY","ip":"10.0.0.1","port":"443","proto":"TCP"}.
Use regex or split—whichever is cleaner.

Q20. Validate Config Nesting Depth
Given a JSON-like config string, use a stack to verify:

  • All brackets are balanced
  • Nesting depth never exceeds 5
  • Return (valid: bool, max_depth: int)

Q21. Stack-Based Undo Log Replay
Given a sequence of operations ("SET", key, value) or ("UNDO"), simulate a simple
key-value store with undo. Return final state of all keys.

  • Input: [("SET","x",1),("SET","y",2),("SET","x",3),("UNDO"),("UNDO")]
  • Output: {"x": 1, "y": 2}

Q22. Recursive Permission Inheritance
A role inherits permissions from parent roles: {role: [parent_roles]}.
Write a recursive function to collect all permissions for a role given:
{role: {"parents": [...], "perms": [...]}}.

  • Return deduplicated merged permission list.

Q23. Regex-Based Log Anomaly Tagger
Given a list of log lines and a dict of {tag: regex_pattern}, annotate each log line
with all matching tags. Return [(line, [tags])].

  • Example tags: {"brute_force": r"FAIL.*login", "port_scan": r"SYN.*multiple"}

Q24. Extract CIDR Ranges
Using regex, extract all CIDR notations (x.x.x.x/y) from a firewall ruleset string.
Validate that the prefix length is 0–32. Return valid CIDRs only.

Q25. Call Stack Depth Limiter
Write a recursive function to compute Fibonacci(n) with a maximum recursion depth of 10.
If n requires deeper recursion, switch to iterative. Include logic to detect which path was taken.

  • Security context: Recursive bombs / stack overflow as a DoS vector.

Study Notes for Day 3

  • A stack is the right tool when the problem involves: nesting, undo/redo, pending work, or "process last seen".
  • Regex in Python: always use re.compile() for patterns used repeatedly (performance).
  • Use re.findall() for extraction, re.match() for full-string validation, re.search() for anywhere in string.
  • Recursion depth in Python defaults to 1000. In practice, note this and offer an iterative alternative.
  • For DFS/tree traversal without explicit recursion, push children onto a stack manually.
  • Complexity: Stack operations are O(1) push/pop, giving O(n) total for most of these problems.
