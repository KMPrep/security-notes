Day 4: Sorting, Searching, Big O

Google Security Engineer practice questions

Focus: Linear vs binary search, sort algorithms conceptually, time/space complexity reasoning.
Complexity: Medium-hard. Every solution must include written complexity analysis.
Time target: 20–30 minutes per problem.

Security framing: Sorting and searching underpin threat correlation (sort by timestamp, find
earliest event), alert triage (find top-k offenders), policy lookup (binary search in ordered
ACLs), and anomaly detection (deviation from sorted baseline).

RULE FOR TODAY: Write O(?) time and O(?) space after every solution. No exceptions.

Binary Search

Q1. Classic Binary Search
Implement binary search from scratch. Return the index or -1 if not found.

  • Input: arr=[1,3,5,7,9,11], target=7 → 3
  • Input: target=4 → -1
  • Complexity: O(log n) time, O(1) space.
  • Edge cases: empty array, single element, first/last element.

Q2. Search Insert Position
Given a sorted list of unique integers, return the index where target would be inserted
to keep it sorted. If it exists, return its index.

  • Input: [1,3,5,6], target=5 → 2
  • Input: [1,3,5,6], target=2 → 1
  • Security context: Where does this new severity score rank in our baseline?

Q3. First and Last Occurrence
Given a sorted array with duplicates, find the first and last position of a given target.

  • Input: [5,7,7,8,8,10], target=8 → [3,4]
  • Input: target=6 → [-1,-1]
  • Complexity: O(log n) using two binary searches.

Q4. Find Peak in Alert Spike Data
Given a list of hourly alert counts where values first increase then decrease (unimodal),
find the peak value's index without scanning linearly.

  • Input: [1, 3, 7, 12, 9, 5, 2] → 3 (value 12)
  • Complexity goal: O(log n).

Q5. Search in Rotated Sorted Array
A sorted array of threat scores was rotated at an unknown pivot. Find target without sorting again.

  • Input: [4,5,6,7,0,1,2], target=0 → 4
  • Input: target=3 → -1
  • Complexity: O(log n). Explain the two-case logic clearly.

Q6. Count Elements in Range
Given a sorted list of timestamps (integers), count how many fall within [lo, hi] inclusive.

  • Input: [100,200,300,400,500], lo=150, hi=450 → 3 (200, 300, 400)
  • Complexity: O(log n) using two binary searches (bisect_left / bisect_right logic).

Sorting

Q7. Sort Log Events by Timestamp (Stable)
Given a list of (timestamp, event_type, message) tuples, sort by timestamp ascending.
When timestamps tie, preserve original order (stable sort).

  • Explain why Python's sort() is stable by default (Timsort).
  • Complexity: O(n log n) time, O(1) extra space (in-place).

Q8. Top-K Most Active IPs
Given a list of IP strings from a log, find the K most frequently occurring IPs.

  • Input: ["1.1.1.1","2.2.2.2","1.1.1.1","3.3.3.3","1.1.1.1","2.2.2.2"], k=2
  • Output: ["1.1.1.1", "2.2.2.2"]
  • Method 1: Sort by count descending. Complexity?
  • Method 2: Use heapq.nlargest. Complexity?
  • When would you prefer a min-heap for top-k over full sort?

Q9. Interval Merge for Firewall Rule Consolidation
Given a list of port ranges [start, end], merge all overlapping ranges.

  • Input: [[1,3],[2,6],[8,10],[15,18]]
  • Output: [[1,6],[8,10],[15,18]]
  • Step 1: Sort by start. Step 2: Merge linearly.
  • Complexity: O(n log n) sort + O(n) merge.

Q10. Custom Sort: Severity-First Triage
Given a list of (severity, timestamp, message) tuples, sort by:

  1. Severity descending (CRITICAL > HIGH > MEDIUM > LOW > INFO)

  2. Timestamp ascending within same severity

  3. Use a custom key function. Do not hardcode comparisons for each pair.

Q11. Sort Strings by Security Relevance (Custom Comparator)
Given a list of process names, sort them so that any name containing "root" or "admin"
comes first, followed by others alphabetically. Use functools.cmp_to_key.

Q12. Find Two Numbers Summing to Target in Sorted Array (Two Pointer)
Given a sorted list of integers, find indices of the two numbers that sum to a target.
Use two pointers — not a hash map.

  • Input: [2, 7, 11, 15], target=9 → [0, 1]
  • Complexity: O(n) time, O(1) space. Contrast with hash map approach O(n)/O(n).

Big O Reasoning

Q13. Complexity Analysis: Log Deduplication
You are given this code. Analyze its time and space complexity:

def dedup_logs(logs):
    seen = set()
    result = []
    for log in logs:
        if log not in seen:
            seen.add(log)
            result.append(log)
    return result
  • What is T(n)? What is S(n)?
  • Could you make it O(1) space? What's the tradeoff?

Q14. Complexity Analysis: Nested Loop Alert Correlation
Analyze this code:

def correlate(alerts, rules):
    matches = []
    for alert in alerts:
        for rule in rules:
            if rule in alert:
                matches.append((alert, rule))
    return matches
  • What is T(n, m)? How would you improve it?
  • Propose an O(n + m) alternative using a dict or trie.

Q15. Space Complexity: Recursive vs Iterative
Compare the space complexity of recursive DFS vs iterative DFS on a policy tree of depth d
with n total nodes.

  • Recursive: call stack uses O(d) space.
  • Iterative (explicit stack): also O(d) in worst case for a path-like tree.
  • When does recursive DFS risk stack overflow in Python? (d > ~900)

Q16. Amortized Analysis: Dynamic Array Append
Python lists use dynamic arrays. Appending n items one by one:

  • Each individual append is O(1) amortized (occasional O(n) resize).
  • Total cost for n appends = O(n).
  • Explain WHY with the doubling strategy. Apply this reasoning to log ingestion pipelines.

Q17. Explain: Why is dict lookup O(1)?
Give a clear verbal explanation:

  • Hash function maps key to bucket index.
  • Collision handling (chaining or open addressing).
  • Worst case O(n) with hash collisions — when does this matter in security? (Hash flooding attacks)
  • Python 3.6+ dicts are insertion-ordered and use compact hash tables.

Mixed / Applied

Q18. Binary Search on Answer: Minimum Scan Threshold
You want to find the minimum alert threshold T such that the number of alerts with
score >= T is exactly <= K. Given sorted scores, find T using binary search on the answer.

  • Input: scores=[1,3,5,7,9,11], K=3 → T=7 (7,9,11 are >=7)
  • Think: binary search over the value space, not the index space.

Q19. Sort and Deduplicate IP Blocklist
Given a messy list of IP strings (with duplicates, unsorted), produce a sorted,
deduplicated blocklist. What is the complexity?

  • Bonus: what if the list has 10 million entries? Discuss memory and disk-sort strategies.

Q20. Kth Largest Anomaly Score
Given an unsorted list of floats (anomaly scores), find the Kth largest.

  • Approach 1: Sort descending, index k-1. Complexity?
  • Approach 2: Min-heap of size k. Complexity?
  • Which is better when n=10M and k=5?

Q21. Count Inversions in Threat Timeline
An "inversion" is a pair (i, j) where i < j but scores[i] > scores[j] — meaning a score
decreased unexpectedly. Count total inversions.

  • Brute force: O(n²). Describe the merge-sort based O(n log n) approach conceptually.

Q22. Stable Partition by Severity
Given a mixed list of events, partition so all CRITICAL events come before non-CRITICAL,
preserving relative order within each group.

  • This is a stable partition. Can you do it in O(n) time and O(1) extra space?
  • What Python built-in achieves this?

Q23. Bisect: Insert Alert into Sorted Feed
Use the bisect module to maintain a sorted list of timestamps as new alerts arrive.
Insert 1000 new alerts one by one. What is total complexity vs. sorting at the end?

  • Per insert with bisect_insort: O(n) due to list shift. Total: O(n²).
  • Heap-based alternative: O(n log n). When does each win?

Q24. Find Earliest Conflicting Firewall Rule
Given a sorted list of rules each with a port range [lo, hi], find the first rule that
overlaps with any previous rule. Return its index.

  • Use binary search to efficiently check overlap against previously seen ranges.

Q25. Design: What Algorithm Would You Use?
For each scenario, name the algorithm and give its complexity — no coding needed, just reasoning:

  1. Search a sorted blocklist of 1M IPs for a single IP.
  2. Find the top 10 most active scanners from 50M log events.
  3. Detect if any two events in a log are exact duplicates.

Study Notes for Day 4

Complexity cheat sheet:

  • O(1): hash lookup, stack push/pop, index access
  • O(log n): binary search, heap insert/pop into fixed-size heap
  • O(n): single pass, linear scan, hash build
  • O(n log n): sort, merge sort, heap build from scratch
  • O(n²): nested loops, brute-force pair scan

Binary search template (memorize):

lo, hi = 0, len(arr) - 1
while lo <= hi:
    mid = (lo + hi) // 2
    if arr[mid] == target:
        return mid
    elif arr[mid] < target:
        lo = mid + 1
    else:
        hi = mid - 1
return -1

Sort key patterns:

# Sort by second element descending, then first ascending
items.sort(key=lambda x: (-x[1], x[0]))

# Custom severity order
order = {"CRITICAL": 0, "HIGH": 1, "MEDIUM": 2, "LOW": 3}
events.sort(key=lambda e: order.get(e[0], 99))
