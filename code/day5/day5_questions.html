Day 5: Real Parsing and Data Handling

Google Security Engineer practice questions

Focus: Applied coding. File I/O, regex, dict aggregation, report generation.
Complexity: Hard applied. These are not toy problems — they simulate actual security tooling work.
Time target: 30–45 minutes per problem. Some are multi-part scripts.

Security framing: Security engineers at Google write detection scripts, parse telemetry,
build log pipelines, and hunt threats. These problems reflect exactly that work. The goal is
a fully working script end-to-end — not a partial sketch.

RULE FOR TODAY: Every solution must run end-to-end. No stubs. Test with real sample input.

Log Parsing

Q1. HTTP Access Log Parser
Build a script that parses HTTP access logs in Combined Log Format:

192.168.1.1 - frank [10/Oct/2023:13:55:36 -0700] "GET /admin HTTP/1.1" 404 512

Extract per line: IP, user, timestamp, method, path, status code, bytes.
Report:

  • Total request count
  • Status code distribution (count per code)
  • Top 5 IPs by request count
  • Top 5 paths by request count
  • List of all IPs that hit paths containing /admin or /passwd

Q2. Failed Login Detector
Given a log file where each line is:

2024-01-15T10:30:00Z auth: user=alice result=FAIL src=10.0.0.1
2024-01-15T10:30:05Z auth: user=alice result=FAIL src=10.0.0.1
2024-01-15T10:30:10Z auth: user=alice result=OK   src=10.0.0.1

Build a script that:

  • Counts consecutive failures per user before a success resets the counter.
  • Flags any user who hits >= 5 consecutive failures.
  • Reports: username, max consecutive fails, source IPs seen.

Q3. Multi-Source Log Correlation
Given two log files (firewall log and auth log), correlate events by IP within a 60-second window:

  • Firewall log: timestamp ip port action (ALLOW/DENY)
  • Auth log: timestamp ip username result (OK/FAIL)
    For each IP with a firewall DENY followed by an auth FAIL within 60s, flag it.
    Output: (ip, fw_timestamp, auth_timestamp, username)

Q4. DNS Query Log Analyzer
Given a DNS query log:

2024-01-15T10:00:00Z client=10.0.0.5 query=evil.example.com type=A

Build a script that:

  • Counts unique domains queried per client IP.
  • Detects possible DGA (Domain Generation Algorithm) domains: random-looking domains
    where the domain label has entropy > 3.5 bits/char (use character frequency).
  • Reports top 10 most queried domains and flags DGA candidates.

Q5. Syslog Severity Summarizer
Given a syslog-format file:

Jan 15 10:00:01 hostname kernel: [12345.678] something happened
Jan 15 10:00:02 hostname sshd[1234]: Failed password for root from 192.168.1.1

Build a script that:

  • Extracts: timestamp, hostname, process name (before :), and message.
  • Groups events by process name.
  • Flags lines matching patterns: Failed password, invalid user, Connection refused.
  • Outputs a per-process summary and a flagged events list.

Data Transformation

Q6. JSON Log Normalizer
Given a JSONL file (one JSON object per line) with inconsistent schemas:

{"ts": "2024-01-15", "src_ip": "1.1.1.1", "action": "BLOCK"}
{"timestamp": "2024-01-15T10:00:00", "ip": "2.2.2.2", "event": "deny"}

Normalize each record to a canonical schema:
{"timestamp": ..., "src_ip": ..., "action": ...}
Map field aliases, handle missing fields with None, and output clean JSONL.

Q7. CSV Security Event Report
Given a CSV file of security events with columns:
event_id, timestamp, severity, source_ip, destination_ip, event_type, description
Build a script that:

  • Reads the CSV using csv.DictReader.
  • Groups events by severity.
  • For each (source_ip, destination_ip) pair, counts how many events occurred.
  • Outputs a sorted report: most active pairs first.
  • Writes the filtered CRITICAL events to a new CSV.

Q8. Log Line Enricher
Given a log file of IPs and a dict of known bad IPs with their threat categories:

threat_intel = {"1.2.3.4": "C2", "5.6.7.8": "scanner", ...}

Enrich each log line: if the IP matches threat intel, append the category to the line.
Flag enriched lines for review. Output: enriched lines, count by category.

Counting and Aggregation

Q9. Rate-of-Change Anomaly Detector
Given a time-series list of (minute, event_count) tuples, flag any minute where the count
increased by more than 3x compared to the previous minute.

  • Output: list of flagged (minute, prev_count, curr_count, multiplier) tuples.
  • Security context: sudden traffic spike = possible DDoS.

Q10. Top Talkers Report
Given a flat list of (src_ip, dst_ip, bytes) flow records, produce:

  • Top 5 source IPs by total bytes sent.
  • Top 5 destination IPs by total bytes received.
  • All (src, dst) pairs where bytes > threshold (e.g., 1GB).
    Use dict aggregation, then sort.

Q11. Session Reconstruction
Given a list of (user, timestamp, action) events sorted by timestamp:

  • Group consecutive events per user into sessions (gap > 30 min = new session).
  • For each session, report: user, session start, session end, actions taken.
  • Flag sessions with any action matching "DELETE" or "EXPORT".

Regex Applied

Q12. Extract and Redact PII from Logs
Given a log dump, write a single-pass script that:

  • Extracts all email addresses, IPv4 addresses, and phone numbers (US format).
  • Redacts them in the output: [EMAIL], [IP], [PHONE].
  • Reports counts by type.
  • Input must be handled line by line (do not load file into memory at once).

Q13. Rule-Based Log Tagger
Given a set of detection rules as (tag, regex) pairs and a log file:

  • For each log line, find all matching tags.
  • Output: (line_number, line, [tags]) for any line with at least one match.
  • Rules example:
    python rules = [ ("brute_force", r"Failed password.*(\d+) times"), ("privesc", r"sudo:.*COMMAND=/bin/bash"), ("exfil", r"bytes_sent=[0-9]{7,}"), ]

Q14. Apache Error Log Parser
Parse Apache error log lines:

[Mon Jan 15 10:00:00.123456 2024] [error] [pid 1234] [client 192.168.1.1:54321] File does not exist: /var/www/html/admin

Extract: timestamp, level, pid, client IP, client port, message.
Group by error message pattern (strip variable parts) and count frequency.

Study Notes for Day 5

File reading patterns:

# Line by line (memory efficient)
with open('file.log', 'r') as f:
    for line in f:
        process(line.strip())

# All lines at once (small files)
lines = open('file.log').readlines()

Dict aggregation pattern:

from collections import defaultdict

counts = defaultdict(int)
groups = defaultdict(list)

for item in data:
    counts[item.key] += 1
    groups[item.key].append(item.value)

Top-K from dict:

top5 = sorted(counts.items(), key=lambda x: -x[1])[:5]

Regex compiled for performance:

import re
PATTERN = re.compile(r'(\d{1,3}\.){3}\d{1,3}')  # compile once, reuse
for line in lines:
    matches = PATTERN.findall(line)
