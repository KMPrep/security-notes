25 Log Parsing Questions — Senior Security Engineer Level

Focus: Production-grade log analysis, threat detection, and security telemetry pipelines.
Time target: 30–45 minutes per problem. Every solution must run end-to-end.
Level: These reflect real work at Google/Meta/AWS security engineering roles.

GOLDEN RULE: Every answer must be a fully runnable script with embedded sample data.
No stubs. No pseudocode. State time and space complexity for every approach.

Security framing: As a senior security engineer you write detection scripts, parse
multi-source telemetry, build correlation pipelines, and hunt threats at scale.
These questions cover exactly that scope — from raw log parsing to multi-source
timeline reconstruction.

Section 1 — Web and Network Traffic

Q1. Apache/Nginx Combined Log Deep Parser
Parse HTTP access logs in Combined Log Format. Beyond basic field extraction, your
script must also:

  • Detect scanner fingerprints in User-Agent (sqlmap, nikto, masscan, nmap, dirbuster).
  • Flag path traversal attempts (../, %2e%2e, %252e).
  • Detect 4xx/5xx error spikes: any IP with > 10 errors in the log window.
  • Identify unusually large responses (bytes > 1 MB) that could indicate data exfil.
  • Report: scan candidates, traversal attempts, error-spike IPs, large-response pairs.

Sample line:

10.0.0.1 - alice [12/Mar/2024:08:15:32 -0500] "GET /../etc/passwd HTTP/1.1" 200 1048600 "http://evil.com" "sqlmap/1.7"

Q2. Nginx Rate-Limit Evasion Detector
Parse Nginx access logs. Detect:

  • IPs rotating user-agents to evade per-UA rate limits (same IP, > 5 distinct UAs, > 50 reqs).
  • Slow-drip attacks: one request every N±2 seconds to avoid threshold triggers.
  • Distributed attack: same target path hit by > 20 distinct IPs in a 60-second bucket.
  • Report each pattern with supporting evidence (IP, UA list, target path, timestamps).

Q3. Zeek HTTP Log C2 Beaconing Detector
Parse Zeek http.log (TSV format). Detect periodic outbound connections (C2 beaconing) by:

  • Computing inter-arrival times between connections per (src_ip, dst_ip, dst_port) tuple.
  • Calculating the jitter coefficient: stddev(intervals) / mean(intervals).
  • Flagging tuples where jitter_coeff < 0.20 AND connection count >= 10.
  • Report: beacon candidates with mean interval, jitter, and connection count.

Zeek log format (tab-separated):

ts          uid      id.orig_h   id.orig_p  id.resp_h    id.resp_p  method  host           uri
1700000000  Cabcdef  10.0.0.5    54321      185.1.2.3    443        GET     c2.evil.com    /check

Q4. DNS Tunneling Detector
Parse DNS query logs. Detect:

  • High-entropy subdomains (Shannon entropy > 3.8 bits/char) — likely base32/hex encoded data.
  • Unusually long FQDNs (> 52 characters before the registered domain).
  • Clients issuing > 100 DNS queries to the same second-level domain in 5 minutes.
  • Suspicious record types from internal clients: TXT, NULL, PRIVATE (type 65399).
  • Report: per-client tunneling risk score and flagged queries.

Q5. NetFlow/IPFIX Log Aggregator
Parse simplified NetFlow records:

timestamp src_ip src_port dst_ip dst_port proto bytes packets flags
2024-03-12T08:00:00Z 10.0.0.5 54321 8.8.8.8 53 UDP 512 1 -

Detect:

  • Port scans: single src_ip contacting > 50 distinct dst_ports on the same dst_ip within 60s.
  • Data exfiltration: single flow with bytes > 50 MB to an external IP on port > 1024.
  • Internal reconnaissance: src_ip contacting > 30 distinct internal dst_ips on port 445 (SMB).
  • Report each alert type with supporting flow records.

Section 2 — Authentication and Identity

Q6. SSH/Auth Brute-Force Detector with IP Clustering
Parse /var/log/auth.log. Detect:

  • Brute-force: >= 10 failed SSH logins from same IP within 60 seconds.
  • Credential stuffing: single IP trying > 20 distinct usernames (regardless of rate).
  • Distributed brute-force: > 5 IPs all targeting the same username within 5 minutes.
  • Successful login following failure burst within 5 minutes (possible compromise).
  • Report each pattern with exact timestamps and involved IPs/usernames.

Q7. Windows Security Event Log Parser (4624/4625/4648)
Parse Windows Security event logs (simplified key=value text format). Detect:

  • Pass-the-Hash indicators: Logon Type 3 + AuthenticationPackage=NTLM + WorkstationName differs from TargetServerName.
  • Lateral movement: single source IP generating Logon Type 3 events to > 5 distinct hosts in 10 minutes.
  • Over-Pass-the-Hash: Logon Type 9 (NewCredentials) followed by Type 3 within 30s from same subject.
  • Kerberos anomalies: Event 4648 (explicit credential logon) outside business hours.
  • Report each detection with a reconstructed event chain.

Q8. OAuth2 Authorization Server Log Analyzer
Parse OAuth2 server logs:

2024-03-12T08:00:00Z client_id=app1 user=alice grant=authorization_code token=tok_abc scope=read:all ip=10.0.0.1 result=OK
2024-03-12T08:01:00Z client_id=app1 user=alice grant=refresh_token token=tok_abc ip=203.0.113.5 result=OK

Detect:

  • Token reuse from multiple distinct IPs within 10 minutes (stolen bearer token).
  • Scope escalation: refresh resulting in wider scope than original grant.
  • High-frequency token refresh: > 20 refreshes in 5 minutes for same token.
  • Expired token replay: result=EXPIRED events with the same token appearing later as result=OK.
  • Report: risk level (HIGH/MEDIUM), evidence, and affected user/client.

Q9. Kerberos TGT/TGS Anomaly Detector
Parse simplified KDC event logs (DC Security Log text format). Detect:

  • Golden Ticket: TGT with ticket lifetime > 10 hours or renewal window > 7 days.
  • Silver Ticket: TGS request for a service with no corresponding TGT request from same host.
  • Kerberoasting: single account requesting TGS for > 5 service accounts within 1 minute.
  • RC4 downgrade: TGS-REQ using etype=RC4 (etype 23) in an environment where AES is default.
  • Overpass-the-Hash: TGT request via AS-REQ with RC4 where the account normally uses AES.

Q10. VPN Session Log Analyzer — Impossible Travel + Exfil
Parse VPN connection logs:

2024-03-12T08:00:00Z user=alice src_ip=203.0.113.10 country=US bytes_in=1024 bytes_out=512000 session_start session_id=s1
2024-03-12T09:00:00Z user=alice src_ip=185.220.101.5 country=RU bytes_in=512 bytes_out=256 session_start session_id=s2

Detect:

  • Impossible travel: same user connecting from IPs in different countries within 2 hours.
  • After-hours access: sessions starting between 22:00–06:00 UTC.
  • Data exfil: bytes_out > 100 MB in a single session.
  • Tor/VPN exit node usage: flag known Tor exit IPs (use a provided list).
  • Report: per-user risk timeline sorted by severity.

Section 3 — Endpoint and Host

Q11. Windows PowerShell ScriptBlock Log Parser
Parse PowerShell ScriptBlock logs (Event ID 4104 text format). Detect:

  • Download cradles: IEX, DownloadString, WebClient, Invoke-Expression with a URL.
  • AMSI bypass patterns: [Ref].Assembly.GetType('System.Management.Automation.AmsiUtils').
  • Encoded commands: -EncodedCommand or -enc followed by Base64; decode and re-analyze.
  • Obfuscation scoring: count string splits, char-array reconstructions, tick-marks, backticks per script.
  • For each flagged script: show decoded payload (if encoded), risk score, and matched indicators.

Q12. Sysmon Process Creation Log Parser (Event ID 1)
Parse Sysmon process creation CSV logs. Detect:

  • LOLBin abuse: certutil.exe, regsvr32.exe, mshta.exe, wscript.exe, cscript.exe,
    rundll32.exe, msiexec.exe with network-loading or decode arguments.
  • Unusual parent-child relationships: Word/Excel/Outlook spawning cmd.exe, powershell.exe, wscript.exe.
  • Process injection precursors: cmd.exe /c spawning a process with a random-looking name.
  • Long command lines (> 500 chars): often indicates obfuscated one-liners.
  • Report: chain of parent → child → grandchild with risk tags.

Q13. sudo / auditd Command Audit Parser
Parse /var/log/audit/audit.log (auditd format) or /var/log/auth.log sudo lines. Detect:

  • Dangerous command patterns: chmod 777, chmod +s (setuid), dd if=/dev/ of=, pipes through base64 -d, curl | bash.
  • Privilege escalation chains: sudo → su → another sudo within 5 minutes on same TTY.
  • Off-hours privileged activity: sudo commands outside 09:00–18:00 local time.
  • Failed sudo attempts followed by success within 60 seconds (possible password guessing).
  • Report each finding with full command, user, TTY, and timestamp.

Q14. Container Runtime Log Parser — Escape Attempt Detector
Parse Docker/containerd runtime and audit logs. Detect:

  • Privileged container launches (--privileged flag or securityContext.privileged: true).
  • Dangerous volume mounts: /etc, /proc, /sys, /var/run/docker.sock, /root.
  • Capability additions: CAP_SYS_ADMIN, CAP_NET_ADMIN, CAP_SYS_PTRACE.
  • seccomp/AppArmor disabled: --security-opt seccomp=unconfined or no AppArmor profile.
  • Namespace escapes: nsenter, unshare executed inside a running container.
  • Report: container ID, image, risk tags, and recommended remediation.

Q15. Memory Dump / Process List Log Analyzer
Parse periodic ps aux snapshots (one per file, filename = timestamp). Detect:

  • New processes appearing between snapshots that have since disappeared (fileless/transient execution).
  • Processes with names resembling system processes but running from unusual paths (masquerading).
  • High-memory processes (RSS > 1 GB) that appear suddenly and are not in an approved list.
  • Processes with (deleted) in /proc/PID/exe link (in-memory execution).
  • Report: delta report across snapshots with risk annotations.

Section 4 — Cloud and Infrastructure

Q16. AWS CloudTrail Anomaly Detector
Parse CloudTrail JSON log records (array of events). Detect:

  • Root account usage: userIdentity.type == "Root" for any non-billing event.
  • IAM privilege escalation: sequence of CreateUser → AttachUserPolicy or PutUserPolicy by same actor.
  • Credential harvesting: GetSecretValue or GetParameter (SSM) calls to > 10 distinct secrets by one principal.
  • Cross-region anomaly: same accessKeyId used in > 2 regions within 10 minutes.
  • Data plane exfil: GetObject calls with > 100 distinct S3 keys by one principal in 5 minutes.
  • Report each finding with the full event chain (actor → actions → resources).

Q17. AWS S3 Access Log Auditor
Parse AWS S3 server access logs. Detect:

  • Bulk object download: > 50 GET requests for unique keys by one IP within 1 hour.
  • Sensitive prefix access: any access to keys matching backup*, export*, pii*, *.sql, *.csv.
  • Cross-account access: requester does not match the bucket owner's account ID prefix.
  • Anonymous access: requester = - on non-public buckets.
  • Failed access storms: > 20 403 AccessDenied responses to same IP within 10 minutes (enumeration).
  • Report with request IDs, S3 URIs, and requester identity.

Q18. Kubernetes Audit Log Analyzer
Parse Kubernetes audit log (JSON, one event per line). Detect:

  • Container exec: verb=create + resource=pods + subresource=exec — interactive shell in pod.
  • Secret reads by unexpected service accounts: verb=get/list on resource=secrets by non-system SA.
  • RBAC privilege escalation: creation of ClusterRoleBinding or RoleBinding granting cluster-admin.
  • Pod with host network/PID/IPC: spec.hostNetwork=true or spec.hostPID=true in pod spec.
  • Anonymous API access: user.username=system:anonymous on any non-health endpoint.
  • Report: severity (CRITICAL/HIGH/MEDIUM), namespace, actor, and affected resource.

Q19. Firewall Rule Hit Aggregator with Policy Gap Finder
Parse firewall allow/deny logs (Palo Alto / iptables simplified format):

2024-03-12T08:00:00Z ALLOW rule=allow_web src=10.0.0.5 dst=1.2.3.4 dport=443 proto=TCP bytes=1024
2024-03-12T08:01:00Z DENY  rule=default_deny src=10.0.0.7 dst=8.8.8.8 dport=4444 proto=TCP bytes=0

Detect:

  • Shadowed rules: a DENY rule that never fires because a broader ALLOW rule precedes it.
  • Policy gaps: traffic to uncommon high ports (> 1024) that is being allowed but not in approved list.
  • East-west movement: internal-to-internal ALLOW on port 445, 3389, 22, 5985 (WinRM).
  • Exfil indicators: large outbound bytes on ALLOW rules to external IPs on non-standard ports.
  • Report: rule hit counts, shadow warnings, and flagged traffic with remediation advice.

Q20. Multi-Log Timeline Reconstructor (Lateral Movement)
Given four log sources in separate files:

  1. Firewall log: timestamp, src_ip, dst_ip, dport, action
  2. Auth log: timestamp, src_ip, username, result
  3. Process log (Sysmon): timestamp, host, parent_process, child_process, cmdline
  4. DNS log: timestamp, client_ip, query, type, answer

Build a correlator that:

  • Joins events by IP within a 5-minute sliding window.
  • Reconstructs attack chains: external_ip → firewall_ALLOW → auth_SUCCESS → process_SPAWN.
  • Assigns a kill-chain stage to each correlated event (Reconnaissance, Initial Access,
    Execution, Persistence, Lateral Movement, Exfiltration).
  • Outputs a human-readable timeline sorted by timestamp with stage annotations.

Section 5 — Advanced Detection and Hunting

Q21. API Gateway Credential Stuffing Detector
Parse API gateway logs with JWT claims embedded:

2024-03-12T08:00:00Z ip=203.0.113.5 method=POST path=/api/login status=401 user=unknown latency_ms=120
2024-03-12T08:00:01Z ip=203.0.113.5 method=POST path=/api/login status=401 user=unknown latency_ms=118
2024-03-12T08:00:05Z ip=203.0.113.5 method=POST path=/api/login status=200 user=alice   latency_ms=135

Detect:

  • Credential stuffing: IP with > 100 failed logins across > 50 distinct usernames.
  • Low-and-slow: distributed stuffing where no single IP exceeds 10 attempts but collectively > 1000 attempts hit /api/login in one hour.
  • Successful logins following a failure burst from any IP for that user within 10 minutes.
  • Latency anomalies: successful logins with latency > 2× the median for that endpoint (caching bypass or unusual code path).
  • Report: risk score per IP, compromised account candidates, and distributed campaign summary.

Q22. WAF Log Attack Pattern Classifier
Parse WAF logs with raw request snippets and matched rule IDs. Classify and report:

  • SQLi signatures: ' OR 1=1, UNION SELECT, --, /**/, SLEEP(, BENCHMARK(.
  • XSS signatures: <script>, onerror=, javascript:, eval(, document.cookie.
  • Path traversal: ../, %2e%2e, %252e%252e, ..%5c.
  • Command injection: ;ls, |id, $(, `cmd`, &&whoami.
  • Bypass techniques: detect URL encoding, double encoding, case variation, null-byte insertion.
  • Report: attack distribution by type, top attacking IPs, bypass technique prevalence, and blocked-vs-passed ratio per rule.

Q23. SIEM-Style Field Normalizer (5-Format Syslog)
Build a universal log normalizer that parses at least 5 different log formats and emits a
unified JSON record with fields: timestamp, host, process, pid, severity,
category, message, src_ip, dst_ip, user, raw.

Formats to support:

  1. RFC 5424 Syslog: <priority>1 2024-03-12T08:00:00Z host proc 1234 - - message
  2. RFC 3164 Syslog: Mar 12 08:00:00 host proc[1234]: message
  3. JSON log: {"time":"...","level":"ERROR","host":"...","msg":"..."}
  4. Windows Event (key=value): TimeCreated=... EventID=4625 SubjectUserName=alice ...
  5. Apache CLF: ip - user [timestamp] "METHOD path HTTP/1.1" status bytes

For each unrecognized line, emit a record with severity=UNKNOWN and preserve raw.

Q24. Certificate Transparency Log Monitor
Parse CT log entries (simplified JSON, one per line):

{
  "timestamp": "2024-03-12T08:00:00Z",
  "issuer": "DigiCert",
  "subject": "CN=evil.acme.com",
  "san": ["evil.acme.com", "*.evil.acme.com"],
  "not_before": "2024-03-12",
  "not_after": "2024-06-12",
  "serial": "deadbeef"
}

Given a set of monitored domains (e.g., acme.com, corp.acme.com), detect:

  • Certificates issued for your domain by unexpected CAs (maintain an approved CA allowlist).
  • Wildcard certificates for your domain: *.acme.com — always alert.
  • Short-lived certificates (validity < 30 days) — often used for phishing infrastructure.
  • SAN mismatches: cert subject does not appear in SAN list.
  • Lookalike domains: substrings or Levenshtein distance <= 2 from monitored domains.
  • Report: per-finding with cert details, risk level, and recommended action.

Q25. Threat Hunt IOC Cross-Reference Pivot Tool
Given:

  • A list of IOCs (IP addresses, domains, file hashes, email addresses) from a threat feed.
  • A corpus of mixed log files (HTTP access log, DNS log, auth log) in a directory.

Build a pivot tool that:

  • Scans all log files for any IOC match (exact match for IPs/hashes, subdomain match for domains).
  • For each hit: extracts 3 lines of context before and after the matching line.
  • Builds an IOC hit timeline: (ioc, log_file, line_number, timestamp, context).
  • Deduplicates repeated hits of the same IOC in the same file within 60-second windows.
  • Reports: IOC hit summary table, timeline sorted by first seen, and a per-IOC pivot showing all log sources where it appears.
  • Complexity goal: O(L × I) where L = total log lines, I = number of IOCs.
