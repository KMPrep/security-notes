"""
=============================================================================
Q01 [Hard] — Streaming Top-K Source IPs
=============================================================================

PROBLEM
-------
Given an *unbounded* stream of network events (each event contains at least a
`src` IP field), maintain an approximate list of the top-K most-frequent
source IPs using **bounded memory** — i.e. memory must NOT grow with the
number of distinct IPs seen.

CLARIFYING QUESTIONS
--------------------
1. Input format — JSONL stream, or already-parsed dicts?
   → Assume parsed dicts: {"src": "1.2.3.4", ...}
2. What is K?  → Configurable, default 10.
3. Exact or approximate top-K?
   → Approximate is fine; must explain the error bound.
4. Should malformed records (missing "src") be skipped silently?
   → Yes, skip and count as "malformed".
5. Are IPv4 and IPv6 both possible?
   → Yes; treat them as opaque strings for counting.
6. Is output sorted by frequency descending?
   → Yes.
7. Batch (offline) vs streaming (online)?
   → Streaming; result queryable at any time.
8. Distributed?
   → Single-node for this solution; note how to scale.

=============================================================================
APPROACH 1 — NAIVE  (exact, unbounded memory)
=============================================================================
Use a plain dict to count every distinct IP.
At query time, call heapq.nlargest(k, counts, key=counts.get).

  Time : O(1) per event insert, O(D log K) per query  (D = distinct IPs)
  Space: O(D) — grows unboundedly with distinct IPs ← PROBLEM for streams

Suitable only for bounded batch inputs or small cardinality streams.

=============================================================================
APPROACH 2 — OPTIMAL  (Space-Saving / Misra-Gries, exact top-K guarantee)
=============================================================================
The **Space-Saving** algorithm (Metwally et al., 2005) tracks exactly m
counters (m ≥ K).  When an IP is already tracked, increment its counter.
When an IP is new AND there are fewer than m slots, add it with count 1.
When an IP is new AND all m slots are taken, evict the minimum-count slot
and replace it with the new IP, inheriting (min_count + 1).

Guarantee: any IP whose true frequency > N/m is guaranteed to appear in the
tracker.  For K heavy-hitters, use m ≥ K / ε where ε is the acceptable
frequency error fraction (e.g. m = 10K covers the top K with < 10% error).

  Time : O(log m) per event  (min-heap for eviction)
  Space: O(m)  — strictly bounded

=============================================================================
APPROACH 3 — COUNT-MIN SKETCH + Min-Heap  (fully probabilistic)
=============================================================================
Use a Count-Min Sketch (CMS) for frequency estimation (O(1) per update,
bounded array size) combined with a min-heap of size K to maintain candidates.
CMS over-estimates but never under-estimates.  Error bound: ε with probability
1-δ for chosen width/depth.

  Time : O(d) per update  (d = CMS depth, typically 5-7)
  Space: O(d × w + K)  — fully bounded

This approach is preferred when the cardinality of distinct values is
astronomically large (e.g. full IPv6 space) and even O(m) dict is too large.

=============================================================================
"""

from __future__ import annotations

import heapq
import hashlib
import math
from collections import defaultdict
from typing import Dict, Iterator, List, Tuple


# ---------------------------------------------------------------------------
# APPROACH 1 — Naive exact counter (unbounded memory)
# ---------------------------------------------------------------------------

class NaiveTopKIPs:
    """Exact top-K using an unbounded dict.  Use only for small/batch inputs."""

    def __init__(self, k: int = 10) -> None:
        self.k = k
        self.counts: Dict[str, int] = defaultdict(int)
        self.malformed = 0

    def process(self, event: dict) -> None:
        src = event.get("src")
        if not src or not isinstance(src, str):
            self.malformed += 1
            return
        self.counts[src] += 1

    def top_k(self) -> List[Tuple[str, int]]:
        """Return top-K IPs sorted by frequency descending."""
        return heapq.nlargest(self.k, self.counts.items(), key=lambda x: x[1])


# ---------------------------------------------------------------------------
# APPROACH 2 — Space-Saving (Misra-Gries variant) — RECOMMENDED
# ---------------------------------------------------------------------------

class SpaceSavingTopKIPs:
    """
    Bounded-memory approximate top-K using the Space-Saving algorithm.

    Tracks at most `capacity` counters.  Any IP with true frequency > N/capacity
    is guaranteed to appear in the result.  Set capacity = k / epsilon for a
    desired error fraction epsilon.

    Data structure:
      - A dict mapping ip → (estimated_count, max_error)
      - A min-heap of (count, ip) for O(log m) eviction of the minimum counter
    """

    def __init__(self, k: int = 10, capacity: int = 100) -> None:
        if capacity < k:
            raise ValueError("capacity must be >= k")
        self.k = k
        self.capacity = capacity
        self._counts: Dict[str, int] = {}   # ip → estimated count
        self._errors: Dict[str, int] = {}   # ip → max over-count error
        self._heap: List[Tuple[int, str]] = []  # min-heap of (count, ip)
        self.total_events = 0
        self.malformed = 0

    def process(self, event: dict) -> None:
        src = event.get("src")
        if not src or not isinstance(src, str):
            self.malformed += 1
            return

        self.total_events += 1

        if src in self._counts:
            # Already tracked — increment
            self._counts[src] += 1
            # Lazy heap update: rebuild is avoided by using "tombstone" approach.
            # For a production system use a Fibonacci heap or indexed heap.
            # Here we do a lazy push and tolerate stale entries during eviction.
            heapq.heappush(self._heap, (self._counts[src], src))
        elif len(self._counts) < self.capacity:
            # Free slot — add new IP
            self._counts[src] = 1
            self._errors[src] = 0
            heapq.heappush(self._heap, (1, src))
        else:
            # No free slot — evict the minimum
            min_count = self._evict_min()
            new_count = min_count + 1
            self._counts[src] = new_count
            self._errors[src] = min_count   # worst-case over-count error
            heapq.heappush(self._heap, (new_count, src))

    def _evict_min(self) -> int:
        """Remove and return the count of the IP with the minimum estimated count."""
        while self._heap:
            count, ip = heapq.heappop(self._heap)
            if ip in self._counts and self._counts[ip] == count:
                # Valid (non-stale) entry
                del self._counts[ip]
                del self._errors[ip]
                return count
        return 0  # should not reach here

    def top_k(self) -> List[Tuple[str, int, int]]:
        """
        Return top-K IPs as (ip, estimated_count, max_error) sorted descending.
        max_error is the upper bound on over-counting introduced by eviction.
        """
        candidates = sorted(self._counts.items(), key=lambda x: x[1], reverse=True)
        return [(ip, cnt, self._errors.get(ip, 0)) for ip, cnt in candidates[: self.k]]

    def frequency_guarantee(self) -> float:
        """Minimum true frequency fraction guaranteed to be in tracker."""
        return 1.0 / self.capacity if self.total_events > 0 else 0.0


# ---------------------------------------------------------------------------
# APPROACH 3 — Count-Min Sketch + Min-Heap
# ---------------------------------------------------------------------------

class CountMinSketch:
    """
    Count-Min Sketch for frequency estimation.

    width  : number of buckets per row (controls error magnitude ε ≈ e/width)
    depth  : number of hash functions / rows (controls failure prob δ ≈ e^-depth)
    Typical: width=2000, depth=7 → ε ≈ 0.0014, δ ≈ 0.001
    """

    def __init__(self, width: int = 2000, depth: int = 7) -> None:
        self.width = width
        self.depth = depth
        self.table = [[0] * width for _ in range(depth)]
        # Pre-compute distinct seeds via SHA256-derived offsets
        self._seeds = [i * 2654435761 & 0xFFFFFFFF for i in range(depth)]

    def _hashes(self, key: str) -> List[int]:
        encoded = key.encode()
        result = []
        for seed in self._seeds:
            # Combine SHA256 digest bytes with seed for independence
            h = int(hashlib.sha256(seed.to_bytes(4, "little") + encoded).hexdigest(), 16)
            result.append(h % self.width)
        return result

    def add(self, key: str, count: int = 1) -> None:
        for row, col in enumerate(self._hashes(key)):
            self.table[row][col] += count

    def estimate(self, key: str) -> int:
        return min(self.table[row][col] for row, col in enumerate(self._hashes(key)))


class CMSTopKIPs:
    """
    Top-K IPs using Count-Min Sketch for counting + min-heap of size K
    for candidate tracking.

    Memory: O(width × depth + K)  — fully bounded regardless of cardinality.
    Limitation: CMS only over-estimates; we may include false positives but
    never miss a true heavy hitter.
    """

    def __init__(self, k: int = 10, width: int = 2000, depth: int = 7) -> None:
        self.k = k
        self.sketch = CountMinSketch(width, depth)
        # min-heap of (estimated_count, ip)
        self._heap: List[Tuple[int, str]] = []
        self._heap_set: set = set()   # O(1) membership check
        self.malformed = 0

    def process(self, event: dict) -> None:
        src = event.get("src")
        if not src or not isinstance(src, str):
            self.malformed += 1
            return

        self.sketch.add(src)
        est = self.sketch.estimate(src)

        if src in self._heap_set:
            # Re-push with updated count; stale entries tolerated (lazy deletion)
            heapq.heappush(self._heap, (est, src))
        elif len(self._heap_set) < self.k:
            heapq.heappush(self._heap, (est, src))
            self._heap_set.add(src)
        else:
            # Check if this IP deserves to replace the current minimum
            min_count, min_ip = self._heap[0]
            if est > min_count:
                # Evict the min candidate
                while self._heap and self._heap[0][1] not in self._heap_set:
                    heapq.heappop(self._heap)
                if self._heap:
                    _, evicted = heapq.heappop(self._heap)
                    self._heap_set.discard(evicted)
                heapq.heappush(self._heap, (est, src))
                self._heap_set.add(src)

    def top_k(self) -> List[Tuple[str, int]]:
        """Return top-K IPs sorted by estimated frequency descending."""
        valid = {ip: self.sketch.estimate(ip) for ip in self._heap_set}
        return sorted(valid.items(), key=lambda x: x[1], reverse=True)[: self.k]

def _make_events(ip_counts: Dict[str, int]) -> List[dict]:
    events = []
    for ip, cnt in ip_counts.items():
        events.extend({"src": ip, "dst_port": 443} for _ in range(cnt))
    return events


def test_naive():
    tracker = NaiveTopKIPs(k=3)

    # Empty input
    assert tracker.top_k() == []

    # Normal input
    events = _make_events({"1.1.1.1": 100, "2.2.2.2": 50, "3.3.3.3": 200, "4.4.4.4": 10})
    for e in events:
        tracker.process(e)
    top = tracker.top_k()
    assert top[0][0] == "3.3.3.3"
    assert top[1][0] == "1.1.1.1"
    assert top[2][0] == "2.2.2.2"

    # Malformed input
    tracker.process({})
    tracker.process({"src": None})
    tracker.process({"src": 12345})
    assert tracker.malformed == 3

    print("NaiveTopKIPs: all tests passed")


def test_space_saving():
    tracker = SpaceSavingTopKIPs(k=3, capacity=20)

    # Empty
    assert tracker.top_k() == []

    # Normal: clear winner should appear
    events = _make_events({"10.0.0.1": 500, "10.0.0.2": 300, "10.0.0.3": 200,
                           **{f"10.0.1.{i}": 1 for i in range(50)}})
    for e in events:
        tracker.process(e)
    top_ips = [ip for ip, _, _ in tracker.top_k()]
    # The three dominant IPs must appear (Space-Saving guarantee)
    assert "10.0.0.1" in top_ips, f"Expected 10.0.0.1 in {top_ips}"

    # Malformed
    tracker.process({"src": ""})
    assert tracker.malformed == 1

    print("SpaceSavingTopKIPs: all tests passed")


def test_cms_top_k():
    tracker = CMSTopKIPs(k=3, width=1000, depth=5)

    # Empty
    assert tracker.top_k() == []

    # Normal
    events = _make_events({"192.168.1.1": 400, "192.168.1.2": 250, "192.168.1.3": 150,
                           **{f"10.0.{i}.{j}": 1 for i in range(5) for j in range(5)}})
    for e in events:
        tracker.process(e)
    top_ips = [ip for ip, _ in tracker.top_k()]
    assert "192.168.1.1" in top_ips, f"Expected 192.168.1.1 in {top_ips}"

    # Malformed
    tracker.process({"other_field": "x"})
    assert tracker.malformed == 1

    print("CMSTopKIPs: all tests passed")

def simulate_stream() -> None:
    """Simulate an unbounded stream and compare all three approaches."""
    import random

    # Simulate 100k events: Zipf-distributed IPs (few dominate)
    random.seed(42)
    ips = [f"10.0.{i}.{j}" for i in range(256) for j in range(256)]  # 65536 IPs
    weights = [1 / (rank + 1) for rank in range(len(ips))]  # Zipf
    total = sum(weights)
    weights = [w / total for w in weights]

    stream = ({"src": random.choices(ips, weights=weights)[0]} for _ in range(100_000))

    naive = NaiveTopKIPs(k=5)
    space_saving = SpaceSavingTopKIPs(k=5, capacity=100)
    cms = CMSTopKIPs(k=5, width=2000, depth=7)

    for event in stream:
        naive.process(event)
        space_saving.process(event)
        cms.process(event)

    print("\n=== Top-5 IPs (Naive exact) ===")
    for ip, cnt in naive.top_k():
        print(f"  {ip:20s}  count={cnt}")

    print("\n=== Top-5 IPs (Space-Saving) ===")
    for ip, cnt, err in space_saving.top_k():
        print(f"  {ip:20s}  est={cnt}  max_error={err}")

    print("\n=== Top-5 IPs (CMS + Heap) ===")
    for ip, cnt in cms.top_k():
        print(f"  {ip:20s}  est={cnt}")

"""
THREAT MODEL
  Attackers generating high-volume traffic (DDoS, brute-force, scanning) will
  appear as heavy hitters in source-IP frequency.  Bounded-memory top-K gives
  the SOC a real-time view of top offenders even on 100 Gbps links.

FAILURE MODES
  - Space-Saving can over-count a rare IP if eviction happens at the wrong time.
    The max_error field bounds this.
  - CMS only over-estimates; under-counting is impossible → no missed heavy hitters.
  - Both algorithms miss attackers who deliberately spread traffic across many IPs
    (distributed attack) — combine with per-subnet aggregation.

FALSE POSITIVES / NEGATIVES
  - False positive: a bursty-but-legitimate CDN egress IP could rank highly.
    Enrich with IP reputation / ASN allowlisting.
  - False negative: low-and-slow attackers below the heavy-hitter threshold.
    Use a separate rate-based sliding window detector for those.

PRODUCTION HARDENING
  - Run one tracker per 5-minute epoch; rotate and archive.
  - Feed output to downstream alerting if any IP exceeds e.g. 1% of all traffic.
  - Use Redis Sorted Sets for distributed top-K across multiple collector nodes.
  - Combine with GeoIP enrichment before surfacing to analysts.
"""
