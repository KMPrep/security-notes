"""
=============================================================================
Q02 [Hard] — Heavy Hitters in Security Logs (Count-Min Sketch deep-dive)
=============================================================================

PROBLEM
-------
Implement a Count-Min Sketch (CMS) to detect "heavy-hitter" IPs whose traffic
exceeds a configurable threshold fraction φ of the total event stream.
Unlike Q01 (which tracks top-K), here the caller specifies a *frequency
threshold* (e.g. "flag any IP that is responsible for > 1% of all events").

CLARIFYING QUESTIONS
--------------------
1. Is the threshold absolute (event count) or relative (fraction of stream)?
   → Support both: absolute threshold and fraction φ.
2. What is an acceptable error?
   → Allow up to ε fraction false over-count; ε < φ so real heavy hitters
     always exceed the threshold minus error.
3. Should we also report *which* IPs exceed the threshold, or just counts?
   → Report IP + estimated count + whether it exceeds threshold.
4. Single-field (src IP) or multi-field (src+dst pair)?
   → Demonstrate both; multi-field uses a composite key.
5. Do we need to expire old counts (sliding window)?
   → Yes, show a decay / epoch-reset variant.
6. Distributed?
   → CMSes are mergeable — show merge() method.
7. Malformed records?
   → Skip silently; expose malformed_count.

=============================================================================
APPROACH 1 — NAIVE  (dict-based exact count)
=============================================================================
Count every distinct IP in a plain dict, iterate to find heavy hitters.
  Time : O(1) amortised per insert, O(D) per query (D = distinct IPs)
  Space: O(D) — unbounded for large cardinality streams

=============================================================================
APPROACH 2 — COUNT-MIN SKETCH  (optimal probabilistic)
=============================================================================
CMS parameters:
  width w = ceil(e / ε)       where e ≈ 2.718  → ε-bounded over-estimate
  depth d = ceil(ln(1/δ))     → failure prob ≤ δ per query

Guarantee: estimated_count ≤ true_count + ε × N
  where N = total events seen.

For heavy-hitter detection with threshold φ:
  Set ε = φ/2 so that an IP with true count > φ×N will have estimate > (φ/2)×N
  and a non-heavy-hitter with true count < (φ/2)×N will never exceed φ×N.

  Time : O(d) per insert/query  (typically d=7, so O(1) practical)
  Space: O(w × d) = O(e/ε × ln(1/δ)) — fully bounded

=============================================================================
APPROACH 3 — LOSSY COUNTING (Manku & Motwani 2002)
=============================================================================
Divide the stream into epochs of size 1/ε.  Within each epoch increment
counts; at epoch boundary decrement all by 1 and evict zeros.
Guarantee: any IP with frequency ≥ φ×N is reported; false positives have
frequency ≥ (φ-ε)×N.

  Space: O(1/ε)  entries (larger constant than CMS)
  Advantage: exact counts for surviving entries (no hash collisions)

=============================================================================
"""

from __future__ import annotations

import hashlib
import math

from typing import Dict, Iterable, List, Optional, Tuple


# ---------------------------------------------------------------------------
# APPROACH 1 — Naive exact dict
# ---------------------------------------------------------------------------

class NaiveHeavyHitters:
    """Exact heavy-hitter detection using an unbounded dict."""

    def __init__(self, phi: float = 0.01) -> None:
        if not 0 < phi < 1:
            raise ValueError("phi must be in (0, 1)")
        self.phi = phi
        self._counts: Dict[str, int] = defaultdict(int)
        self.total = 0
        self.malformed_count = 0

    def add(self, event: dict, key: str = "src") -> None:
        val = event.get(key)
        if not val or not isinstance(val, str):
            self.malformed_count += 1
            return
        self._counts[val] += 1
        self.total += 1

    def heavy_hitters(self) -> List[Tuple[str, int, float]]:
        """Return (ip, count, fraction) for IPs above threshold φ."""
        threshold = self.phi * self.total
        return sorted(
            [(ip, cnt, cnt / self.total) for ip, cnt in self._counts.items()
             if cnt >= threshold],
            key=lambda x: x[1], reverse=True,
        )


# ---------------------------------------------------------------------------
# APPROACH 2 — Count-Min Sketch  (RECOMMENDED)
# ---------------------------------------------------------------------------

class CountMinSketch:
    """
    Count-Min Sketch with configurable error bounds.

    Parameters:
      epsilon : maximum over-count fraction  (e.g. 0.005 → 0.5% of N)
      delta   : failure probability per query (e.g. 0.001 → 0.1% failure)
    """

    def __init__(self, epsilon: float = 0.005, delta: float = 0.001) -> None:
        if not 0 < epsilon < 1:
            raise ValueError("epsilon must be in (0, 1)")
        if not 0 < delta < 1:
            raise ValueError("delta must be in (0, 1)")
        self.epsilon = epsilon
        self.delta = delta
        self.width: int = math.ceil(math.e / epsilon)
        self.depth: int = math.ceil(math.log(1.0 / delta))
        # 2-D list: depth rows × width columns
        self.table: List[List[int]] = [[0] * self.width for _ in range(self.depth)]
        self.total: int = 0
        # Pre-compute per-row hash salts for independence
        self._salts: List[bytes] = [
            (i * 0x9e3779b97f4a7c15).to_bytes(8, "little") for i in range(self.depth)
        ]

    def _hash(self, row: int, key: str) -> int:
        h = hashlib.sha256(self._salts[row] + key.encode()).digest()
        return int.from_bytes(h[:4], "little") % self.width

    def add(self, key: str, count: int = 1) -> None:
        for row in range(self.depth):
            self.table[row][self._hash(row, key)] += count
        self.total += count

    def estimate(self, key: str) -> int:
        """Return estimated count.  True count ≤ estimate ≤ true count + ε×N."""
        return min(self.table[row][self._hash(row, key)] for row in range(self.depth))

    def merge(self, other: "CountMinSketch") -> "CountMinSketch":
        """
        Merge another CMS into this one (both must have identical dimensions).
        Enables distributed aggregation: each shard builds its own CMS,
        then all are merged at the coordinator.
        """
        if self.width != other.width or self.depth != other.depth:
            raise ValueError("Sketches must have identical dimensions to merge")
        merged = CountMinSketch.__new__(CountMinSketch)
        merged.epsilon = self.epsilon
        merged.delta = self.delta
        merged.width = self.width
        merged.depth = self.depth
        merged._salts = self._salts  # same salts = same hash functions
        merged.table = [
            [self.table[r][c] + other.table[r][c] for c in range(self.width)]
            for r in range(self.depth)
        ]
        merged.total = self.total + other.total
        return merged

    def memory_bytes(self) -> int:
        """Approximate memory usage in bytes."""
        return self.depth * self.width * 8  # 64-bit counters


class CMSHeavyHitterDetector:
    """
    Heavy-hitter detection combining Count-Min Sketch with a candidate set.

    Design:
    - CMS handles frequency estimation with bounded memory.
    - A secondary set tracks *candidate* heavy hitters seen so far.
      Candidates are only added when their CMS estimate crosses φ×N.
    - This avoids scanning the full sketch at query time.
    """

    def __init__(self, phi: float = 0.01, epsilon: float = None,
                 delta: float = 0.001, key_field: str = "src") -> None:
        if not 0 < phi < 1:
            raise ValueError("phi must be in (0, 1)")
        # Set ε = φ/2 so the error zone sits squarely below the threshold
        if epsilon is None:
            epsilon = phi / 2
        self.phi = phi
        self.key_field = key_field
        self.sketch = CountMinSketch(epsilon=epsilon, delta=delta)
        self._candidates: set = set()
        self.malformed_count = 0

    def add(self, event: dict) -> None:
        key = event.get(self.key_field)
        if not key or not isinstance(key, str):
            self.malformed_count += 1
            return
        self.sketch.add(key)
        # Lazily promote to candidate if estimate crosses threshold
        est = self.sketch.estimate(key)
        if est >= self.phi * self.sketch.total:
            self._candidates.add(key)

    def add_composite(self, event: dict, fields: List[str]) -> None:
        """Track a composite key such as (src, dst) pairs."""
        parts = [str(event.get(f, "")) for f in fields]
        if any(not p for p in parts):
            self.malformed_count += 1
            return
        composite_key = "|".join(parts)
        self.sketch.add(composite_key)
        est = self.sketch.estimate(composite_key)
        if est >= self.phi * self.sketch.total:
            self._candidates.add(composite_key)

    def heavy_hitters(self) -> List[Tuple[str, int, float]]:
        """
        Return confirmed heavy hitters: candidates whose *current* estimate
        still exceeds φ×N (prunes false promotions from early in the stream).
        """
        threshold = self.phi * self.sketch.total
        result = []
        for candidate in self._candidates:
            est = self.sketch.estimate(candidate)
            if est >= threshold:
                result.append((candidate, est, est / max(self.sketch.total, 1)))
        return sorted(result, key=lambda x: x[1], reverse=True)

    def is_heavy_hitter(self, key: str) -> bool:
        """O(d) point query — is this key currently a heavy hitter?"""
        return self.sketch.estimate(key) >= self.phi * self.sketch.total

class LossyCountingHeavyHitters:
    """
    Lossy Counting algorithm (Manku & Motwani 2002).

    Exact counts for surviving entries; guarantees no false negatives for
    IPs with frequency ≥ φ × N, and no false positives above (φ-ε) × N.

    epoch_size = floor(1 / epsilon)
    """

    def __init__(self, phi: float = 0.01, epsilon: float = 0.005,
                 key_field: str = "src") -> None:
        if epsilon >= phi:
            raise ValueError("epsilon must be < phi")
        self.phi = phi
        self.epsilon = epsilon
        self.key_field = key_field
        self.epoch_size: int = max(1, math.floor(1.0 / epsilon))
        # (key) → (count, max_error_delta)
        self._buckets: Dict[str, Tuple[int, int]] = {}
        self.total = 0
        self.malformed_count = 0
        self._current_epoch = 1

    def add(self, event: dict) -> None:
        key = event.get(self.key_field)
        if not key or not isinstance(key, str):
            self.malformed_count += 1
            return
        self.total += 1
        if key in self._buckets:
            cnt, delta = self._buckets[key]
            self._buckets[key] = (cnt + 1, delta)
        else:
            # delta = current_epoch - 1 (the max over-count from previous epochs)
           self._buckets[key] = (1, self._current_epoch - 1)

        # At epoch boundary, decrement all counts and prune zeros
        if self.total % self.epoch_size == 0:
            self._prune()
            self._current_epoch += 1

    def _prune(self) -> None:
        to_delete = [k for k, (cnt, delta) in self._buckets.items()
                     if cnt + delta <= self._current_epoch]
        for k in to_delete:
            del self._buckets[k]

    def heavy_hitters(self) -> List[Tuple[str, int, float]]:
        """Return IPs with frequency estimate ≥ (φ - ε) × N."""
        threshold = (self.phi - self.epsilon) * self.total
        result = []
        for key, (cnt, _) in self._buckets.items():
            if cnt >= threshold:
                result.append((key, cnt, cnt / max(self.total, 1)))

def _stream(ip_map: Dict[str, int]) -> List[dict]:
    events = []
    for ip, cnt in ip_map.items():
        events.extend({"src": ip} for _ in range(cnt))
    return events


def test_naive_hh():
    detector = NaiveHeavyHitters(phi=0.1)
    assert detector.heavy_hitters() == []

    for e in _stream({"10.0.0.1": 500, "10.0.0.2": 300, "10.0.0.3": 50,
                      "10.0.0.4": 50, "10.0.0.5": 100}):
        detector.add(e)
    hh = [ip for ip, _, _ in detector.heavy_hitters()]
    assert "10.0.0.1" in hh
    assert "10.0.0.2" in hh

    detector.add({})
    assert detector.malformed_count == 1
    print("NaiveHeavyHitters: passed")


def test_cms_hh():
    detector = CMSHeavyHitterDetector(phi=0.1, delta=0.001)
    assert detector.heavy_hitters() == []

    events = _stream({"10.0.0.1": 600, "10.0.0.2": 300,
                      **{f"10.1.{i}.1": 5 for i in range(20)}})
    for e in events:
        detector.add(e)

    hh = [ip for ip, _, _ in detector.heavy_hitters()]
    # 10.0.0.1 ~50% and 10.0.0.2 ~25% of traffic; both > 10% threshold
    assert "10.0.0.1" in hh, f"Expected 10.0.0.1 in {hh}"
    assert "10.0.0.2" in hh, f"Expected 10.0.0.2 in {hh}"

    # Composite key test
    det2 = CMSHeavyHitterDetector(phi=0.5, key_field="src")
    for _ in range(100):
        det2.add_composite({"src": "1.1.1.1", "dst": "2.2.2.2"}, ["src", "dst"])
    for _ in range(10):
        det2.add_composite({"src": "3.3.3.3", "dst": "4.4.4.4"}, ["src", "dst"])
    hh2 = [k for k, _, _ in det2.heavy_hitters()]
    assert "1.1.1.1|2.2.2.2" in hh2

    print("CMSHeavyHitterDetector: passed")


def test_cms_merge():
    """Verify two independent sketches merge correctly."""
    s1 = CountMinSketch(epsilon=0.01, delta=0.001)
    s2 = CountMinSketch(epsilon=0.01, delta=0.001)
    for _ in range(100):
        s1.add("hot_ip")
    for _ in range(200):
        s2.add("hot_ip")
    merged = s1.merge(s2)
    assert merged.estimate("hot_ip") >= 300
    assert merged.total == 300
    print("CMS merge: passed")


def test_lossy_counting():
    detector = LossyCountingHeavyHitters(phi=0.1, epsilon=0.05)
    assert detector.heavy_hitters() == []

    events = _stream({"192.168.1.1": 500, "192.168.1.2": 400,
                      **{f"10.0.{i}.1": 2 for i in range(50)}})
    for e in events:
        detector.add(e)
    hh = [ip for ip, _, _ in detector.heavy_hitters()]
    assert "192.168.1.1" in hh, f"Expected 192.168.1.1 in {hh}"
    print("LossyCountingHeavyHitters: passed")

"""
THREAT MODEL
  Heavy-hitter detection identifies IPs responsible for a disproportionate
  share of traffic or failed auth events — direct indicators of:
  - DDoS amplification sources
  - Brute-force / credential stuffing attackers
  - Exfiltration egress sources (large byte volumes)
  - C2 beaconing originators

WHY CMS OVER NAIVE DICT
  At 100 Gbps with 64-byte packets, a line-rate event stream exceeds 100M
  events/second.  A naive dict storing all distinct /32 IPv4 or /128 IPv6
  sources can easily require gigabytes of RAM.  A CMS with ε=0.001 and δ=0.001
  uses only 2718 × 7 × 8 bytes ≈ 150 KB regardless of stream size.

DISTRIBUTED USE CASE
  Deploy one CMS per Kafka partition (one per network tap).  Every minute,
  serialize and ship to a coordinator which calls merge().  The coordinator
  runs heavy_hitters() and emits alerts.  Sketch serialisation is O(w×d) and
  fits in a single UDP packet for small ε.

ERROR ANALYSIS EXAMPLE
  φ=0.01, ε=0.005, N=10,000,000 events
  - Any IP with true count > 100,000 will have estimate > 50,000 → reported ✓
  - Any IP with true count < 50,000 will have estimate < 100,000 → not reported ✓
  - IPs in the grey zone (50k–100k) may or may not appear → acceptable trade-off.

RESIDUAL RISK
  - Hash collisions in CMS can inflate estimates for innocent IPs.
    Mitigate: use d≥7 independent hash functions; accept false-positive rate δ.
  - Epoch-based decay (Lossy Counting) introduces latency before stale counts
    are cleared.  Use small epoch size for low-latency detection.
"""
