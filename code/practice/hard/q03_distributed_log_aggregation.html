"""
=============================================================================
Q03 [Hard] — Distributed Log Aggregation with Clock Skew
=============================================================================

PROBLEM
-------
Logs arrive from many machines.  Each machine has its own wall clock that may
drift up to S seconds from true time.  Design and implement a merge strategy
that produces a globally ordered event sequence that is *approximately*
correct given the clock skew.

CLARIFYING QUESTIONS
--------------------
1. Input format — list-of-lists (one list per machine) or a single stream
   with a "machine_id" field?
   → Support both; show list-of-lists as primary, streaming as secondary.
2. What is the known or estimated maximum clock skew S?
   → Configurable; default 5 seconds.
3. Are timestamps ISO-8601 strings or epoch floats?
   → ISO-8601; parse to float internally.
4. Can a single machine produce out-of-order events relative to itself?
   → Assume each machine's local log is monotonically increasing.
5. Should we drop events that are too old (outside the watermark)?
   → Expose a drop_late parameter; default False (buffer late events).
6. What should happen for malformed timestamps?
   → Skip and count as malformed.
7. Distributed / streaming or offline batch?
   → Show offline K-way merge first; then sketch a streaming watermark design.

=============================================================================
APPROACH 1 — NAIVE  (collect-all then sort)
=============================================================================
Collect every event from every machine into one list, then sort by timestamp.
  Time : O(N log N)   N = total events
  Space: O(N)  — must hold all events in memory simultaneously ← problem for
                 large N

=============================================================================
APPROACH 2 — K-WAY MERGE WITH MIN-HEAP  (optimal for offline batch)
=============================================================================
Use a min-heap of size K (one entry per machine).  Each heap entry is
(timestamp, machine_id, event).  At each step, pop the minimum, emit it,
and push the next event from that machine.

  Time : O(N log K)   — far better than O(N log N) when K << N
  Space: O(K + output buffer)  — only one event per machine in memory at once

=============================================================================
APPROACH 3 — WATERMARK-BASED STREAMING  (optimal for real-time)
=============================================================================
Maintain a per-machine "latest timestamp seen" cursor.
Global watermark W = min(cursor_i) − S  (account for clock skew S).
Events with ts ≤ W are safe to emit in order.
Events with ts > W are buffered per machine.
When watermark advances, flush buffered events up to the new W using a heap.

  Time : O(log K) per event emit
  Space: O(K + events within the skew window S) — bounded by event rate × S

=============================================================================
"""

from __future__ import annotations

import heapq
from collections import defaultdict
from datetime import datetime, timezone
from typing import Dict, Generator, Iterator, List, Optional, Tuple


# ---------------------------------------------------------------------------
# Helpers
# ---------------------------------------------------------------------------

def _parse_ts(ts_str: str) -> Optional[float]:
    """Parse ISO-8601 timestamp to epoch float.  Returns None if malformed."""
    if not ts_str:
        return None
    try:
        # Handle both 'Z' suffix and +00:00
        ts_str = ts_str.replace("Z", "+00:00")
        dt = datetime.fromisoformat(ts_str)
        return dt.timestamp()
    except (ValueError, TypeError):
        return None


# ---------------------------------------------------------------------------
# APPROACH 1 — Naive collect-all sort
# ---------------------------------------------------------------------------

class NaiveLogMerger:
    """Collect all events into one list then sort.  O(N log N) time, O(N) space."""

    def __init__(self) -> None:
        self._events: List[Tuple[float, str, dict]] = []
        self.malformed_count = 0

    def add_machine_log(self, machine_id: str, events: List[dict]) -> None:
        for event in events:
            ts = _parse_ts(event.get("ts", ""))
            if ts is None:
                self.malformed_count += 1
                continue
            self._events.append((ts, machine_id, event))

    def merged(self) -> List[dict]:
        self._events.sort(key=lambda x: x[0])
        return [dict(e, _machine=m) for ts, m, e in self._events]


# ---------------------------------------------------------------------------
# APPROACH 2 — K-Way Merge with Min-Heap (offline batch, optimal)
# ---------------------------------------------------------------------------

class KWayLogMerger:
    """
    K-way merge using a min-heap.  Each machine contributes an iterator.
    Memory at any time: O(K) heap entries + the output list.
    Time: O(N log K).

    Clock-skew tolerance: after popping an event from machine M, we
    immediately push M's next event.  We do NOT re-order across machines
    beyond the natural heap ordering — the caller must ensure each machine's
    log is locally sorted.  For cross-machine skew, see WatermarkLogMerger.
    """

    def merge(
        self,
        machine_logs: Dict[str, List[dict]],
        ts_field: str = "ts",
    ) -> List[dict]:
        """
        machine_logs: {machine_id: [event, ...]}
        Returns merged list in approximately-ascending timestamp order.
        """
        malformed = 0
        # Build iterators and seed the heap
        heap: List[Tuple[float, str, Iterator]] = []
        iters: Dict[str, Iterator] = {}
        for machine_id, events in machine_logs.items():
            iters[machine_id] = iter(events)

        for machine_id, it in iters.items():
            event = next(it, None)
            while event is not None:
                ts = _parse_ts(event.get(ts_field, ""))
                if ts is not None:
                    heapq.heappush(heap, (ts, machine_id, event, it))
                    break
                malformed += 1
                event = next(it, None)

        result = []
        while heap:
            ts, machine_id, event, it = heapq.heappop(heap)
            result.append(dict(event, _machine=machine_id))
            # Advance this machine's iterator
            while True:
                next_event = next(it, None)
                if next_event is None:
                    break  # Machine exhausted
                next_ts = _parse_ts(next_event.get(ts_field, ""))
                if next_ts is not None:
                    heapq.heappush(heap, (next_ts, machine_id, next_event, it))
                    break
                malformed += 1

        self.malformed_count = malformed
        return result


# ---------------------------------------------------------------------------
# APPROACH 3 — Watermark-Based Streaming Merger (optimal for real-time)
# ---------------------------------------------------------------------------

class WatermarkLogMerger:
    """
    Streaming merge with watermark-based out-of-order handling.

    Each machine reports events in real-time (possibly out of order by at most
    skew_seconds relative to true time).  Events are buffered until the global
    watermark advances past their timestamp.

    Design:
      - per-machine latest_ts cursor
      - global watermark = min(latest_ts per machine) - skew_seconds
      - output buffer: sorted list of (ts, machine_id, event)
      - A real production system would use a priority queue for the buffer.
    """

    def __init__(self, machine_ids: List[str], skew_seconds: float = 5.0) -> None:
        self.skew = skew_seconds
        # Initialise each machine's cursor to -inf (no events seen yet)
        self._latest: Dict[str, float] = {m: float("-inf") for m in machine_ids}
        # Per-machine event buffer: sorted by ts within each machine
        self._buffer: List[Tuple[float, str, dict]] = []  # heap
        heapq.heapify(self._buffer)
        self._emitted: List[dict] = []
        self.malformed_count = 0

    @property
    def watermark(self) -> float:
        """Current safe-to-emit boundary."""
        min_cursor = min(self._latest.values())
        if min_cursor == float("-inf"):
            return float("-inf")
        return min_cursor - self.skew

    def ingest(self, machine_id: str, event: dict) -> List[dict]:
        """
        Ingest one event from machine_id.
        Returns a (possibly empty) list of events now safe to emit.
        """
        if machine_id not in self._latest:
            raise ValueError(f"Unknown machine: {machine_id}")
        ts = _parse_ts(event.get("ts", ""))
        if ts is None:
            self.malformed_count += 1
            return []

        # Advance this machine's cursor (machines are locally monotonic)
        if ts > self._latest[machine_id]:
            self._latest[machine_id] = ts

        heapq.heappush(self._buffer, (ts, machine_id, event))
        return self._flush()

    def _flush(self) -> List[dict]:
        """Emit all buffered events with ts ≤ current watermark."""
        w = self.watermark
        emitted = []
        while self._buffer and self._buffer[0][0] <= w:
            ts, machine_id, event = heapq.heappop(self._buffer)
            emitted.append(dict(event, _machine=machine_id))
        self._emitted.extend(emitted)
        return emitted

    def flush_all(self) -> List[dict]:
        """Force-flush remaining buffer at end of stream."""
        emitted = []
        while self._buffer:
            ts, machine_id, event = heapq.heappop(self._buffer)
            emitted.append(dict(event, _machine=machine_id))
        self._emitted.extend(emitted)
        return emitted

def _make_log(machine: str, ts_event_pairs: List[Tuple[str, str]]) -> List[dict]:
    return [{"ts": ts, "machine": machine, "msg": msg} for ts, msg in ts_event_pairs]


def test_naive_merge():
    merger = NaiveLogMerger()
    merger.add_machine_log("m1", _make_log("m1", [
        ("2026-05-29T08:00:01Z", "login"),
        ("2026-05-29T08:00:05Z", "file_open"),
    ]))
    merger.add_machine_log("m2", _make_log("m2", [
        ("2026-05-29T08:00:00Z", "ssh_connect"),
        ("2026-05-29T08:00:03Z", "command_exec"),
    ]))
    # Malformed
    merger.add_machine_log("m3", [{"ts": "not-a-date", "msg": "bad"}])

    result = merger.merged()
    assert result[0]["msg"] == "ssh_connect"
    assert result[1]["msg"] == "login"
    assert result[2]["msg"] == "command_exec"
    assert result[3]["msg"] == "file_open"
    assert merger.malformed_count == 1
    print("NaiveLogMerger: passed")


def test_kway_merge():
    merger = KWayLogMerger()
    logs = {
        "m1": _make_log("m1", [
            ("2026-05-29T08:00:01Z", "login"),
            ("2026-05-29T08:00:10Z", "file_open"),
        ]),
        "m2": _make_log("m2", [
            ("2026-05-29T08:00:00Z", "ssh_connect"),
            ("2026-05-29T08:00:03Z", "command_exec"),
            ("2026-05-29T08:00:15Z", "exfil"),
        ]),
        "m3": [],  # empty machine
    }
    result = merger.merge(logs)
    msgs = [e["msg"] for e in result]
    assert msgs == ["ssh_connect", "login", "command_exec", "file_open", "exfil"], msgs
    print("KWayLogMerger: passed")


def test_watermark_merge():
    merger = WatermarkLogMerger(["m1", "m2"], skew_seconds=2.0)

    # m1 at T=3, m2 at T=1 → watermark = min(3,1) - 2 = -1 → nothing flushed yet
    out = merger.ingest("m1", {"ts": "2026-05-29T08:00:03Z", "msg": "m1_event"})
    assert out == []

    out = merger.ingest("m2", {"ts": "2026-05-29T08:00:01Z", "msg": "m2_event"})
    assert out == []  # watermark = min(3,1)-2 = -1 still

    # Advance m2 to T=10 → watermark = min(3,10)-2 = 1; flush events at ts ≤ 1
    out = merger.ingest("m2", {"ts": "2026-05-29T08:00:10Z", "msg": "m2_late"})
    # m2_event (ts=1) should be flushed; m1_event (ts=3) not yet
    assert any(e["msg"] == "m2_event" for e in out), f"Expected m2_event in {out}"

    # Advance m1 to T=20 → watermark = min(20,10)-2 = 8; flush up to ts=8
    out = merger.ingest("m1", {"ts": "2026-05-29T08:00:20Z", "msg": "m1_later"})
    assert any(e["msg"] == "m1_event" for e in out), f"Expected m1_event in {out}"

    # Malformed
    out2 = merger.ingest("m1", {"ts": "bad", "msg": "x"})
    assert merger.malformed_count == 1

    print("WatermarkLogMerger: passed")


def test_empty_inputs():
    merger = KWayLogMerger()
    assert merger.merge({}) == []
    assert merger.merge({"m1": []}) == []
    print("Empty input: passed")

"""
THREAT CONTEXT
  SOC analysts correlate events from dozens of machines to reconstruct attack
  timelines.  Incorrect ordering caused by clock skew can:
  - Hide causality (lateral movement appears before the initial foothold)
  - Create false time gaps (attacker appears idle when they weren't)
  - Break sequence-based detection rules (e.g. "process spawn within 5s of login")

CLOCK SKEW SOURCES
  - NTP drift: typically ±0.5s on well-configured Linux; up to ±30s on neglected hosts
  - Offline hosts: clock may be minutes or hours off after reconnection
  - Containerised workloads: container clock inherits host; orchestrator introduces skew
  - Attacker manipulation: an adversary with host access can set the clock backward
    to hide events from time-based detection rules ← critical threat

TAMPER DETECTION
  - Cross-check machine timestamps against authoritative log-forwarding receipt timestamps
  - Alert if |machine_ts - receipt_ts| > threshold (e.g. 60 seconds)
  - Store receipt timestamps in immutable logging backend (CloudTrail, Splunk HEC)

PRODUCTION HARDENING
  - Use Apache Flink or Apache Beam with built-in watermark APIs for production-scale
    streaming; our WatermarkLogMerger is the conceptual equivalent.
  - Retain out-of-order events beyond the watermark for post-hoc forensic re-analysis.
  - Set skew_seconds based on empirical NTP monitoring of your fleet.

COMPLEXITY SUMMARY
  | Approach         | Time       | Space    | Skew-Aware |
  |------------------|------------|----------|------------|
  | Naive sort       | O(N log N) | O(N)     | No         |
  | K-way heap merge | O(N log K) | O(K)     | Partial    |
  | Watermark stream | O(log K)   | O(rate×S)| Yes        |
"""
