"""
=============================================================================
Q08 [Hard] — Rule Engine for Security Events
=============================================================================

PROBLEM
-------
Implement a simple SIEM-style detection rule engine that can evaluate rules
with the following condition types against a stream of security events:

  - equals      : field == value
  - contains    : field contains substring (case-insensitive)
  - regex       : field matches compiled regex pattern
  - gt/lt       : field > value  /  field < value  (numeric)
  - count       : count of matching events per group key ≥ threshold in window
  - time_window : event timestamp within a time window

Rules can combine conditions with AND / OR logic.
Rules have a name, severity, and a list of conditions.

CLARIFYING QUESTIONS
--------------------
1. Rule format — YAML, JSON, Python dict, or DSL string?
   → Python dicts here; note YAML/JSON for production.
2. Should rules be hot-reloadable?
   → Show a reload() method but don't implement dynamic parsing.
3. Can a single event match multiple rules?
   → Yes; return all matching rule names.
4. Count rules — what is the group key?
   → Configurable per rule: e.g. group_by=["src", "username"].
5. Are regex patterns compiled at load time or per event?
   → Compiled at load time (never compile per event — ReDoS risk).
6. Should we validate rules at load time for safety?
   → Yes; reject rules with unsafe regex patterns.
7. How to handle missing fields?
   → Missing fields evaluate the condition as False (do not throw).
8. Time window for count rules — rolling or tumbling?
   → Rolling window with deque-based eviction.

=============================================================================
APPROACH 1 — NAIVE (linear scan of all rules per event)
=============================================================================
For each incoming event, iterate all rules, evaluate all conditions.
  Time : O(R × C) per event  (R rules, C conditions per rule)
  Space: O(R × W × E) for count windows (W window size in events, E events)

For small rule sets this is perfectly fine and is the standard approach in
many SIEM products (Splunk, Sigma).

=============================================================================
APPROACH 2 — INDEX-ACCELERATED RULE DISPATCH  (optimal for large rule sets)
=============================================================================
Build an inverted index: field_value → list of rules that test that field.
On ingestion, look up the field's value in the index to find candidate rules,
then evaluate only those rules instead of all rules.

For a rule set of 10,000 rules and events with 10 fields, this reduces the
average rules evaluated per event from 10,000 to ~50–200.

  Time : O(F × I + M × C) per event
    F = fields in event, I = index lookup O(1), M = matched rule count, C = avg conditions
  Space: O(R × C) for the index

=============================================================================
"""

from __future__ import annotations

import re
import time
from collections import defaultdict, deque
from dataclasses import dataclass, field
from datetime import datetime
from typing import Any, Callable, Dict, List, Optional, Set, Tuple

# ---------------------------------------------------------------------------
# Safe regex compilation helper
# ---------------------------------------------------------------------------

# Simple blocklist of patterns known to cause catastrophic backtracking
_REDOS_PATTERNS = [
    re.compile(r'\(.*\+\)\+'),       # (a+)+
    re.compile(r'\(.*\*\)\*'),       # (a*)*
    re.compile(r'\(\?:.*\|\)\+'),    # (?:a|b)+
]

def _safe_compile(pattern: str) -> re.Pattern:
    """Compile a regex, rejecting patterns that look like ReDoS traps."""
    for bad in _REDOS_PATTERNS:
        if bad.search(pattern):
            raise ValueError(f"Potentially unsafe regex (ReDoS risk): {pattern!r}")
    # Use a timeout via re.compile without full execution — just compilation safety
    try:
        return re.compile(pattern, re.IGNORECASE)
    except re.error as exc:
        raise ValueError(f"Invalid regex {pattern!r}: {exc}") from exc


# ---------------------------------------------------------------------------
# Condition Classes
# ---------------------------------------------------------------------------

@dataclass
class Condition:
    field: str
    op: str          # equals | contains | regex | gt | lt | exists
    value: Any = None
    _compiled: Optional[re.Pattern] = field(default=None, repr=False, compare=False)

    def __post_init__(self) -> None:
        if self.op == "regex" and self.value:
            self._compiled = _safe_compile(str(self.value))

    def evaluate(self, event: dict) -> bool:
        # Safely retrieve nested field via dot notation: "a.b.c"
        val = event
        for part in self.field.split("."):
            if not isinstance(val, dict):
                return False
            val = val.get(part)
            if val is None:
                return self.op == "not_exists"
        if self.op == "exists":
            return val is not None
        if self.op == "not_exists":
            return val is None
        if val is None:
            return False

        str_val = str(val).lower()
        cmp_val = str(self.value).lower() if self.value is not None else ""

        if self.op == "equals":
            return str_val == cmp_val
        if self.op == "not_equals":
            return str_val != cmp_val
        if self.op == "contains":
            return cmp_val in str_val
        if self.op == "not_contains":
            return cmp_val not in str_val
        if self.op == "startswith":
            return str_val.startswith(cmp_val)
        if self.op == "regex":
            return bool(self._compiled and self._compiled.search(str(val)))
        if self.op in ("gt", ">"):
            try:
                return float(val) > float(self.value)
            except (ValueError, TypeError):
                return False
        if self.op in ("lt", "<"):
            try:
                return float(val) < float(self.value)
            except (ValueError, TypeError):
                return False
        if self.op in ("gte", ">="):
            try:
                return float(val) >= float(self.value)
            except (ValueError, TypeError):
                return False
        if self.op in ("lte", "<="):
            try:
                return float(val) <= float(self.value)
            except (ValueError, TypeError):
                return False
        return False


# ---------------------------------------------------------------------------
# Rule Definition
# ---------------------------------------------------------------------------

@dataclass
class Rule:
    name: str
    severity: str             # critical | high | medium | low | info
    conditions: List[Condition]
    logic: str = "AND"        # AND | OR
    # Count-based fields (optional — only for count rules)
    count_threshold: Optional[int] = None
    group_by: List[str] = field(default_factory=list)
    window_seconds: Optional[float] = None
    ts_field: str = "ts"
    # Internal state for count rules
    _windows: Dict[Tuple, deque] = field(
        default_factory=lambda: defaultdict(deque), repr=False)

    def matches_conditions(self, event: dict) -> bool:
        if not self.conditions:
            return False
        if self.logic == "AND":
            return all(c.evaluate(event) for c in self.conditions)
        else:  # OR
            return any(c.evaluate(event) for c in self.conditions)

    def evaluate(self, event: dict) -> bool:
        """
        Evaluate this rule against a single event.
        For count rules, also updates the rolling window counter.
        """
        if not self.matches_conditions(event):
            return False

        # Simple rule (no count threshold) — match immediately
        if self.count_threshold is None:
            return True

        # Count rule — increment counter and check threshold
        return self._check_count(event)

    def _check_count(self, event: dict) -> bool:
        ts_raw = event.get(self.ts_field, time.time())
        ts = float(ts_raw) if isinstance(ts_raw, (int, float)) else time.time()
        try:
            ts = datetime.fromisoformat(str(ts_raw).replace("Z", "+00:00")).timestamp()
        except (ValueError, TypeError):
            pass

        # Group key
        key = tuple(str(event.get(f, "")) for f in self.group_by)
        window = self._windows[key]

        # Evict old events
        if self.window_seconds:
            cutoff = ts - self.window_seconds
            while window and window[0] < cutoff:
                window.popleft()

        window.append(ts)
        return len(window) >= self.count_threshold

    @classmethod
    def from_dict(cls, d: dict) -> "Rule":
        """Build a Rule from a plain dict (YAML/JSON compatible)."""
        conditions = []
        for c in d.get("conditions", []):
            conditions.append(Condition(
                field=c["field"],
                op=c["op"],
                value=c.get("value"),
            ))
        return cls(
            name=d["name"],
            severity=d.get("severity", "medium"),
            conditions=conditions,
            logic=d.get("logic", "AND"),
            count_threshold=d.get("count_threshold"),
            group_by=d.get("group_by", []),
            window_seconds=d.get("window_seconds"),
            ts_field=d.get("ts_field", "ts"),
        )


# ---------------------------------------------------------------------------
# Rule Engine
# ---------------------------------------------------------------------------

@dataclass
class RuleMatch:
    rule_name: str
    severity: str
    event: dict
    ts: float


class RuleEngine:
    """
    SIEM-style rule engine.

    Approach 1 (default): Linear scan — O(R × C) per event.
    Approach 2 (use_index=True): Inverted index dispatch — O(F × I + M × C).
    """

    def __init__(self, rules: List[Rule] = None, use_index: bool = False) -> None:
        self._rules: List[Rule] = []
        self._use_index = use_index
        # Inverted index: field_name → set of rule indices
        self._field_index: Dict[str, Set[int]] = defaultdict(set)
        self._matches: List[RuleMatch] = []
        self.malformed = 0
        if rules:
            for rule in rules:
                self.add_rule(rule)

    def add_rule(self, rule: Rule) -> None:
        idx = len(self._rules)
        self._rules.append(rule)
        if self._use_index:
            for cond in rule.conditions:
                self._field_index[cond.field].add(idx)

    def reload_rules(self, rules: List[Rule]) -> None:
        """Hot-reload: replace all rules atomically."""
        old_rules = self._rules
        self._rules = []
        self._field_index = defaultdict(set)
        try:
            for rule in rules:
                self.add_rule(rule)
        except Exception:
            self._rules = old_rules
            raise

    def process(self, event: dict) -> List[RuleMatch]:
        if not isinstance(event, dict):
            self.malformed += 1
            return []

        ts = self._extract_ts(event)
        matches = []

        if self._use_index:
            # Index-accelerated: only test rules that reference fields in this event
            candidate_indices: Set[int] = set()
            for key in event:
                candidate_indices.update(self._field_index.get(key, set()))
            rules_to_test = [self._rules[i] for i in candidate_indices]
        else:
            rules_to_test = self._rules

        for rule in rules_to_test:
            try:
                if rule.evaluate(event):
                    m = RuleMatch(rule_name=rule.name, severity=rule.severity,
                                  event=event, ts=ts)
                    matches.append(m)
                    self._matches.append(m)
            except Exception:
                # Individual rule failure must not crash the engine
                pass

        return matches

    def _extract_ts(self, event: dict) -> float:
        raw = event.get("ts", time.time())
        if isinstance(raw, (int, float)):
            return float(raw)
        try:
            return datetime.fromisoformat(str(raw).replace("Z", "+00:00")).timestamp()
        except (ValueError, TypeError):
            return time.time()

    def get_all_matches(self) -> List[RuleMatch]:
        return self._matches

# Test Cases

def test_simple_rule():
    engine = RuleEngine(rules=[SAMPLE_RULES[1]])  # encoded_powershell
    event = {"process": "powershell.exe", "cmdline": "powershell -enc SQBFAFgA", "ts": 0}
    matches = engine.process(event)
    assert len(matches) == 1
    assert matches[0].rule_name == "encoded_powershell"
    print("Simple rule: passed")


def test_no_match():
    engine = RuleEngine(rules=[SAMPLE_RULES[1]])
    event = {"process": "notepad.exe", "cmdline": "open file.txt", "ts": 0}
    matches = engine.process(event)
    assert len(matches) == 0
    print("No match: passed")


def test_count_rule():
    engine = RuleEngine(rules=[SAMPLE_RULES[0]])  # brute_force_ssh
    base_ts = 1_000_000.0
    results = []
    for i in range(6):
        e = {"type": "auth", "result": "failure", "port": "22",
             "src": "10.0.0.1", "ts": base_ts + i * 5}
        results += engine.process(e)
    # Alert should fire at the 5th event
    assert any(m.rule_name == "brute_force_ssh" for m in results), \
        f"Expected brute_force_ssh; got {results}"
    print("Count rule: passed")


def test_count_rule_window_expiry():
    """Events spread across more than 60s should NOT trigger."""
    engine = RuleEngine(rules=[SAMPLE_RULES[0]])
    base_ts = 1_000_000.0
    results = []
    for i in range(5):
        e = {"type": "auth", "result": "failure", "port": "22",
             "src": "10.0.0.1", "ts": base_ts + i * 30}  # 30s apart → span 120s
        results += engine.process(e)
    assert not any(m.rule_name == "brute_force_ssh" for m in results), \
        "Should not alert for events spread > window"
    print("Count rule window expiry: passed")


def test_regex_rule():
    engine = RuleEngine(rules=[SAMPLE_RULES[4]])
    event = {"user_agent": "Mozilla/5.0 sqlmap/1.7", "ts": 0}
    matches = engine.process(event)
    assert matches[0].rule_name == "suspicious_useragent"
    print("Regex rule: passed")


def test_missing_field():
    engine = RuleEngine(rules=[SAMPLE_RULES[2]])  # large_outbound
    event = {"dst_port": "443", "ts": 0}  # missing bytes_out
    matches = engine.process(event)
    assert len(matches) == 0
    print("Missing field: passed")


def test_index_accelerated():
    engine = RuleEngine(rules=SAMPLE_RULES, use_index=True)
    event = {"process": "powershell.exe", "cmdline": "powershell -enc abc", "ts": 0}
    matches = engine.process(event)
    assert any(m.rule_name == "encoded_powershell" for m in matches)
    print("Index-accelerated engine: passed")


def test_unsafe_regex():
    try:
        Condition(field="cmd", op="regex", value=r"(a+)+")
        print("WARN: unsafe regex should have been rejected")
    except ValueError:
        print("Unsafe regex rejected: passed")


def test_multiple_rules_match():
    engine = RuleEngine(rules=SAMPLE_RULES)
    event = {"process": "powershell.exe", "cmdline": "powershell -enc x",
             "bytes_out": 5_000_000, "dst_port": "443", "ts": 0}
    matches = engine.process(event)
    rule_names = {m.rule_name for m in matches}
    assert "encoded_powershell" in rule_names
    assert "large_outbound" in rule_names
    print(f"Multiple rules: {rule_names}")

"""
THREAT MODEL
  Rule engines are the detection core of every SIEM.  Weaknesses include:
  - Rule bypass: attacker obfuscates known-bad strings (base64, Unicode escaping)
    → combine regex rules with decoding/normalization before matching.
  - ReDoS (regex denial of service): a single malicious log line with a
    crafted input and an unsafe regex can lock a CPU core indefinitely.
    → Always compile at load time; reject exponentially backtracking patterns;
      use re2 / hyperscan for production.
  - Rule explosion: too many rules → high CPU per event.
    → Use the index-accelerated dispatch (Approach 2) for large rule sets.
  - Silent rule failure: a buggy rule throws an exception and is silently skipped.
    → Log and alert on rule evaluation errors; never swallow them silently in prod.

PRODUCTION CONSIDERATIONS
  - Use Sigma (open standard) for portable rule authoring; translate to engine DSL.
  - Store rule set in version control; CI/CD pipeline validates rules before deploy.
  - Measure rule performance: log evaluation latency per rule; prune slow rules.
  - Count rules require state; shard state by group_by key for distributed engines.
  - For threat hunting: allow retroactive rule application over historical logs.
"""
