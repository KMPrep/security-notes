"""
=============================================================================
Q09 [Hard] — Correlate Alerts into Incidents
=============================================================================

PROBLEM
-------
Raw alerts from a SIEM fire continuously.  Many alerts are related — they
share the same user, host, IP, or happen close in time.  Group related alerts
into *incidents* to reduce analyst noise and surface the full picture.

Correlation dimensions:
  1. Shared entity: same user, same host, or same source IP.
  2. Temporal proximity: alerts within a configurable window of each other.

An incident is a cluster of alerts linked by at least one of those dimensions.
If alert A shares a user with B, and B shares a host with C, then A, B, C
belong to the same incident (transitive closure).

CLARIFYING QUESTIONS
--------------------
1. Input: stream of alert dicts or batch list?
   → Support both; streaming incremental update is the hard case.
2. What fields define "same entity"?
   → user, host, src_ip — configurable.
3. What does "temporal proximity" mean exactly?
   → Two alerts are proximate if |ts_a - ts_b| ≤ time_window_seconds.
4. What is the incident severity?
   → Max severity across member alerts.
5. Should incidents be updated when new alerts arrive?
   → Yes (incremental merge).
6. When should an incident be closed?
   → After no new alerts for close_after_seconds (configurable, default 3600s).
7. Memory bound?
   → Evict closed incidents; keep only open ones in memory.

=============================================================================
APPROACH 1 — NAIVE (pairwise comparison + Union-Find offline)
=============================================================================
For each pair of alerts, check entity overlap and time proximity.
Build a graph; connected components = incidents.
  Time : O(N²) per batch — too slow for streaming
  Space: O(N²) edges

=============================================================================
APPROACH 2 — ENTITY INDEX + UNION-FIND (RECOMMENDED)
=============================================================================
Maintain an inverted index: entity_value → incident_id.
When a new alert arrives:
  1. Find all incidents sharing any entity (user, host, src_ip) with the alert.
  2. Also find incidents within the time window (checked via last-seen timestamp).
  3. Union all matched incidents + the new alert into one incident.

Data structures:
  - entity_index: Dict[entity_value → incident_id]  — O(1) lookup
  - incidents: Dict[incident_id → Incident]
  - Union-Find (path compression + rank) for O(α(N)) merge ≈ O(1)

  Time : O(E × α(N)) per alert   (E = entity fields per alert, ≈ 3)
  Space: O(A)  active alerts + O(I) active incidents

=============================================================================
APPROACH 3 — GRAPH CLUSTERING with NetworkX-style adjacency
=============================================================================
Build an adjacency list per entity; at query time run BFS/DFS for components.
Easier to reason about but O(V+E) per query — suitable for offline analytics.

=============================================================================
"""

from __future__ import annotations

import uuid
from collections import defaultdict, deque
from dataclasses import dataclass, field
from datetime import datetime
from typing import Dict, List, Optional, Set, Tuple

# ---------------------------------------------------------------------------
# Severity ordering
# ---------------------------------------------------------------------------
SEVERITY_ORDER = {"critical": 4, "high": 3, "medium": 2, "low": 1, "info": 0}

def _max_severity(severities: List[str]) -> str:
    return max(severities, key=lambda s: SEVERITY_ORDER.get(s.lower(), 0))


# ---------------------------------------------------------------------------
# Data Types
# ---------------------------------------------------------------------------

@dataclass
class Alert:
    alert_id: str
    ts: float
    rule_name: str
    severity: str
    user: Optional[str] = None
    host: Optional[str] = None
    src_ip: Optional[str] = None
    raw: dict = field(default_factory=dict)


@dataclass
class Incident:
    incident_id: str
    alert_ids: List[str] = field(default_factory=list)
    severity: str = "info"
    first_seen: float = 0.0
    last_seen: float = 0.0
    users: Set[str] = field(default_factory=set)
    hosts: Set[str] = field(default_factory=set)
    src_ips: Set[str] = field(default_factory=set)
    rule_names: Set[str] = field(default_factory=set)
    status: str = "open"  # open | closed

    def add_alert(self, alert: Alert) -> None:
        self.alert_ids.append(alert.alert_id)
        self.rule_names.add(alert.rule_name)
        sev = _max_severity([self.severity, alert.severity])
        self.severity = sev
        if self.first_seen == 0.0 or alert.ts < self.first_seen:
            self.first_seen = alert.ts
        if alert.ts > self.last_seen:
            self.last_seen = alert.ts
        if alert.user:
            self.users.add(alert.user)
        if alert.host:
            self.hosts.add(alert.host)
        if alert.src_ip:
            self.src_ips.add(alert.src_ip)

    def merge_from(self, other: "Incident") -> None:
        """Merge another incident's data into this one."""
        for aid in other.alert_ids:
            if aid not in self.alert_ids:
                self.alert_ids.append(aid)
        self.rule_names.update(other.rule_names)
        self.severity = _max_severity([self.severity, other.severity])
        self.first_seen = min(self.first_seen, other.first_seen) if self.first_seen else other.first_seen
        self.last_seen = max(self.last_seen, other.last_seen)
        self.users.update(other.users)
        self.hosts.update(other.hosts)
        self.src_ips.update(other.src_ips)


# ---------------------------------------------------------------------------
# Helpers
# ---------------------------------------------------------------------------

def _parse_ts(ts) -> Optional[float]:
    if isinstance(ts, (int, float)):
        return float(ts)
    try:
        return datetime.fromisoformat(str(ts).replace("Z", "+00:00")).timestamp()
    except (ValueError, TypeError):
        return None


def _alert_from_dict(d: dict) -> Optional[Alert]:
    ts = _parse_ts(d.get("ts"))
    if ts is None:
        return None
    return Alert(
        alert_id=d.get("alert_id") or str(uuid.uuid4()),
        ts=ts,
        rule_name=d.get("rule_name", "unknown"),
        severity=d.get("severity", "info"),
        user=d.get("user"),
        host=d.get("host"),
        src_ip=d.get("src_ip") or d.get("src"),
        raw=d,
    )


# ---------------------------------------------------------------------------
# APPROACH 1 — Naive offline batch (pairwise + BFS connected components)
# ---------------------------------------------------------------------------

class NaiveIncidentCorrelator:
    """
    Offline batch correlator.  Stores all alerts then builds a graph of
    related alerts, finds connected components via BFS.
    O(N²) time — fine for forensic analysis on small datasets.
    """

    def __init__(
        self,
        time_window_seconds: float = 3600.0,
        entity_fields: Tuple[str, ...] = ("user", "host", "src_ip"),
    ) -> None:
        self.window = time_window_seconds
        self.entity_fields = entity_fields
        self._alerts: List[Alert] = []
        self.malformed = 0

    def add(self, raw: dict) -> None:
        alert = _alert_from_dict(raw)
        if alert is None:
            self.malformed += 1
            return
        self._alerts.append(alert)

    def _are_related(self, a: Alert, b: Alert) -> bool:
        if abs(a.ts - b.ts) > self.window:
            return False
        for f in self.entity_fields:
            va, vb = getattr(a, f), getattr(b, f)
            if va and vb and va == vb:
                return True
        return False

    def build_incidents(self) -> List[Incident]:
        n = len(self._alerts)
        parent = list(range(n))

        def find(x):
            while parent[x] != x:
                parent[x] = parent[parent[x]]
                x = parent[x]
            return x

        def union(x, y):
            px, py = find(x), find(y)
            if px != py:
                parent[px] = py

        for i in range(n):
            for j in range(i + 1, n):
                if self._are_related(self._alerts[i], self._alerts[j]):
                    union(i, j)

        groups: Dict[int, List[int]] = defaultdict(list)
        for i in range(n):
            groups[find(i)].append(i)

        incidents = []
        for indices in groups.values():
            inc = Incident(incident_id=str(uuid.uuid4()))
            for idx in indices:
                inc.add_alert(self._alerts[idx])
            incidents.append(inc)
        return incidents


# ---------------------------------------------------------------------------
# APPROACH 2 — Streaming Incremental Correlator (RECOMMENDED)
# ---------------------------------------------------------------------------

class StreamingIncidentCorrelator:
    """
    Incremental streaming incident correlator using entity index + Union-Find.

    On each new alert:
    1. Look up entity index to find existing incidents sharing user/host/src_ip.
    2. Find time-proximate incidents (those with last_seen within window).
    3. Union all found incidents + new alert.
    4. Update entity index.
    5. Periodically close idle incidents.

    Memory: O(open_incidents × average_alerts_per_incident).
    """

    def __init__(
        self,
        time_window_seconds: float = 3600.0,
        close_after_seconds: float = 3600.0,
        entity_fields: Tuple[str, ...] = ("user", "host", "src_ip"),
    ) -> None:
        self.window = time_window_seconds
        self.close_after = close_after_seconds
        self.entity_fields = entity_fields
        # entity_value → incident_id
        self._entity_index: Dict[str, str] = {}
        # incident_id → Incident
        self._incidents: Dict[str, Incident] = {}
        # alert_id → incident_id (for quick membership lookup)
        self._alert_incident: Dict[str, str] = {}
        self._closed: List[Incident] = []
        self.malformed = 0

    def process(self, raw: dict) -> Tuple[Optional[Incident], bool]:
        """
        Process one alert.
        Returns (incident, is_new) where is_new=True if a new incident was created.
        """
        alert = _alert_from_dict(raw)
        if alert is None:
            self.malformed += 1
            return None, False

        # 1. Find all existing incident IDs related to this alert
        matched_iids: Set[str] = set()

        for f in self.entity_fields:
            val = getattr(alert, f)
            if val and val in self._entity_index:
                matched_iids.add(self._entity_index[val])

        # 2. Filter: only keep incidents within the time window
        valid_iids = {
            iid for iid in matched_iids
            if iid in self._incidents
            and abs(self._incidents[iid].last_seen - alert.ts) <= self.window
        }

        is_new = len(valid_iids) == 0

        if is_new:
            # Create new incident
            inc = Incident(incident_id=str(uuid.uuid4()))
            inc.add_alert(alert)
            self._incidents[inc.incident_id] = inc
            target_iid = inc.incident_id
        else:
            # Merge all matching incidents into the first one
            iids = sorted(valid_iids)
            primary_iid = iids[0]
            primary = self._incidents[primary_iid]

            for other_iid in iids[1:]:
                other = self._incidents.pop(other_iid, None)
                if other:
                    primary.merge_from(other)
                    # Redirect all entity index entries pointing to merged incident
                    for f in self.entity_fields:
                        for val in list(self._entity_index):
                            if self._entity_index[val] == other_iid:
                                self._entity_index[val] = primary_iid
                    # Redirect alert ownership
                    for aid in other.alert_ids:
                        self._alert_incident[aid] = primary_iid

            primary.add_alert(alert)
            target_iid = primary_iid

        # 3. Update entity index
        for f in self.entity_fields:
            val = getattr(alert, f)
            if val:
                self._entity_index[val] = target_iid
        self._alert_incident[alert.alert_id] = target_iid

        return self._incidents[target_iid], is_new

    def close_idle_incidents(self, now: float) -> List[Incident]:
        """Close incidents with no activity for close_after_seconds."""
        to_close = [
            iid for iid, inc in self._incidents.items()
            if now - inc.last_seen > self.close_after
        ]
        closed = []
        for iid in to_close:
            inc = self._incidents.pop(iid)
            inc.status = "closed"
            self._closed.append(inc)
            closed.append(inc)
            # Clean up entity index
            for val, vid in list(self._entity_index.items()):
                if vid == iid:
                    del self._entity_index[val]
        return closed

    def open_incidents(self) -> List[Incident]:
        return list(self._incidents.values())

    def get_incident_for_alert(self, alert_id: str) -> Optional[Incident]:
        iid = self._alert_incident.get(alert_id)
        return self._incidents.get(iid) if iid else None

def _raw(rule: str, sev: str, ts: float, user=None, host=None, src_ip=None) -> dict:
    d = {"rule_name": rule, "severity": sev, "ts": ts}
    if user:
        d["user"] = user
    if host:
        d["host"] = host
    if src_ip:
        d["src_ip"] = src_ip
    return d


def test_naive_same_user():
    corr = NaiveIncidentCorrelator(time_window_seconds=3600)
    corr.add(_raw("brute_force", "high", 1000, user="alice"))
    corr.add(_raw("lateral_movement", "critical", 1100, user="alice"))
    corr.add(_raw("unrelated", "low", 5000, user="bob"))
    incidents = corr.build_incidents()
    # alice's two alerts should merge; bob's should be separate
    assert len(incidents) == 2, f"Expected 2 incidents, got {len(incidents)}"
    alice_inc = next(i for i in incidents if "alice" in i.users)
    assert len(alice_inc.alert_ids) == 2
    assert alice_inc.severity == "critical"
    print("Naive same-user: passed")


def test_naive_transitive():
    """A→B via user, B→C via host → A,B,C in one incident."""
    corr = NaiveIncidentCorrelator(time_window_seconds=3600)
    corr.add(_raw("a", "low",  1000, user="alice", host=None))
    corr.add(_raw("b", "high", 1050, user="alice", host="pc-01"))
    corr.add(_raw("c", "med",  1100, user=None,    host="pc-01"))
    incidents = corr.build_incidents()
    assert len(incidents) == 1, f"Expected 1 incident, got {len(incidents)}"
    assert len(incidents[0].alert_ids) == 3
    print("Naive transitive: passed")


def test_streaming_same_user():
    corr = StreamingIncidentCorrelator(time_window_seconds=3600)
    inc1, is_new1 = corr.process(_raw("r1", "high", 1000, user="alice"))
    assert is_new1
    inc2, is_new2 = corr.process(_raw("r2", "critical", 1100, user="alice"))
    assert not is_new2
    assert inc1.incident_id == inc2.incident_id
    assert inc2.severity == "critical"
    assert len(inc2.alert_ids) == 2
    print("Streaming same-user: passed")


def test_streaming_merge_incidents():
    """Two separate incidents (different entities) later get merged via a bridging alert."""
    corr = StreamingIncidentCorrelator(time_window_seconds=3600)
    # Incident A: src_ip=10.0.0.1
    corr.process(_raw("r1", "high", 1000, src_ip="10.0.0.1"))
    # Incident B: host=server-01
    corr.process(_raw("r2", "medium", 1050, host="server-01"))
    assert len(corr.open_incidents()) == 2

    # Bridge alert: same src_ip AND same host → merges both incidents
    _, is_new = corr.process(_raw("r3", "critical", 1100,
                                    src_ip="10.0.0.1", host="server-01"))
    assert not is_new
    assert len(corr.open_incidents()) == 1
    merged = corr.open_incidents()[0]
    assert merged.severity == "critical"
    assert len(merged.alert_ids) == 3
    print("Streaming merge: passed")


def test_time_window_separation():
    """Alerts outside the window should NOT be correlated."""
    corr = StreamingIncidentCorrelator(time_window_seconds=60)
    corr.process(_raw("r1", "high", 0.0, user="bob"))
    _, is_new = corr.process(_raw("r2", "high", 5000.0, user="bob"))  # 5000s later
    assert is_new, "Alert outside window should start a new incident"
    assert len(corr.open_incidents()) == 2
    print("Time window separation: passed")


def test_close_idle():
    corr = StreamingIncidentCorrelator(close_after_seconds=100)
    corr.process(_raw("r1", "low", 0.0, user="carol"))
    closed = corr.close_idle_incidents(now=200.0)
    assert len(closed) == 1
    assert closed[0].status == "closed"
    assert len(corr.open_incidents()) == 0
    print("Close idle: passed")


def test_malformed():
    corr = StreamingIncidentCorrelator()
    inc, _ = corr.process({"rule_name": "x"})  # missing ts
    assert inc is None
    assert corr.malformed == 1
    print("Malformed: passed")

"""
THREAT MODEL
  Without incident correlation, a SIEM analyst faces hundreds of raw alerts
  per hour.  An attacker executing a multi-stage campaign generates many
  individual alerts that individually look unremarkable (medium/low severity)
  but together reveal the full attack chain.

  Incident grouping reduces alert fatigue and enables:
  - Severity escalation: two medium alerts = one high incident.
  - Unified timeline: all evidence in one place.
  - Faster containment: responder sees all entities at once (user+host+IP).

EXAMPLE INCIDENT
  Alert 1: "brute_force_ssh" — src_ip=1.2.3.4 → medium
  Alert 2: "new_ssh_login"   — src_ip=1.2.3.4, host=db-01 → medium
  Alert 3: "db_dump_command" — host=db-01 → high
  → Same incident (1.2.3.4 then db-01); severity = high; analysts alerted once.

FAILURE MODES
  - Entity aliasing: attacker uses different usernames but same real identity
    (john@corp vs john.doe@corp).  Normalise identities before indexing.
  - Time zone errors: incorrect timestamp parsing makes proximity check fail.
  - Incident explosion: if entity_index is too broad (e.g. correlate by ASN
    instead of src_ip), unrelated alerts merge.  Calibrate entity granularity.
  - Cascading merges: one bridge alert merges thousands of small incidents into
    a mega-incident.  Limit max_alerts_per_incident (e.g. 1000).

PRODUCTION HARDENING
  - Store incidents in a DB (Elasticsearch, Postgres) for durability.
  - Expose REST API: GET /incidents, POST /alerts.
  - Feed incident enrichment (threat intel, user risk score) at merge time.
  - Implement incident lifecycle: open → triaging → in_response → closed.
"""
