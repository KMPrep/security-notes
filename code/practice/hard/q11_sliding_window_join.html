"""
=============================================================================
Q11 [Hard] — Sliding Window Join of Two Security Event Streams
=============================================================================

PROBLEM
-------
Given two streams of security events (e.g. firewall deny logs and IDS alerts),
join events from the two streams that share a common key (e.g. src_ip) and
occur within a time window of each other.

This is a core operation in SIEM correlation: "did a firewall block from IP X
occur within 5 minutes of an IDS alert from IP X?"

CLARIFYING QUESTIONS
--------------------
1. Are events in each stream ordered by timestamp?
   → Yes, each stream is monotonically increasing; cross-stream ordering not guaranteed.
2. What is the join key?
   → Configurable; default is src_ip.  Composite keys supported (tuple of fields).
3. What is the join predicate?
   → |ts_a - ts_b| <= window_seconds  AND  same join key value.
4. Should we emit results for each match, or a grouped result per key?
   → Emit each matched pair.
5. Can one event from stream A match multiple events from stream B?
   → Yes — emit all pairs.
6. Memory bound?
   → Evict events older than window_seconds from each side.
7. What about late-arriving events (out-of-order)?
   → Support a watermark slack; events arriving more than slack seconds late
     are dropped (with a counter).
8. Output format?
   → List of (event_a, event_b, join_key, ts_a, ts_b) tuples.

=============================================================================
APPROACH 1 — NAIVE (nested-loop join, offline batch)
=============================================================================
For each event in stream A, scan all events in stream B for matches.
  Time : O(|A| × |B|) per query  — O(N²)
  Space: O(N) storing both streams
  Fine for offline forensic analysis on small datasets.

=============================================================================
APPROACH 2 — SLIDING WINDOW JOIN with per-key deque buffers (RECOMMENDED)
=============================================================================
Maintain two buffers keyed by the join attribute:
  buf_a: Dict[join_key → Deque[event]]
  buf_b: Dict[join_key → Deque[event]]

When event e from stream A arrives:
  1. Evict stale entries from buf_a[key] (ts < now - window).
  2. Lookup buf_b[key]; emit pairs for events within window.
  3. Append e to buf_a[key].
  (Symmetric for stream B.)

  Time : O(M) per event  (M = avg matches per key per window)
  Space: O(W × R) where W = window_seconds, R = event rate per key

=============================================================================
APPROACH 3 — INTERVAL TREE JOIN (for very large out-of-order streams)
=============================================================================
Index each event as an interval [ts - window, ts + window].
On new event: query interval tree for overlapping events with same key.
  Time : O(log N + M) per event
  Space: O(N) — better balanced for high-event-rate scenarios

=============================================================================
"""

from __future__ import annotations

from collections import defaultdict, deque
from dataclasses import dataclass, field
from datetime import datetime
from typing import Any, Dict, Generator, List, Optional, Set, Tuple

# ---------------------------------------------------------------------------
# Data Types
# ---------------------------------------------------------------------------

@dataclass
class StreamEvent:
    stream: str           # "A" or "B"
    event_id: str
    ts: float
    join_key: Any         # value of the join field (e.g. src_ip)
    data: dict = field(default_factory=dict)


@dataclass
class JoinResult:
    event_a: StreamEvent
    event_b: StreamEvent
    join_key: Any
    delta_seconds: float

    def __repr__(self) -> str:
        return (f"JoinResult(key={self.join_key!r}, "
                f"a_id={self.event_a.event_id}, b_id={self.event_b.event_id}, "
                f"delta={self.delta_seconds:.1f}s)")


# ---------------------------------------------------------------------------
# Helpers
# ---------------------------------------------------------------------------

def _parse_ts(ts) -> Optional[float]:
    if isinstance(ts, (int, float)):
        return float(ts)
    try:
        return datetime.fromisoformat(str(ts).replace("Z", "+00:00")).timestamp()
    except (ValueError, TypeError):
        return None


def _extract_key(event: dict, key_fields: Tuple[str, ...]) -> Optional[tuple]:
    """Extract composite join key from an event dict."""
    parts = []
    for f in key_fields:
        val = event.get(f)
        if val is None:
            return None
        parts.append(str(val))
    return tuple(parts)


# ---------------------------------------------------------------------------
# APPROACH 1 — Naive Offline Batch Join
# ---------------------------------------------------------------------------

class NaiveWindowJoin:
    """
    Offline batch join: store everything, then scan all pairs.
    O(|A| × |B|) — good for small forensic datasets, not streaming.
    """

    def __init__(self, window_seconds: float = 300.0,
                 key_fields: Tuple[str, ...] = ("src_ip",)) -> None:
        self.window = window_seconds
        self.key_fields = key_fields
        self._stream_a: List[StreamEvent] = []
        self._stream_b: List[StreamEvent] = []
        self.malformed = 0
        self._counter = 0

    def _parse_event(self, raw: dict, stream: str) -> Optional[StreamEvent]:
        ts = _parse_ts(raw.get("ts"))
        key = _extract_key(raw, self.key_fields)
        if ts is None or key is None:
            self.malformed += 1
            return None
        self._counter += 1
        return StreamEvent(stream=stream, event_id=raw.get("id", str(self._counter)),
                           ts=ts, join_key=key, data=raw)

    def add_a(self, raw: dict) -> None:
        e = self._parse_event(raw, "A")
        if e:
            self._stream_a.append(e)

    def add_b(self, raw: dict) -> None:
        e = self._parse_event(raw, "B")
        if e:
            self._stream_b.append(e)

    def join(self) -> List[JoinResult]:
        results = []
        for ea in self._stream_a:
            for eb in self._stream_b:
                if ea.join_key == eb.join_key and abs(ea.ts - eb.ts) <= self.window:
                    results.append(JoinResult(ea, eb, ea.join_key,
                                              abs(ea.ts - eb.ts)))
        return results

# APPROACH 2 — Streaming Sliding Window Join (RECOMMENDED)
# ---------------------------------------------------------------------------

class SlidingWindowJoin:
    """
    Streaming sliding window join.  Process events from A or B in arrival order.
    O(M) per event where M is the average number of matching events per key in window.

    late_tolerance_seconds: events arriving more than this amount late (behind
    the high watermark for their stream) are counted as late and dropped.
    """

    def __init__(
        self,
        window_seconds: float = 300.0,
        key_fields: Tuple[str, ...] = ("src_ip",),
        late_tolerance_seconds: float = 30.0,
    ) -> None:
        self.window = window_seconds
        self.key_fields = key_fields
        self.late_tolerance = late_tolerance_seconds
        # buf_a[key] = deque of StreamEvent (sorted by ts)
        self._buf_a: Dict[tuple, deque] = defaultdict(deque)
        self._buf_b: Dict[tuple, deque] = defaultdict(deque)
        # high watermark per stream
        self._hwm_a: float = 0.0
        self._hwm_b: float = 0.0
        self.late_drops_a: int = 0
        self.late_drops_b: int = 0
        self.malformed: int = 0
        self._counter = 0

    def _make_event(self, raw: dict, stream: str) -> Optional[StreamEvent]:
        ts = _parse_ts(raw.get("ts"))
        key = _extract_key(raw, self.key_fields)
        if ts is None or key is None:
            self.malformed += 1
            return None
        self._counter += 1
        return StreamEvent(stream=stream, event_id=raw.get("id", str(self._counter)),
                           ts=ts, join_key=key, data=raw)

    def _evict(self, buf: Dict, key: tuple, cutoff_ts: float) -> None:
        """Remove events older than cutoff_ts from buf[key]."""
        d = buf.get(key)
        if not d:
            return
        while d and d[0].ts < cutoff_ts:
            d.popleft()

    def process_a(self, raw: dict) -> List[JoinResult]:
        return self._process(raw, "A")

    def process_b(self, raw: dict) -> List[JoinResult]:
        return self._process(raw, "B")

    def _process(self, raw: dict, stream: str) -> List[JoinResult]:
        e = self._make_event(raw, stream)
        if e is None:
            return []

        # Late arrival check
        hwm = self._hwm_a if stream == "A" else self._hwm_b
        if e.ts < hwm - self.late_tolerance:
            if stream == "A":
                self.late_drops_a += 1
            else:
                self.late_drops_b += 1
            return []  # drop late event

        # Update high watermark
        if stream == "A":
            self._hwm_a = max(self._hwm_a, e.ts)
        else:
            self._hwm_b = max(self._hwm_b, e.ts)

        key = e.join_key
        own_buf = self._buf_a if stream == "A" else self._buf_b
        other_buf = self._buf_b if stream == "A" else self._buf_a

        # Evict stale entries from own buffer
        cutoff = e.ts - self.window
        self._evict(own_buf, key, cutoff)

        # Find matches in the other stream's buffer
        results = []
        self._evict(other_buf, key, cutoff)
        for other_evt in other_buf.get(key, []):
            if abs(e.ts - other_evt.ts) <= self.window:
                if stream == "A":
                    results.append(JoinResult(e, other_evt, key, abs(e.ts - other_evt.ts)))
                else:
                    results.append(JoinResult(other_evt, e, key, abs(e.ts - other_evt.ts)))

        # Append to own buffer
        own_buf[key].append(e)
        return results

    def flush_stats(self) -> dict:
        return {
            "late_drops_a": self.late_drops_a,
            "late_drops_b": self.late_drops_b,
            "malformed": self.malformed,
            "buf_a_keys": len(self._buf_a),
            "buf_b_keys": len(self._buf_b),
        }

# APPROACH 3 — Simple Interval Index (sorted list + binary search)

import bisect

class IntervalWindowJoin:
    """
    Each event in stream A is stored with its interval [ts-window, ts+window].
    When a stream-B event arrives, binary search for A events within range.

    This gives O(log N + M) per event.  Good for moderate out-of-order streams.
    """

    def __init__(self, window_seconds: float = 300.0,
                 key_fields: Tuple[str, ...] = ("src_ip",)) -> None:
        self.window = window_seconds
        self.key_fields = key_fields
        # sorted by ts for each key: (ts, StreamEvent)
        self._by_key_a: Dict[tuple, List[Tuple[float, StreamEvent]]] = defaultdict(list)
        self.malformed = 0
        self._counter = 0

    def _make_event(self, raw: dict, stream: str) -> Optional[StreamEvent]:
        ts = _parse_ts(raw.get("ts"))
        key = _extract_key(raw, self.key_fields)
        if ts is None or key is None:
            self.malformed += 1
            return None
        self._counter += 1
        return StreamEvent(stream=stream, event_id=raw.get("id", str(self._counter)),
                           ts=ts, join_key=key, data=raw)

    def add_a(self, raw: dict) -> None:
        e = self._make_event(raw, "A")
        if e:
            lst = self._by_key_a[e.join_key]
            bisect.insort(lst, (e.ts, e))

    def query_b(self, raw: dict) -> List[JoinResult]:
        e = self._make_event(raw, "B")
        if e is None:
            return []
        key = e.join_key
        lst = self._by_key_a.get(key, [])
        if not lst:
            return []
        lo = bisect.bisect_left(lst, (e.ts - self.window, ))
        hi = bisect.bisect_right(lst, (e.ts + self.window, ))
        results = []
        for _, ea in lst[lo:hi]:
            results.append(JoinResult(ea, e, key, abs(ea.ts - e.ts)))
        return results

def _ev(stream, sid, ts, src_ip, **kwargs) -> dict:
    return {"id": f"{stream}-{sid}", "ts": ts, "src_ip": src_ip, **kwargs}


def test_naive_join():
    j = NaiveWindowJoin(window_seconds=300)
    j.add_a(_ev("A", 1, 1000, "10.0.0.1"))
    j.add_a(_ev("A", 2, 2000, "10.0.0.2"))
    j.add_b(_ev("B", 1, 1100, "10.0.0.1"))  # matches A-1 (Δ100s)
    j.add_b(_ev("B", 2, 9999, "10.0.0.1"))  # too far from A-1 (Δ8999s)
    results = j.join()
    assert len(results) == 1
    assert results[0].event_a.event_id == "A-1"
    assert results[0].event_b.event_id == "B-1"
    print("Naive join: passed")


def test_naive_no_match():
    j = NaiveWindowJoin(window_seconds=60)
    j.add_a(_ev("A", 1, 0, "10.0.0.1"))
    j.add_b(_ev("B", 1, 1000, "10.0.0.1"))  # 1000s > 60s window
    assert len(j.join()) == 0
    print("Naive no-match: passed")


def test_sliding_basic():
    j = SlidingWindowJoin(window_seconds=300)
    # Seed B buffer first
    j.process_b(_ev("B", 1, 1000, "192.168.1.1"))
    # A arrives; should match B-1
    results = j.process_a(_ev("A", 1, 1200, "192.168.1.1"))
    assert len(results) == 1
    assert results[0].event_a.event_id == "A-1"
    assert results[0].event_b.event_id == "B-1"
    print("Sliding basic: passed")


def test_sliding_multiple_matches():
    j = SlidingWindowJoin(window_seconds=300)
    j.process_b(_ev("B", 1, 1000, "1.2.3.4"))
    j.process_b(_ev("B", 2, 1100, "1.2.3.4"))
    results = j.process_a(_ev("A", 1, 1050, "1.2.3.4"))
    assert len(results) == 2, f"Expected 2 matches, got {len(results)}"
    print("Sliding multiple matches: passed")


def test_sliding_eviction():
    """Events outside the window should be evicted and not match."""
    j = SlidingWindowJoin(window_seconds=100)
    j.process_b(_ev("B", 1, 0, "5.5.5.5"))
    results = j.process_a(_ev("A", 1, 500, "5.5.5.5"))  # 500s > 100s window
    assert len(results) == 0
    print("Sliding eviction: passed")


def test_sliding_different_keys():
    j = SlidingWindowJoin(window_seconds=300)
    j.process_b(_ev("B", 1, 1000, "10.0.0.1"))
    results = j.process_a(_ev("A", 1, 1100, "10.0.0.2"))  # different key
    assert len(results) == 0
    print("Sliding different keys: passed")


def test_late_arrival():
    j = SlidingWindowJoin(window_seconds=300, late_tolerance_seconds=50)
    j.process_a(_ev("A", 1, 1000, "9.9.9.9"))
    j.process_a(_ev("A", 2, 2000, "9.9.9.9"))  # hwm_a = 2000
    results = j.process_a(_ev("A", 3, 900, "9.9.9.9"))  # 900 < 2000-50 → late drop
    assert j.late_drops_a == 1
    print("Late arrival drop: passed")


def test_interval_join():
    j = IntervalWindowJoin(window_seconds=200)
    j.add_a(_ev("A", 1, 1000, "8.8.8.8"))
    j.add_a(_ev("A", 2, 3000, "8.8.8.8"))
    results = j.query_b(_ev("B", 1, 1100, "8.8.8.8"))  # matches A-1 (Δ100)
    assert len(results) == 1
    assert results[0].event_a.event_id == "A-1"
    print("Interval join: passed")


def test_malformed_events():
    j = SlidingWindowJoin()
    j.process_a({"ts": "not-a-date", "src_ip": "1.2.3.4"})
    j.process_a({"ts": 1000})  # missing src_ip
    assert j.malformed == 2
    print("Malformed events: passed")

"""
THREAT MODEL
  Sliding window join is the mechanism behind correlated detection rules like:
    "firewall blocked IP X within 5 minutes of IDS alert for IP X"
    "DNS query for domain Y within 10 minutes of process creation for browser Z"

  This pattern detects:
  - Reconnaissance followed by exploitation
  - Command-and-control beacon followed by data staging
  - Lateral movement (auth event in one stream, file access in another)

COMMON USE CASES
  Stream A: Netflow (firewall allows/denies)
  Stream B: Endpoint alerts (process, registry, file)
  Join key: src_ip or (user, host)
  → Detect: "host was allowed outbound AND spawned suspicious process"

FAILURE MODES
  - Key cardinality explosion: if join key = src_ip and source is a NAT, all
    events share the same IP.  Use (src_ip, dst_port) composite keys.
  - Clock skew: event timestamps from different collectors may differ by
    minutes.  Widen the join window and use a generous late_tolerance.
  - Memory growth: if a key generates events only in stream A (never B),
    buf_a[key] grows without bound.  Set a max_buffer_size_per_key.
  - False positive storms: after a network event (planned reboot), all hosts
    generate events → join explodes.  Rate-limit join output per key.

PRODUCTION HARDENING
  - Use Apache Flink or Kafka Streams for distributed sliding window joins.
  - Partition streams by join key for horizontal scaling.
  - Monitor late_drops counter → increase late_tolerance or fix clock sync.
  - Use exactly-once semantics (Kafka transactions) to avoid duplicate joins.
"""
