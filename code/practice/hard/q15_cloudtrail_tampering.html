"""
=============================================================================
Q15 [Hard] — CloudTrail Tampering Detection
=============================================================================

PROBLEM
-------
AWS CloudTrail is the audit log for all API calls in an AWS account.
Attackers who gain sufficient privileges try to tamper with CloudTrail to
cover their tracks:

  1. StopLogging     — disables the trail entirely
  2. DeleteTrail     — permanently destroys the trail
  3. UpdateTrail     — changes S3 bucket, reduces log scope (e.g. disables
                       data events, removes MFA delete protection)
  4. PutEventSelectors — removes certain event types from logging
  5. DeleteLogGroup  — deletes CloudWatch Logs log group receiving CT events
  6. PutBucketPolicy — overly permissive S3 policy on the CT bucket (public access)
  7. DeleteBucketLifecycle — removes S3 lifecycle that preserves CT logs
  8. DisableKey / ScheduleKeyDeletion — deletes the KMS key encrypting CT logs

Additionally detect:
  - Caller using root account for any CloudTrail action
  - Actions from an unusual region (not in company's approved regions)
  - Burst of CloudTrail API calls from the same principal within a window
    (attacker probing what logging is configured)

CLARIFYING QUESTIONS
--------------------
1. Input format?
   → CloudTrail event dicts (subset of real CT schema).
2. What is the approved region list?
   → Configurable; default {"us-east-1", "us-west-2", "eu-west-1"}.
3. Burst threshold?
   → > burst_threshold CT API calls from same principal in burst_window_seconds.
4. Should we look at only CloudTrail-namespace events or also S3/KMS?
   → Yes — include S3 and KMS events that affect CT log storage.
5. False positive handling for UpdateTrail by legitimate automation?
   → Accept an allowlist of principals (e.g. a CT-admin IAM role).
6. What severity to assign each finding type?
   → StopLogging/DeleteTrail = critical; others = high; probing = medium.

=============================================================================
APPROACH 1 — NAIVE (rule list scan per event)
=============================================================================
Hardcode a list of (eventName → alert) rules.
For each event, check event name against all rules.
  Time : O(R) per event  (R = number of rules, ≈ 20)
  Space: O(1) — stateless

=============================================================================
APPROACH 2 — INDEX + STATEFUL BURST DETECTOR (RECOMMENDED)
=============================================================================
  - dict-lookup for O(1) event-name → rule classification
  - Sliding window counter per principal for burst detection
  - Allowlist set for suppressing known-good principals

  Time : O(1) per event
  Space: O(active principals × burst_window)

=============================================================================
"""

from __future__ import annotations

from collections import defaultdict, deque
from dataclasses import dataclass, field
from datetime import datetime
from typing import Dict, FrozenSet, List, Optional, Set, Tuple

# Rule definitions

# eventName → (alert_type, severity, description)
_CT_CRITICAL_EVENTS: Dict[str, Tuple[str, str, str]] = {
    "StopLogging":             ("ct_stop_logging",      "critical", "CloudTrail logging stopped"),
    "DeleteTrail":             ("ct_delete_trail",       "critical", "CloudTrail trail deleted"),
    "UpdateTrail":             ("ct_update_trail",       "high",     "CloudTrail trail modified"),
    "PutEventSelectors":       ("ct_event_selectors",    "high",     "CloudTrail event selectors changed"),
    "DeleteLogGroup":          ("ct_delete_log_group",   "high",     "CloudWatch log group deleted"),
    "PutBucketPolicy":         ("ct_s3_bucket_policy",   "high",     "S3 CT bucket policy changed"),
    "DeleteBucket":            ("ct_delete_bucket",      "critical", "S3 CT bucket deleted"),
    "DeleteBucketLifecycle":   ("ct_lifecycle_deleted",  "high",     "S3 lifecycle policy removed"),
    "PutBucketAcl":            ("ct_bucket_acl",         "high",     "S3 CT bucket ACL changed"),
    "DisableKey":              ("ct_kms_disable",        "critical", "KMS key encrypting CT disabled"),
    "ScheduleKeyDeletion":     ("ct_kms_schedule_del",   "critical", "KMS key encrypting CT scheduled for deletion"),
    "CreateKey":               ("ct_kms_new_key",        "medium",   "New KMS key created (verify scope)"),
    "PutRetentionPolicy":      ("ct_retention_policy",   "high",     "CloudWatch Logs retention policy changed"),
}

# Services for CT storage
_CT_SERVICES: FrozenSet[str] = frozenset({
    "cloudtrail.amazonaws.com",
    "s3.amazonaws.com",
    "logs.amazonaws.com",
    "kms.amazonaws.com",
})

# Data Types

@dataclass
class CloudTrailEvent:
    event_id: str
    ts: float
    region: str
    event_name: str
    event_source: str
    user_identity: dict       # {"type": "IAMUser", "arn": ..., "principalId": ...}
    source_ip: str
    request_params: dict = field(default_factory=dict)
    error_code: Optional[str] = None

@dataclass
class CTAlert:
    alert_type: str
    severity: str
    event_name: str
    ts: float
    principal: str
    region: str
    detail: str
    event_id: str


def _parse_ts(ts) -> Optional[float]:
    if isinstance(ts, (int, float)):
        return float(ts)
    try:
        return datetime.fromisoformat(str(ts).replace("Z", "+00:00")).timestamp()
    except (ValueError, TypeError):
        return None


def _extract_principal(user_identity: dict) -> str:
    """Extract a normalised principal identifier from CT userIdentity."""
    arn = user_identity.get("arn") or user_identity.get("principalId", "unknown")
    # Strip session suffix from assumed-role ARNs for normalisation
    if "assumed-role" in str(arn):
        parts = str(arn).split("/")
        if len(parts) >= 3:
            return "/".join(parts[:3])
    return str(arn)


def _parse_ct_event(raw: dict) -> Optional[CloudTrailEvent]:
    ts = _parse_ts(raw.get("eventTime") or raw.get("ts"))
    if ts is None:
        return None
    return CloudTrailEvent(
        event_id=raw.get("eventID", "unknown"),
        ts=ts,
        region=raw.get("awsRegion") or raw.get("region", "us-east-1"),
        event_name=raw.get("eventName", ""),
        event_source=raw.get("eventSource", ""),
        user_identity=raw.get("userIdentity", {}),
        source_ip=raw.get("sourceIPAddress", ""),
        request_params=raw.get("requestParameters") or {},
        error_code=raw.get("errorCode"),
    )


# ---------------------------------------------------------------------------
# APPROACH 1 — Naive rule-check (stateless)
# ---------------------------------------------------------------------------

class NaiveCTTampering:
    """
    Simple event-name lookup table detector.
    No burst detection, no stateful tracking.
    O(1) per event.
    """

    def __init__(
        self,
        approved_regions: Optional[Set[str]] = None,
        allowlist_principals: Optional[Set[str]] = None,
    ) -> None:
        self.approved_regions = approved_regions or {"us-east-1", "us-west-2", "eu-west-1"}
        self.allowlist = allowlist_principals or set()
        self.malformed = 0

    def process(self, raw: dict) -> List[CTAlert]:
        evt = _parse_ct_event(raw)
        if evt is None:
            self.malformed += 1
            return []

        principal = _extract_principal(evt.user_identity)
        if principal in self.allowlist:
            return []

        alerts = []

        # Rule: known tampering event
        if evt.event_name in _CT_CRITICAL_EVENTS:
            at, sev, desc = _CT_CRITICAL_EVENTS[evt.event_name]
            alerts.append(CTAlert(
                alert_type=at, severity=sev,
                event_name=evt.event_name, ts=evt.ts,
                principal=principal, region=evt.region,
                detail=desc,
                event_id=evt.event_id,
            ))

        # Rule: root account usage
        if evt.user_identity.get("type") == "Root":
            alerts.append(CTAlert(
                alert_type="root_account_usage", severity="critical",
                event_name=evt.event_name, ts=evt.ts,
                principal="root", region=evt.region,
                detail="Root account used for API call",
                event_id=evt.event_id,
            ))

        # Rule: unusual region
        if evt.region not in self.approved_regions:
            alerts.append(CTAlert(
                alert_type="unusual_region", severity="medium",
                event_name=evt.event_name, ts=evt.ts,
                principal=principal, region=evt.region,
                detail=f"API call from unapproved region {evt.region!r}",
                event_id=evt.event_id,
            ))

        return alerts


# ---------------------------------------------------------------------------
# APPROACH 2 — Full stateful detector with burst detection (RECOMMENDED)
# ---------------------------------------------------------------------------

class CloudTrailTamperingDetector:
    """
    Full stateful CloudTrail tampering detector.
    Adds burst detection (rapid probing of CT APIs) on top of the naive checks.
    O(1) per event.
    """

    def __init__(
        self,
        approved_regions: Optional[Set[str]] = None,
        allowlist_principals: Optional[Set[str]] = None,
        burst_threshold: int = 5,
        burst_window_seconds: float = 60.0,
    ) -> None:
        self.approved_regions = approved_regions or {"us-east-1", "us-west-2", "eu-west-1"}
        self.allowlist = allowlist_principals or set()
        self.burst_threshold = burst_threshold
        self.burst_window = burst_window_seconds
        # principal → deque of ts for burst detection
        self._burst: Dict[str, deque] = defaultdict(deque)
        # Dedup set to avoid re-alerting on same burst in same window
        self._burst_alerted: Set[str] = set()
        self.malformed = 0

    def process(self, raw: dict) -> List[CTAlert]:
        evt = _parse_ct_event(raw)
        if evt is None:
            self.malformed += 1
            return []

        principal = _extract_principal(evt.user_identity)
        if principal in self.allowlist:
            return []

        alerts: List[CTAlert] = []

        # --- Known tampering event ---
        if evt.event_name in _CT_CRITICAL_EVENTS:
            at, sev, desc = _CT_CRITICAL_EVENTS[evt.event_name]
            # Escalate to critical if called by root
            if evt.user_identity.get("type") == "Root":
                sev = "critical"
            alerts.append(CTAlert(
                alert_type=at, severity=sev,
                event_name=evt.event_name, ts=evt.ts,
                principal=principal, region=evt.region,
                detail=f"{desc} (params: {list(evt.request_params.keys())[:5]})",
                event_id=evt.event_id,
            ))

        # --- Root account ---
        if evt.user_identity.get("type") == "Root":
            alerts.append(CTAlert(
                alert_type="root_account_usage", severity="critical",
                event_name=evt.event_name, ts=evt.ts,
                principal="root", region=evt.region,
                detail="Root account used — no valid use case for normal operations",
                event_id=evt.event_id,
            ))

        # --- Unusual region ---
        if evt.region not in self.approved_regions:
            alerts.append(CTAlert(
                alert_type="unusual_region", severity="medium",
                event_name=evt.event_name, ts=evt.ts,
                principal=principal, region=evt.region,
                detail=f"Event in unapproved region {evt.region!r}",
                event_id=evt.event_id,
            ))

        # --- Burst detection: rapid CT API calls from same principal ---
        if evt.event_source in _CT_SERVICES:
            q = self._burst[principal]
            cutoff = evt.ts - self.burst_window
            while q and q[0] < cutoff:
                q.popleft()
            q.append(evt.ts)

            if len(q) > self.burst_threshold and principal not in self._burst_alerted:
                self._burst_alerted.add(principal)
                alerts.append(CTAlert(
                    alert_type="ct_api_burst", severity="high",
                    event_name=evt.event_name, ts=evt.ts,
                    principal=principal, region=evt.region,
                    detail=(f"Principal made {len(q)} CloudTrail-related API calls "
                            f"in {self.burst_window}s"),
                    event_id=evt.event_id,
                ))
            elif len(q) <= self.burst_threshold:
                # Reset dedup if burst subsided
                self._burst_alerted.discard(principal)

        return alerts

    def stats(self) -> dict:
        return {
            "tracked_principals": len(self._burst),
            "malformed": self.malformed,
        }

def _raw_event(name, region="us-east-1", principal_type="IAMUser",
               principal_arn="arn:aws:iam::123:user/alice",
               ts=1_000_000.0, source="cloudtrail.amazonaws.com", **kwargs) -> dict:
    return {
        "eventID": f"e-{name}-{ts}",
        "eventTime": ts,
        "awsRegion": region,
        "eventName": name,
        "eventSource": source,
        "userIdentity": {"type": principal_type, "arn": principal_arn},
        "sourceIPAddress": "10.0.0.1",
        **kwargs,
    }


def test_stop_logging():
    det = CloudTrailTamperingDetector()
    alerts = det.process(_raw_event("StopLogging"))
    assert any(a.alert_type == "ct_stop_logging" for a in alerts)
    assert any(a.severity == "critical" for a in alerts)
    print("StopLogging: passed")


def test_root_usage():
    det = CloudTrailTamperingDetector()
    alerts = det.process(_raw_event("DescribeTrails", principal_type="Root"))
    assert any(a.alert_type == "root_account_usage" for a in alerts)
    print("Root usage: passed")


def test_unusual_region():
    det = CloudTrailTamperingDetector()
    alerts = det.process(_raw_event("ListTrails", region="ap-southeast-2"))
    assert any(a.alert_type == "unusual_region" for a in alerts)
    print("Unusual region: passed")


def test_allowlist():
    det = CloudTrailTamperingDetector(allowlist_principals={"arn:aws:iam::123:user/alice"})
    alerts = det.process(_raw_event("StopLogging"))  # alice is allowlisted
    assert len(alerts) == 0
    print("Allowlist suppression: passed")


def test_burst_detection():
    det = CloudTrailTamperingDetector(burst_threshold=3, burst_window_seconds=60)
    results = []
    for i in range(5):
        results += det.process(_raw_event("DescribeTrails", ts=float(1000 + i * 5)))
    assert any(a.alert_type == "ct_api_burst" for a in results), \
        f"Expected burst alert; got {[a.alert_type for a in results]}"
    print("Burst detection: passed")


def test_no_alert_on_benign():
    det = CloudTrailTamperingDetector()
    alerts = det.process(_raw_event("GetEventSelectors",
                                    source="cloudtrail.amazonaws.com"))
    # Not a tampering event — no alert
    assert all(a.alert_type not in _CT_CRITICAL_EVENTS for a in alerts)
    print("No alert on benign: passed")


def test_malformed():
    det = CloudTrailTamperingDetector()
    r = det.process({})  # no eventTime
    assert r == []
    assert det.malformed == 1
    print("Malformed event: passed")


def test_naive_all_sample():
    det = NaiveCTTampering()
    all_alerts = []
    for evt in SAMPLE_EVENTS:
        all_alerts += det.process(evt)
    types = {a.alert_type for a in all_alerts}
    assert "ct_stop_logging" in types
    assert "ct_delete_trail" in types
    assert "root_account_usage" in types
    print(f"Sample events: {types}")

"""
THREAT MODEL
  The first action a skilled attacker takes after gaining cloud admin access
  is to cover their tracks by disabling CloudTrail.  Examples:

  - TeamTNT (2020): disabled CloudTrail in compromised AWS accounts.
  - Pacu framework: has a dedicated "cloudtrail__backdoor_event_selectors" module.
  - Rhino Security Labs research: showed that UpdateTrail + PutEventSelectors
    can silently stop logging data plane events while management events continue,
    making the trail appear healthy.

KEY EVASION TECHNIQUES TO DETECT
  1. UpdateTrail to remove S3 data events (appears healthy, misses S3 exfil logs).
  2. PutEventSelectors to set ReadWriteType=ReadOnly (misses write actions).
  3. DeleteLogGroup in CloudWatch to destroy ingested logs while trail "runs".
  4. Using IAM role from a different region to do CT operations (unusual region).

HARDENING CONTROLS
  - Enable CloudTrail organisation trail with SCP blocking StopLogging/DeleteTrail.
  - Enable S3 MFA Delete on the CT bucket.
  - Use S3 Object Lock (WORM) on the CT bucket.
  - Set up SNS notification on StopLogging event (AWS Managed Rule: cloudtrail-enabled).
  - Use separate "security" AWS account for CT storage; no cross-account delete.
  - Log aggregation: ship CT to SIEM in real-time — attacker cannot delete historical logs.

DETECTION GAPS
  - If attacker deletes logs first then stops trail — historical events are gone.
  - Root account usage cannot be blocked by SCPs (root is exempt from SCPs).
  - KMS key deletion has a 7-30 day waiting period — detect ScheduleKeyDeletion.
"""
