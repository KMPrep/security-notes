Disk Forensics

Category: Digital Forensics — Storage and File System Analysis

Overview

Disk forensics examines persistent storage — HDDs, SSDs, USB drives, VM disk images, cloud storage snapshots — to reconstruct attacker activity, recover deleted data, and extract evidence. Even after deletion, significant data remains recoverable because most operating systems only mark blocks as free rather than overwriting them immediately.

File Systems

Understanding the file system is essential because forensic artefacts are structured by it.

File System OS Key Features for Forensics
NTFS Windows MFT (Master File Table), alternate data streams, $LogFile, $UsnJrnl, timestamps (MACE)
ext4 Linux Journal, inode table, deleted file recovery via inode inspection
APFS macOS (2017+) Snapshots, copy-on-write, strong encryption, time-stamped clones
FAT32/exFAT USB, embedded Simple; limited metadata; easy to recover deleted files
HFS+ macOS (legacy) Catalog B-tree, resource forks

Windows NTFS Artefacts

Master File Table (MFT)

Every file and directory has an MFT entry. Even after deletion, the MFT entry persists until overwritten. Contains:

  • File name, size, timestamps (MACE: Modified, Accessed, Created, MFT Entry Changed)
  • File data (inline for small files) or pointers to data clusters
  • Security descriptor, owner SID

Tool: mftdump, analyzeMFT, FTK Imager

NTFS Timestamps (MACE)

Timestamp Meaning Forensic Use
M Modified Last data write When file content last changed
A Accessed Last read Touched by attacker? (unreliable — often disabled)
C Created File creation When malware was dropped
E MFT Entry Changed Metadata change When permissions or name changed

Timestomping: Attackers modify MACE timestamps to disguise when malware was placed. Detection: compare $STANDARD_INFORMATION timestamps with $FILE_NAME timestamps — the latter is harder to modify from user space.

$LogFile and $UsnJrnl

  • $LogFile: NTFS transaction log for crash recovery. Contains recent file operations. Useful for recovering recently deleted files.
  • $UsnJrnl ($Extend\$UsnJrnl): Update Sequence Number Journal. Logs all file and directory changes with timestamps. Critical for reconstructing file activity timeline even after deletion.

Tool: MFTECmd by Eric Zimmermann

Windows Prefetch

Prefetch files record when executables were run, the file path, and up to 8 run times.

  • Location: C:\Windows\Prefetch\
  • Format: EXECUTABLE.EXE-XXXXXXXX.pf
  • Provides: execution evidence even if malware binary was deleted

Tool: PECmd (Zimmermann Tools)

Windows Event Logs

Stored at C:\Windows\System32\winevt\Logs\. Key logs:

Log Key Events
Security 4624/4625 logon, 4688 process creation, 4698 scheduled task
System 7045 service install, 6005/6006 system start/stop
Application Application errors, crash info
Sysmon (if deployed) Process creation with command line, network connections, file hash
PowerShell Script block logging, module logging

Windows Registry Hives

Hive Location Forensic Value
SYSTEM C:\Windows\System32\config\SYSTEM Services, USB device history, network config, timezone
SOFTWARE C:\Windows\System32\config\SOFTWARE Installed software, run keys, MRU lists
SAM C:\Windows\System32\config\SAM Local user accounts and password hashes
NTUSER.DAT C:\Users\<user>\NTUSER.DAT Per-user run keys, recently accessed files, typed paths
USRCLASS.DAT C:\Users\<user>\AppData\Local\Microsoft\Windows\UsrClass.dat ShellBags (folder access history)

Key persistence locations in registry:

  • HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run
  • HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run
  • HKLM\SYSTEM\CurrentControlSet\Services

Tool: RegRipper, Registry Explorer (Zimmermann)

ShellBags

ShellBags record every folder a user has browsed via Windows Explorer, including folders on USB drives and network shares — even after the drive is disconnected.

Location: USRCLASS.DAT → Local Settings\Software\Microsoft\Windows\Shell\BagMRU

Tool: ShellBagsExplorer

LNK Files and Jump Lists

  • LNK files: Windows shortcut files created when a file is opened. Contain: target file path, MAC timestamps, volume serial number, MAC address of originating machine.
  • Jump Lists: Recent file lists per application. Persist after target file deletion.

Location: C:\Users\<user>\AppData\Roaming\Microsoft\Windows\Recent\

Tool: LECmd, JLECmd (Zimmermann)

Linux Forensic Artefacts

Artefact Location Forensic Value
Bash history ~/.bash_history Commands executed by user

Systemd unit files /etc/systemd/system/ Persistence via new service

Data Carving

Data carving recovers files from unallocated disk space by looking for file format signatures (magic bytes) rather than file system metadata. Useful when:

  • File system is damaged
  • Attacker deleted files
  • Files were never recorded in the file system (e.g., in-memory-only files that were swapped)

Common file signatures (magic bytes):

Tools: foremost, scalpel, bulk_extractor, PhotoRec

Anti-Forensics Techniques

Attackers attempt to destroy or mislead forensic analysis:

Technique What It Does Detection
Timestomping Modifies file MACE timestamps Compare $STANDARD_INFORMATION vs $FILE_NAME timestamps
Secure delete Overwrites file content before deletion Recovery impossible; detect via $UsnJrnl entries
Log clearing Clears Windows Event Logs Event ID 1102 (Security log cleared), 104 (System log cleared)
HISTFILE manipulation unset HISTFILE or export HISTSIZE=0 Absence of history file or audit log gaps
Rootkits Hide files, processes from OS APIs Detect by comparing OS API results vs raw disk reads
Volume encryption (at exfil) Encrypt data before exfil to prevent content analysis Network metadata still visible
Steganography Hide data in image/audio files Detect via statistical steganalysis
