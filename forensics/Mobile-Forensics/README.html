Mobile Forensics

Category: Digital Forensics — Mobile Devices, iOS and Android

Overview

Mobile forensics extracts, preserves, and analyses data from mobile devices — smartphones, tablets, wearables. Compared to traditional computer forensics, mobile forensics is harder because:

  • Devices are encrypted by default (iOS since iPhone 3GS; Android since 6.0 required)
  • Remote wipe can destroy evidence instantly
  • Proprietary file systems and backup formats vary by vendor
  • Anti-forensics is built in (biometric timeouts lock devices quickly)
  • Cloud sync means evidence may be in multiple places simultaneously

Mobile vs Computer Forensics

Property Computer Forensics Mobile Forensics
Default encryption Often optional, OS/user-configured On by default (iOS, modern Android)
Acquisition method Live acquisition or disk imaging Device must be unlocked for most data
File system NTFS, ext4, APFS — well documented Custom file systems; proprietary layouts
Cloud integration Optional Deep — iCloud, Google Photos, etc.
Remote wipe risk Low High — MDM or user can wipe remotely
Evidence locations Mostly local Split: local + cloud + carrier
Anti-forensics built-in Uncommon Biometric timeout, USB restricted mode

Acquisition Methods

Mobile acquisition has four levels of depth, with increasing difficulty and risk of data alteration:

Level Method What It Gets Requires Unlock
Manual Photographing screen, manually scrolling Visible on-screen data only No (but limited)
Logical ADB backup (Android) or iTunes/Finder backup (iOS) App data, messages, call history Yes (trusted pairing)
File system Full file system extraction via agent or exploit All user files, databases, media Yes + trust or jailbreak
Physical Chip-off or JTAG; full flash dump Everything including deleted data No (hardware access)

iOS Acquisition Notes

  • Locked device: Limited to metadata unless passcode is known or device is exploited.
  • After first unlock (AFU) vs Before first unlock (BFU): In AFU state, most data is decrypted. In BFU state (immediately after reboot), only some data is accessible. Law enforcement and investigators prefer AFU.
  • USB Restricted Mode: After 1 hour locked, iOS blocks USB data transfer. Disable by unlocking within that window.
  • GrayKey / Cellebrite UFED: Commercial tools used by law enforcement to unlock and extract iOS devices via exploit chains. Capabilities change with each iOS version.

Android Acquisition Notes

  • ADB (Android Debug Bridge): Must be enabled in developer options. Provides logical backup.
  • Rooted device: Allows full file system access and physical image.
  • EDL (Emergency Download Mode) / Qualcomm 9008: Low-level mode for Qualcomm chips; provides physical acquisition without root.
  • Full Disk Encryption (FDE) vs File-Based Encryption (FBE): Modern Android uses FBE — some data (emergency calls, accessibility) accessible before unlock; user data requires unlock.

Jailbreaking and Rooting

Concept iOS Android
Term Jailbreak Root
Purpose Remove iOS sandboxing; gain root access Gain superuser access
Forensic value Allows full file system extraction Allows full file system extraction
Risk May damage evidence; can trigger remote wipe May trigger data wipe on some devices
Persistence Tethered (requires USB on reboot) or untethered Permanent (if using Magisk, etc.)

Forensic implication: Jailbreaking or rooting for acquisition is a last resort. Always attempt logical acquisition first. Document any jailbreak used in chain of custody notes.

SQLite on Mobile

Almost all mobile apps store data in SQLite databases. This makes them highly recoverable:

  • Messages: sms.db, WhatsApp msgstore.db, Telegram cache4.db
  • Browser history: SQLite History database
  • Call logs: SQLite
  • App data: Virtually all major apps use SQLite

Recovery of deleted records: SQLite marks records as free but does not zero the space. Deleted rows remain in the database file until the space is reused. Tools like undark and SQLite Forensic Explorer recover deleted rows.

Common analysis:

-- WhatsApp messages
SELECT key_remote_jid, data, timestamp FROM messages ORDER BY timestamp;

-- iOS Safari history
SELECT id, url, title, visit_time FROM history_visits JOIN history_items USING (history_item) ORDER BY visit_time;

Anti-Forensics on Mobile

Technique How It Prevents Forensics Countermeasure
Remote wipe (MDM / Find My) Instantly erases all data Isolate device from network (Faraday bag) immediately
Strong passcode Blocks most acquisition methods Logical or physical tools; social engineering for passcode
Biometric + auto-lock Device locks quickly; biometrics can be revoked Keep screen on; acquire before timeout
Encrypted backups iTunes backup encrypted; cloud backup encrypted Extract passcode or key first
Disappearing messages Signal, WhatsApp disappearing messages Must capture before expiry
Factory reset Overwrites user data partition Physical acquisition of flash chip may recover residual data
Faraday concerns (attacker) Attacker puts device in Faraday to prevent remote access Respect evidence — acquire quickly anyway

First step with any mobile device in forensic custody: place in a Faraday bag to prevent remote wipe and block network access.
