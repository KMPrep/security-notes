Mobile Application Security

Category: Mobile Security — App-Level Vulnerabilities, Testing, and Defences

Overview

Mobile application security covers the vulnerabilities specific to apps running on Android and iOS — beyond the OS-level controls covered in the platform notes. This includes how to assess apps (static and dynamic analysis), the OWASP Mobile Top 10, certificate pinning and its bypass, runtime instrumentation, and how apps leak data or mishandle authentication.

Mobile AppSec Testing Methodology

flowchart TD
    RECON[Reconnaissance\nApp Store metadata\nDomain enumeration\nAPI endpoint discovery] --> STATIC[Static Analysis\nAPK IPA decompile\nManifest entitlements review\nHardcoded secrets strings]
    STATIC --> DYNAMIC[Dynamic Analysis\nProxy traffic Burp\nFrida runtime hooks\nStrace ptrace]
    DYNAMIC --> STORAGE[Storage Analysis\nSQLite SharedPrefs NSUserDefaults\nKeychain dump on jailbreak\nLog file inspection]
    STORAGE --> AUTH[Auth Testing\nToken handling\nBiometric bypass\nOAuth flow analysis]
    AUTH --> REPORT[Report\nOWASP Mobile findings\nCVSS ratings\nRemediation guidance]

Static Analysis

Android (APK)

# Decompile APK to Java source
jadx -d output/ app.apk

# Decompile to smali bytecode
apktool d app.apk -o output/

# Search for hardcoded secrets
grep -r "password\|apikey\|secret\|token\|aws\|private" output/ --include="*.java"

# Check AndroidManifest for exported components
grep -i "exported=\"true\"" output/AndroidManifest.xml

# Check for debuggable flag
grep "debuggable" output/AndroidManifest.xml

# Check network security config for cleartext
grep -r "cleartextTrafficPermitted" output/

iOS (IPA)

# Extract IPA (it's a ZIP)
unzip app.ipa -d output/
cd output/Payload/App.app/

# Check entitlements
codesign -d --entitlements :- App

# Decrypt a locally installed app (on jailbroken device)
frida-ios-dump -u mobile -H 127.0.0.1 -p 2222 com.example.app

# Dump strings from binary
strings App | grep -i "http\|password\|key\|token"

# Decompile binary to pseudo-code (Hopper / Ghidra / Binary Ninja)

Certificate Pinning

Certificate pinning prevents MITM by hard-coding one or more expected certificates or public key hashes into the app. Even if the attacker has a CA-signed certificate, it fails if it doesn't match the pinned value.

Android Pinning Implementation

<!-- Network Security Config pinning -->
<pin-set expiration="2027-01-01">
    <pin digest="SHA-256">base64encodedSPKIhash=</pin>
    <pin digest="SHA-256">backupPinHash=</pin>
</pin-set>

// OkHttp programmatic pinning
CertificatePinner pinner = new CertificatePinner.Builder()
    .add("api.example.com", "sha256/AAAA...==")
    .build();
OkHttpClient client = new OkHttpClient.Builder()
    .certificatePinner(pinner)
    .build();

iOS Pinning Implementation

// URLSession with custom delegate
func urlSession(_ session: URLSession, didReceive challenge: URLAuthenticationChallenge,
    completionHandler: @escaping (URLSession.AuthChallengeDisposition, URLCredential?) -> Void) {
    guard let serverTrust = challenge.protectionSpace.serverTrust,
          let certificate = SecTrustGetCertificateAtIndex(serverTrust, 0) else {
        completionHandler(.cancelAuthenticationChallenge, nil)
        return
    }
    let remoteCertData = SecCertificateCopyData(certificate) as Data
    let localCertData = localCertificate  // bundled in app
    if remoteCertData == localCertData {
        completionHandler(.useCredential, URLCredential(trust: serverTrust))
    } else {
        completionHandler(.cancelAuthenticationChallenge, nil)
    }
}

Frida — Runtime Instrumentation

Frida is a dynamic instrumentation framework that injects a JavaScript agent into running processes. It is the primary tool for mobile runtime analysis, pinning bypass, biometric bypass, and reverse engineering.

# Start Frida server on Android (rooted)
adb push frida-server /data/local/tmp/
adb shell "chmod 755 /data/local/tmp/frida-server && /data/local/tmp/frida-server &"

# List running apps
frida-ps -U

# Attach to app and run script
frida -U -f com.example.app -l bypass_pinning.js --no-pause

# Interactive REPL
frida -U com.example.app

Insecure Data Storage

One of the most common mobile findings. Sensitive data stored in locations that are accessible to attackers (malicious app, backup, rooted device, or physical extraction).

Android Storage Locations and Risks

Location Accessibility Risk
SharedPreferences (/data/data/<pkg>/shared_prefs/) App-private; accessible with root Tokens stored in plaintext — common finding
SQLite database (/data/data/<pkg>/databases/) App-private; accessible with root Messages, PII, cached API responses
External storage (/sdcard/) World-readable pre-API 19; READ_EXTERNAL_STORAGE API 19-29 Avoid for sensitive data
Logcat All apps can read their own logs; adb logcat by device owner Tokens/PII in log statements
App backups ADB backup includes app-private storage by default Disable with allowBackup="false"
Clipboard Any app can read clipboard (pre-Android 10) Passwords copied to clipboard

iOS Storage Locations and Risks

Location Accessibility Risk
NSUserDefaults (plist) App container; included in backup Avoid for secrets; plaintext
Keychain Hardware-backed; configurable Data Protection class Safest storage; use always for credentials
SQLite in Documents/ In iCloud backup; accessible on jailbreak May contain conversation history, PII
Files in tmp/ Not backed up; may be accessible on jailbreak Temporary sensitive processing data
Clipboard Pasteboard accessible to other apps (pre-iOS 14) iOS 14+ shows banner; still leaks
NSLog / print Not accessible to other apps; in crash logs Dev logs left in release builds

Authentication and Biometrics

Biometric Authentication Bypass

On Android, apps using BiometricPrompt can choose whether authentication is crypto-bound or not:

// STRONG — key only usable after biometric auth; cannot be bypassed by Frida alone
BiometricPrompt.AuthenticationCallback callback = ...;
CryptoObject cryptoObject = new BiometricPrompt.CryptoObject(cipher);
biometricPrompt.authenticate(cryptoObject, cancellationSignal, executor, callback);

// WEAK — no crypto binding; biometric check is advisory; Frida can bypass
biometricPrompt.authenticate(cancellationSignal, executor, callback);

Bypass: Frida can hook onAuthenticationSucceeded() and call it directly — defeating non-crypto-bound biometrics. Only crypto-bound implementations tie authentication to actual hardware key use.

On iOS, LAContext.evaluatePolicy similarly has two modes:

  • LAPolicyDeviceOwnerAuthenticationWithBiometrics — biometric only; Frida-bypassable
  • Keychain item with kSecAccessControlBiometryCurrentSet — hardware-bound; not bypassable without physical biometric

JWT and Token Storage

Storage Platform Risk
