Wi-Fi PHY - Wireless LAN Physical Layer

OSI Layer: 1 - Physical
Standard: IEEE 802.11 family

Overview

The Wi-Fi physical layer defines how bits become radio signals over the air. It specifies frequency bands, channel widths, modulation schemes, coding, preambles, spatial streams, and power levels for 802.11 networks. Modern Wi-Fi PHY generations evolved from DSSS and early OFDM into MIMO, MU-MIMO, OFDMA, beamforming, and higher-order QAM to improve throughput and spectral efficiency.

For security engineering, the PHY layer sets the real-world constraints attackers and defenders must live with: interference, jamming, signal leakage, coverage mapping, directional attacks, and side-channel clues all begin here.

Core Mechanics

Important PHY concepts:

Concept Description
Band 2.4 GHz, 5 GHz, and 6 GHz operating ranges
Channel width 20, 40, 80, or wider channels affect throughput and interference
Modulation BPSK through high-order QAM determine bit density per symbol
Coding Adds redundancy to improve recoverability in noisy environments
MIMO Uses multiple antennas and spatial streams
OFDM/OFDMA Splits transmission across many subcarriers

Representative PHY evolution:

Generation PHY characteristics
802.11b DSSS in 2.4 GHz
802.11a/g OFDM, higher data rates
802.11n MIMO and channel bonding
802.11ac Wider channels, MU-MIMO, high-order QAM
802.11ax OFDMA, better efficiency, dense-client support

Step-by-Step Flow

1. Turning bits into RF energy

flowchart TD
	A[MAC frame bits] --> B[Add coding and interleaving]
	B --> C[Map bits to modulation symbols]
	C --> D[Place symbols on subcarriers]
	D --> E[Transmit over antenna chain]
	E --> F[Receiver samples, demodulates, and reconstructs bits]

2. PHY exchange on the air

sequenceDiagram
	participant T as Transmitter
	participant A as Air
	participant R as Receiver
	T->>A: Preamble and PHY header
	T->>A: Modulated data symbols
	A->>R: RF signal with attenuation and noise
	R->>R: Synchronize, estimate channel, decode symbols
	R-->>T: Recover frame for MAC processing

3. Rate adaptation decision path

flowchart TD
	A[Measure signal and error rate] --> B{Channel quality good?}
	B -- Yes --> C[Use higher modulation and coding rate]
	B -- No --> D[Fall back to more robust rate]
	C --> E[Transmit next frame]
	D --> E

What Can Go Wrong

Failure mode What happens Security impact
Jamming Noise or interference overwhelms signal Wireless DoS
Co-channel interference Nearby radios compete for the same spectrum Reduced reliability and degraded coverage
Coverage leakage Signal extends beyond intended boundary Drive-by attack surface and rogue client access
Weak rate adaptation Links oscillate between rates Performance collapse under load
Hidden node conditions Stations cannot hear each other well Retries, poor throughput, local contention abuse
Poor RF design Bad channel plan or antenna placement Easier disruption and uneven security posture

Mitigation Controls

Control How it helps
RF site surveys Quantify coverage, leakage, interference, and channel overlap
Channel planning Reduces contention and adjacent-channel problems
Directional and placement controls Shrinks unnecessary signal exposure
Spectrum monitoring Detects jamming, rogue transmitters, and abnormal interference
Adaptive power and rate tuning Improves stability in changing environments
Layered security above PHY Assumes radio reachability cannot be perfectly constrained

Top 3 Critical Mitigations

  1. Perform regular RF surveys and spectrum monitoring instead of relying on static assumptions.
  2. Design channel plans and AP placement to limit interference and unnecessary signal leakage.
  3. Assume the radio layer is observable by attackers and enforce strong security at higher layers.

  5. Why do hidden nodes matter even when encryption is strong?

  7. What would an enterprise RF monitoring program need to measure continuously?
  8. Why is perfect physical containment of Wi-Fi rarely realistic?

  10. When is it rational to accept lower throughput for better reliability and security margin?

Key Takeaways

  • Wi-Fi PHY determines how reliably and how far wireless signals travel, which directly shapes the attack surface.
  • Jamming, interference, and signal leakage are practical security concerns, not just performance issues.
  • Good RF engineering and continuous monitoring reduce exposure, but higher-layer security still has to assume hostile listeners.
