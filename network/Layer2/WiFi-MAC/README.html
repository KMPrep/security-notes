Wi-Fi MAC - IEEE 802.11 MAC Layer

OSI Layer: 2 - Data Link
Standard: IEEE 802.11

Overview

The Wi-Fi MAC layer controls how stations share a wireless medium that cannot reliably detect collisions while transmitting. Instead of Ethernet's classic collision detection model, 802.11 uses collision avoidance, randomized backoff, acknowledgments, management frames, and association state to coordinate clients and access points. The MAC layer also carries security signaling for authentication, association, QoS, roaming support, and power-save behavior.

For defenders, Wi-Fi MAC is where a large share of wireless abuse becomes visible: rogue access points, deauthentication attacks, spoofed management traffic, client tracking, and airtime exhaustion all happen here.

Core Mechanics

Core MAC functions include:

Function Description
Channel access Uses CSMA/CA and random backoff before transmitting
Framing Builds management, control, and data frames
Reliability Uses ACKs and retransmissions for unicast delivery
Association state Tracks authentication, association, and roaming
Power save Coordinates client sleep and wake behavior
QoS Prioritizes traffic classes through EDCA/WMM

Important frame classes:

Frame type Typical use
Management Beacon, probe, auth, association, deauth
Control RTS, CTS, ACK, Block Ack
Data User traffic carried after association

	C -- Yes --> E[Start random backoff]

	F --> G[Transmit frame]

What Can Go Wrong

Failure mode What happens Security impact
Deauthentication or disassociation abuse Attackers spoof management frames Client disconnects and forced roaming
Rogue AP / evil twin Fake AP mimics trusted SSID Credential theft and traffic interception
Airtime exhaustion Excessive management or retry traffic fills channel Local DoS
MAC spoofing Attacker impersonates another station Weak identity controls and policy bypass
Weak management-frame protection Unprotected control path remains forgeable Downgrade and session disruption
Probe privacy leakage Clients broadcast preferred networks Tracking and targeting

Mitigation Controls

Control How it helps
WPA3 or strong WPA2-Enterprise Strengthens authentication and session keying
802.11w Protected Management Frames Protects key management frames from spoofing
Rogue AP detection Finds evil twins and unauthorized radios
Client isolation and network segmentation Limits blast radius after association
Wireless IDS/IPS monitoring Detects deauth floods, spoofing, and abnormal airtime patterns
Disable legacy weak modes Reduces downgrade paths and insecure compatibility features

Top 3 Critical Mitigations

  1. Enable Protected Management Frames wherever client support allows it.
  2. Use enterprise-grade authentication and continuously monitor for rogue access points.
  3. Treat wireless as a hostile shared medium and segment associated clients aggressively.

Key Takeaways

  • Wi-Fi MAC is responsible for access control, framing, acknowledgments, and station state on a shared wireless medium.
  • Many practical wireless attacks target management behavior rather than payload encryption directly.
  • Protected management frames, strong authentication, and wireless monitoring are the highest-value controls.
