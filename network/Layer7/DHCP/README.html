DHCP — Dynamic Host Configuration Protocol

OSI Layer: 7 — Application
RFC: RFC 2131 (DHCPv4), RFC 8415 (DHCPv6)

Overview

DHCP assigns IP addresses and related network settings to hosts so they can join a network without manual configuration. Typical settings include an address, subnet mask, default gateway, DNS servers, lease duration, and other option data. DHCPv4 is best known for the DORA exchange over UDP ports 67 and 68, while DHCPv6 uses a related but distinct message set and often coexists with SLAAC. Because DHCP controls who gets network parameters, it is operationally critical and a useful target for abuse.

How It Works

Core Mechanics

When a DHCPv4 client boots without an address, it broadcasts DHCPDISCOVER on the local link. One or more servers reply with DHCPOFFER, and the client selects an offer by sending DHCPREQUEST. The server finalizes the lease with DHCPACK, after which the client can use the assigned address until renewal time.

DHCP messages carry a client identifier, transaction ID, flags, offered address, server identifier, and a flexible set of options. Lease management matters as much as initial assignment: clients renew before expiration, servers reclaim abandoned leases, and reservations can pin specific clients to specific addresses. DHCPv6 changes the message names and transport details, and in many environments it operates alongside router advertisements rather than fully replacing them.

Top 3 Critical Mitigations

1. Enable DHCP Snooping on Managed Switches

DHCP snooping marks only known uplinks or server-facing ports as trusted for DHCP replies. This blocks rogue servers on access ports from assigning malicious network settings to clients. The trade-off is operational discipline: if trust is configured incorrectly, valid DHCP can break.

2. Design Lease Pools and Durations Deliberately

Many DHCP incidents are caused by poor scope design rather than adversaries. Short enough leases help recovery in churn-heavy environments, while long enough leases avoid unnecessary renew traffic in stable segments. The right values depend on the environment, not a universal default.

3. Treat DHCP as Security-Relevant Infrastructure

Because DHCP hands out default gateway and DNS settings, compromise or misconfiguration can redirect traffic at scale. Good logging, restricted administration, redundancy, and change review are essential. The trade-off is more process and monitoring overhead, but the blast radius of mistakes justifies it.

Practice Questions

Tier 1 — Fundamentals

  1. What does DHCP provide to a host besides an IP address? It provides related network parameters such as gateway, mask, DNS, and lease duration. This is asked to check practical familiarity.
  2. What are the four DORA steps? Discover, Offer, Request, and Acknowledge. This is asked because it is foundational DHCP behavior.
  3. Why does DHCP often use broadcast at first? The client may not yet know its own address or the server location. This is asked to test first-principles reasoning.
  4. What is a lease in DHCP? A lease is a time-bound assignment of an address and options to a client. This is asked because lease management drives real behavior.
  5. What is a rogue DHCP server? It is an unauthorized server that responds to clients with malicious or incorrect settings. This is asked to connect configuration with security impact.

Tier 2 — System Design and Trade-offs

  1. How would you design DHCP for a campus with many VLANs? I would separate scopes per segment, use relays where needed, centralize logging, and test trust boundaries on access switches. This is asked to test network design thinking.
  2. How do lease duration choices affect user experience and operations? Short leases recover quickly but increase renew traffic, while long leases reduce churn but hold stale assignments longer. This is asked to probe trade-off reasoning.
  3. Why is DHCP snooping so effective? It turns the switch into an enforcement point for who may answer DHCP. This is asked to see whether the candidate understands where controls belong.
  4. How would you troubleshoot intermittent address assignment failures? I would check pool exhaustion, relay path, trust configuration, client broadcasts, and server logs. This is asked to test structured debugging.
  5. What changes in DHCPv6 environments? Message types, addressing model, and coexistence with router advertisements all matter. This is asked to test whether the candidate can move beyond DHCPv4.

Tier 3 — Expert and Deep-Dive

  1. Why is DHCP more security-sensitive than many teams assume? It effectively influences routing and name resolution for every client that accepts its options. This is asked to test blast-radius awareness.
  2. How would you investigate a suspected rogue DHCP incident? Capture broadcasts, inspect switch snooping tables, identify the responder port, and compare lease logs to intended scopes. This is asked to test operational depth.
  3. What are the hardest DHCP problems in hybrid IPv4 and IPv6 networks? Policy consistency, client behavior differences, and overlap with router advertisements create subtle failures. This is asked to test multi-stack reasoning.
  4. How would you make DHCP resilient without making it opaque? Use redundant servers, deterministic relays, centralized logs, and clear IPAM ownership. This is asked to test reliability design.
  5. What is the difference between address assignment being correct and access actually being secure? DHCP can assign valid settings to an untrusted client unless the surrounding access-control system is strong. This is asked to separate configuration from authorization.

Key Takeaways

  • DHCP automates addressing and related host configuration.
  • Lease management is as important as initial assignment.
  • Rogue servers and starvation are common abuse patterns.
  • DHCP snooping is one of the most important practical defenses.
  • Pool sizing and lease duration choices are operationally critical.
  • DHCPv6 introduces different behavior and should be designed explicitly.
  • DHCP is part of the network trust path, not just convenience tooling.
