OSI Model and Protocols by Layer

The OSI (Open Systems Interconnection) model is a 7-layer conceptual framework that explains how data moves across a network, from an application on one device to an application on another.

Each layer has a specific role. Real-world networking stacks (like TCP/IP) do not always map 1:1 to OSI, but OSI is still the best model for learning and troubleshooting.

OSI Layers (Top to Bottom)

  1. Application (Layer 7)
  2. Presentation (Layer 6)
  3. Session (Layer 5)
  4. Transport (Layer 4)
  5. Network (Layer 3)
  6. Data Link (Layer 2)
  7. Physical (Layer 1)

Protocols at Every OSI Layer

Note: There is no truly finite list of "all" protocols. Below is a broad, commonly used study/reference list.

Layer 7 - Application

Provides services directly to user applications.

Common protocols:

  • HTTP, HTTPS
  • FTP, FTPS, SFTP, TFTP
  • SMTP, POP3, IMAP
  • DNS, mDNS
  • DHCP
  • SNMP
  • NTP
  • SSH, Telnet
  • LDAP, LDAPS
  • SMB/CIFS
  • NFS
  • SIP, SDP
  • RTP (often treated as app/session in OSI discussions)
  • RTSP
  • MQTT, AMQP, CoAP
  • WebSocket
  • RDP

Layer 6 - Presentation

Handles data format, translation, encryption, and compression.

Common standards/protocols and formats:

  • TLS/SSL
  • MIME
  • ASN.1
  • XDR
  • JSON, XML
  • JPEG, PNG, GIF
  • ASCII, Unicode

Layer 5 - Session

Manages sessions/dialogs between systems.

Common protocols/technologies:

  • NetBIOS Session Service
  • RPC
  • PPTP (often mapped to Layer 2/5 depending on source)
  • SIP session control functions
  • SMB session mechanisms

Layer 4 - Transport

Provides end-to-end host communication, reliability, segmentation, and flow control.

Common protocols:

  • TCP
  • UDP
  • SCTP
  • DCCP
  • QUIC (commonly viewed as transport in modern stacks)

Layer 3 - Network

Provides logical addressing and routing between networks.

Common protocols:

  • IPv4, IPv6
  • ICMP, ICMPv6
  • IGMP
  • IPsec (AH, ESP)
  • GRE
  • OSPF
  • RIP
  • EIGRP
  • IS-IS
  • BGP
  • PIM
  • L3 MPLS concepts/protocol support

Layer 2 - Data Link

Provides framing, MAC addressing, media access control, and error detection on the local link.

Common protocols/standards:

  • Ethernet (IEEE 802.3)
  • Wi-Fi MAC (IEEE 802.11)
  • PPP
  • HDLC
  • Frame Relay
  • ATM
  • ARP (often shown between Layer 2 and 3)
  • VLAN (IEEE 802.1Q)
  • STP, RSTP, MSTP
  • LACP (IEEE 802.1AX)
  • LLDP
  • CDP (Cisco proprietary)
  • MPLS (often called Layer 2.5)
  • MACsec (802.1AE)

Layer 1 - Physical

Transmits raw bits over physical media.

Common standards/technologies:

  • 10BASE-T, 100BASE-TX, 1000BASE-T, 10GBASE-SR
  • Fiber optics (single-mode, multi-mode)
  • Copper cabling standards (Cat5e, Cat6, Cat6a)
  • DSL
  • ISDN
  • SONET/SDH
  • RS-232
  • Bluetooth PHY
  • Wi-Fi PHY (802.11a/b/g/n/ac/ax)

Quick Memory Tip

A classic mnemonic from Layer 7 to Layer 1:

All People Seem To Need Data Processing

Why OSI Is Useful

  • Helps troubleshoot by isolating where failures occur.
  • Makes it easier to compare protocols and technologies.
  • Provides a common language for network engineers and security teams.
