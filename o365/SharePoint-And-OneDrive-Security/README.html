SharePoint & OneDrive Security

Category: Microsoft 365 Data Plane / Collaboration & Content Security
Key standards / frameworks: Microsoft Purview, Entra ID Conditional Access, NIST 800-53 (AC-3, AC-4, AC-6), ISO 27001 A.5.15/A.8.3, CIS Microsoft 365 Foundations Benchmark, MITRE ATT&CK T1530 (Data from Cloud Storage)

Overview

SharePoint Online and OneDrive for Business are the primary content repositories of Microsoft 365. Nearly every other workload stores its files here: Teams channels are backed by SharePoint document libraries, private chat attachments live in the sender's OneDrive, and Loop components and Stream recordings all resolve to these stores. Because of that, a permissions or sharing misconfiguration in SharePoint or OneDrive is rarely local; it becomes a tenant-wide data exposure.

The central security tension is that the platform is designed to make sharing frictionless while security engineering needs sharing to be intentional, scoped, and auditable. Sharing links, guest access, delegated site ownership, and the sync client each create legitimate collaboration paths that attackers and careless users can turn into exfiltration paths.

Effective control combines tenant-level guardrails, per-site posture, data-centric protection through sensitivity labels and DLP, and continuous access review to fight the slow accumulation of over-broad permissions known as sharing sprawl.

How It Works

The Sharing Model

SharePoint and OneDrive express access through sharing links and direct permissions. A sharing link can be one of three scopes. An Anyone link (often called an anonymous or "Anyone with the link" link) grants access to whoever holds the URL with no authentication, making the resource effectively public. An organization or "People in your organization" link works for any authenticated tenant member and cannot reach external users. A specific people link is scoped to named recipients, including named guests, and is the least-privilege option. Guest sharing invites an external identity into Entra ID as a B2B guest account, after which that guest can be a link target or a direct permission holder.

Tenant Versus Site Level Controls

External sharing is governed by a two-tier ceiling. The tenant-wide external sharing setting in the SharePoint admin center sets the maximum permissiveness for the whole organization, ranging from Anyone down through New and existing guests, Existing guests, and Only people in your organization. Each individual site collection then has its own external sharing setting that can only be equal to or more restrictive than the tenant ceiling. A site can never be more open than the tenant allows, so the tenant value is the true blast radius limiter. OneDrive has its own equivalent ceiling that should generally be set no higher than SharePoint.

Data Protection and Exfiltration Paths

Sensitivity labels from Microsoft Purview can be applied at the container level to a site to enforce privacy, unmanaged-device access, and external sharing on that site, and at the file level to encrypt individual documents so protection travels with the file even if it leaves the tenant. Data Loss Prevention policies scan SharePoint and OneDrive content for sensitive information types and can block or restrict sharing. The main bulk exfiltration paths are mass browser download and the OneDrive sync client, which can pull an entire library onto an unmanaged endpoint. Conditional Access App Control through Defender for Cloud Apps enforces session policies such as block download for unmanaged devices.

Detection & Telemetry

The Unified Audit Log in Microsoft Purview is the primary source. Watch operations such as AnonymousLinkCreated, SharingInvitationCreated, AddedToSecureLink, FileSyncDownloadedFull, and FileDownloaded spikes per user. Entra ID sign-in logs surface guest sign-ins and risky sessions, and Defender for Cloud Apps raises mass-download and unusual-file-share alerts. Purview DLP generates policy-match alerts, while Insider Risk Management correlates download-then-exfiltrate sequences into cases.

Key Takeaways

  • The tenant external sharing setting is the true ceiling; a site can be more restrictive but never more open, so set the tenant value deliberately.
  • Prefer specific people links over organization links, and treat Anyone links as a high-risk exception to be disabled by default.
  • Sharing sprawl is a slow-accumulation risk; recurring access reviews and permission reporting are what keep least privilege true over time.
  • Sensitivity labels and DLP move protection to the data itself, so encryption survives even when a file leaves the tenant.
  • The sync client and mass browser download are the main bulk exfiltration paths; gate them with Conditional Access App Control on unmanaged devices.
  • Guests are identities with a lifecycle; use entitlement management and access reviews so external access expires automatically.
  • Because Teams and other workloads store data in SharePoint and OneDrive, hardening these stores protects the whole M365 data plane.
