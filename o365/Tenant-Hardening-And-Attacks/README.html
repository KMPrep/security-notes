M365 Tenant Hardening & Attacks

Category: Microsoft 365 Identity & Tenant Security / Threat Defense
Key standards / frameworks: Entra ID, Microsoft Secure Score, CIS Microsoft 365 Benchmark, NIST 800-53 (IA-2, AC-6, AU-6), MITRE ATT&CK T1528 (Steal Application Access Token), T1557 (Adversary-in-the-Middle), T1114 (Email Collection), T1606 (Forge Web Credentials)

Overview

Once an organization moves to Microsoft 365, the tenant becomes the crown-jewel identity and data boundary, and attackers have shifted accordingly. Modern intrusions rarely break cryptography or exploit the platform itself; they abuse legitimate features such as OAuth consent, session tokens, and mailbox rules to gain and keep access. The defining challenge is that many of these techniques defeat traditional MFA, so hardening must move beyond the first factor.

This note covers the highest-signal tenant attacks and the layered defenses that counter them. The through-line is identity: strong phishing-resistant authentication, tight application and consent governance, least-privilege administration, and pervasive auditing. Microsoft Secure Score provides a running measure of posture, while the Unified Audit Log and Defender XDR provide the detection surface.

How It Works

Consent and Token Attacks

An illicit OAuth application consent grant, or consent phishing, tricks a user into approving a malicious multi-tenant app that requests Graph permissions such as reading mail or files. Because the user grants a legitimate OAuth token, no password is stolen and MFA is never challenged again for that app; the attacker retains programmatic access until the grant is revoked. Adversary-in-the-Middle phishing uses a reverse-proxy kit such as Evilginx that sits between the victim and the real Microsoft sign-in page, relaying credentials and MFA in real time and stealing the resulting session cookie, which lets the attacker replay an already-authenticated session and bypass MFA entirely.

Persistence and Credential Attacks

After compromise, business email compromise persistence is established through hidden inbox rules that auto-forward or delete messages, or through tenant or mailbox forwarding, so the attacker keeps reading mail even after a password reset. Golden SAML and token forgery are deeper attacks where a stolen federation signing key lets an adversary mint valid tokens for any user without touching the identity provider. Password spray tries a few common passwords across many accounts to stay under lockout thresholds and often rides in over legacy authentication protocols that cannot enforce MFA.

Layered Defense

Defenses map to each stage: restrict user consent and require an admin consent workflow, enforce phishing-resistant MFA and token protection through Conditional Access, disable legacy authentication, tier administrative roles, and place privileged roles behind just-in-time elevation with Privileged Identity Management. Continuous monitoring through the Unified Audit Log and Defender XDR closes the loop, and Secure Score tracks the aggregate posture.

Detection & Telemetry

The Unified Audit Log records the highest-value tenant events including Consent to application, Add service principal, New-InboxRule, Set-Mailbox forwarding changes, and Add member to role. Entra ID sign-in logs and Identity Protection flag impossible-travel, anomalous-token, and risky sign-in signals characteristic of AiTM replay, while Defender XDR correlates consent phishing, mailbox rule, and session-hijack activity into incidents. Microsoft Secure Score tracks posture over time, and exporting audit data to a SIEM enables long-horizon hunting for token forgery and slow persistence.

Key Takeaways

  • Modern tenant attacks abuse legitimate features such as OAuth consent, session tokens, and inbox rules rather than exploiting the platform.
  • AiTM and stolen session cookies defeat ordinary MFA, so phishing-resistant methods and token protection are now essential.
  • Illicit consent grants give attackers password-free, MFA-immune access; restricting user consent and requiring admin approval is a top control.
  • Business email compromise persists through inbox rules and forwarding, so revoke sessions and block external auto-forwarding on compromise.
  • Disabling legacy authentication removes the main path that lets password spray bypass MFA entirely.
  • Least-privilege administration through role tiering and just-in-time PIM shrinks the blast radius of any single compromise.
  • Secure Score, the Unified Audit Log, and Defender XDR together provide the posture measurement and detection surface that tie the defenses together.
