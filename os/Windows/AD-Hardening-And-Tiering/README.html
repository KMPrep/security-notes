AD Hardening & Tiered Admin Model

Category: Windows / Defensive Architecture
Key standards / frameworks: Microsoft Enterprise Access Model, Microsoft Security Compliance Toolkit baselines, CIS Benchmarks, NIST SP 800-53 AC/IA, MITRE D3FEND

Overview

Hardening Active Directory is fundamentally about controlling credential exposure and reducing attack paths, not just patching individual vulnerabilities. The organizing idea is the tiered administration model: privileged credentials must never be exposed on lower-trust systems where they can be harvested. Layered on top are technical controls that remove legacy weaknesses (NTLM, unsigned LDAP/SMB), protect credentials in memory, and shrink the number of accounts and paths that lead to domain dominance.

A mature program treats these as defensive controls measured against attack-path reality, using tools like BloodHound not just offensively but to continuously find and eliminate the shortest routes to Tier 0.

How It Works

Tiered Administration and the Enterprise Access Model

Microsoft's model divides assets into Tier 0 (identity systems: domain controllers, AD CS, and anything that can control them), Tier 1 (servers and applications), and Tier 2 (workstations and user devices). The rule is that a credential from a higher tier is never used to log on to a lower tier, preventing the credential-theft cascade that turns one workstation compromise into domain takeover. Privileged Access Workstations (PAWs) are hardened, dedicated machines from which Tier 0 administration is performed, isolated from email and web browsing.

Credential and Authentication Controls

LAPS and the newer Windows LAPS randomize and rotate every local administrator password, defeating pass-the-hash reuse across machines. The Protected Users group forces Kerberos-only authentication, disables NTLM, WDigest, and CredSSP delegation, and blocks credential caching for its members. Authentication Policies and Silos restrict which machines a privileged account can authenticate from and enforce short TGT lifetimes, effectively fencing Tier 0 accounts to PAWs.

Protocol and Service Hardening

Defenders disable or restrict NTLM, enforce SMB signing and LDAP signing with channel binding, and enable Extended Protection for Authentication to break relay attacks. AD CS hardening closes the ESC1 through ESC8 template and enrollment misconfigurations. Microsoft security baselines (delivered via the Security Compliance Toolkit) provide vetted GPO settings, and attack-path management with BloodHound quantifies exposure by mapping ACL and session edges to Tier 0.

Detection & Telemetry

Hardening is verified as much as it is monitored. NTLM usage can be surfaced for eventual removal by enabling NTLM auditing (Event IDs 8004/8003 from the Netlogon and NTLM operational channels) before restricting it. Authentication Policy and Silo enforcement generates Event IDs 4820/4821/4822/4823 when a TGT or claim is denied, which both proves the control works and flags misconfigured or abused accounts. Group membership changes to Tier 0 groups (Event IDs 4728/4732/4756) and modifications to protected ACLs should feed a SIEM. Periodic BloodHound collections provide a measurable exposure metric, turning hardening into a trend rather than a one-time project.

Key Takeaways

  • Tiered administration is the foundational control: never expose higher-tier credentials on lower-tier systems.
  • Privileged Access Workstations give Tier 0 admins a clean, isolated platform free of email and browsing risk.
  • Windows LAPS eliminates shared local admin passwords and the pass-the-hash lateral movement they enable.
  • Protected Users, Authentication Policies, and Silos fence privileged accounts to trusted hosts and enforce Kerberos-only, short-lived tickets.
  • Disabling NTLM and enforcing SMB/LDAP signing with Extended Protection removes the relay and coercion attack class.
  • AD CS hardening and Microsoft security baselines close common escalation paths and prevent configuration drift.
  • Attack-path management with BloodHound converts hardening into a measurable, continuously improving exposure metric.
