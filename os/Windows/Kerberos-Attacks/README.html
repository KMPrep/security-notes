Kerberos Attacks — Roasting, Tickets, Delegation

Category: Windows / Authentication Protocol Abuse
Key standards / frameworks: RFC 4120 (Kerberos v5), MS-KILE, MS-SFU (delegation), MITRE ATT&CK T1558 (Steal or Forge Kerberos Tickets), T1550.003 (Pass the Ticket)

Overview

Kerberos is the default authentication protocol in Active Directory, but its design places enormous trust in the secrecy of account password hashes. Every ticket is encrypted with a key derived from an account secret, so an attacker who can crack, steal, or forge those keys can impersonate users and services without ever touching a password prompt. The attacks in this note exploit weak service credentials, missing pre-authentication, delegation misconfiguration, and the ability to forge tickets once a critical hash is known.

These techniques span the kill chain from credential access to privilege escalation to domain-wide persistence, and most leave subtle rather than obvious traces, which is why targeted Kerberos telemetry matters.

How It Works

Roasting Attacks

Kerberoasting abuses the fact that any authenticated user can request a service ticket (TGS) for any SPN. The ticket is encrypted with the service account's password-derived key. If the attacker requests an RC4 (etype 0x17) ticket, they can crack it offline with Hashcat mode 13100 to recover the plaintext service account password. AS-REP roasting targets accounts that have Kerberos pre-authentication disabled: the KDC returns an AS-REP whose encrypted portion is derived from the user's password, crackable offline with Hashcat mode 18200, and no domain credentials are needed to request it.

Ticket Forgery

A Golden Ticket is a fully forged TGT created with the krbtgt account hash. Because the KDC trusts anything encrypted with that key, the attacker sets arbitrary group memberships and lifetimes, granting domain-wide access that survives password resets of individual users. A Silver Ticket is a forged TGS for one specific service, signed with that service account's hash; it never contacts a DC, so it is stealthier but scoped to a single service. Overpass-the-hash (pass-the-key) converts a stolen NTLM hash or AES key into a legitimate Kerberos TGT, upgrading a hash into full Kerberos access.

Delegation Abuse

Unconstrained delegation caches the TGT of any user who authenticates to the host, so compromising such a host plus coercing a DC to authenticate yields a DC TGT. Constrained delegation (S4U2Proxy) lets a service impersonate users to a defined SPN list, but the protocol transition can be abused to reach services the target never intended. Resource-based constrained delegation (RBCD) moves the trust to the target object; if an attacker can write the msDS-AllowedToActOnBehalfOfOtherIdentity attribute on a computer, they can impersonate any user, including Domain Admins, to that host.

Detection & Telemetry

Event IDMeaningWhat To Watch
4768Kerberos TGT requested (AS-REQ)RC4 requests, AS-REP roasting patterns, anomalous lifetimes
4769Kerberos service ticket requested (TGS-REQ)Encryption type 0x17 (RC4) and high-volume SPN requests
4770Service ticket renewedUnusually long-lived tickets from forgery

Encryption-type downgrade to RC4 on accounts that otherwise use AES is a strong Kerberoasting signal. Deploying honeypot SPNs on decoy accounts that no legitimate service ever requests provides high-fidelity alerts, since any 4769 for that SPN is malicious. Golden Tickets often show a TGT with no preceding 4768, and mismatched domain or username fields in the PAC.

Key Takeaways

  • Kerberoasting and AS-REP roasting turn weak account passwords into offline cracking targets; length and AES are the defense.
  • The krbtgt hash is the master key of the domain; its compromise enables Golden Tickets and demands a double password rotation to remediate.
  • Silver Tickets and overpass-the-hash trade breadth for stealth by forging or upgrading single-service or single-account access.
  • All three delegation flavors are dangerous when misconfigured; RBCD write access on a computer object is a full impersonation primitive.
  • Disabling RC4 and NTLM, enforcing AES, and using group managed service accounts remove most roasting and forgery opportunities.
  • Events 4768 and 4769 with RC4 encryption types, plus honeypot SPNs, are the highest-value Kerberos detections.
  • Mark Tier 0 accounts as sensitive and not delegatable, and add them to Protected Users to blunt delegation and ticket theft.
