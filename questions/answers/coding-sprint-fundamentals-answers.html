1. Find the first non-repeating character in a string

Approach: Two passes with a frequency map. First pass counts characters; second pass returns the first character whose count is 1. Insertion order is preserved by iterating the original string, so we get the first unique character.

from collections import Counter


def first_non_repeating(s: str) -> str | None:
    """Return the first character that appears exactly once, or None.

    Edge cases:
      - None input        -> TypeError (fail fast at the boundary)
      - Empty string      -> None (no characters at all)
      - All repeating      -> None
    """
    if s is None:
        raise TypeError("Input cannot be None")
    if not isinstance(s, str):
        raise TypeError("Input must be a string")

    counts = Counter(s)              # O(n) single pass to tally frequencies
    for ch in s:                     # O(n) scan in original order
        if counts[ch] == 1:
            return ch
    return None                      # every character repeats


# Time:  O(n) - two linear passes
# Space: O(k) - k distinct characters
# Security relevance: same scan pattern finds the first one-off token/IP in a stream.

assert first_non_repeating("leetcode") == "l"
assert first_non_repeating("aabb") is None
assert first_non_repeating("") is None
assert first_non_repeating("aabbc") == "c"

Practical use cases:

  • Stream triage: surface the first request ID, nonce, or session token that appears only once amid mostly duplicated traffic.
  • Log analysis: flag the first one-off field value (e.g., a lone user-agent) that may indicate a probe.
  • Text processing: pick the first character safe to reuse as a unique placeholder or delimiter.

2. Find the first repeating character in a string

Approach: Single pass with a set of seen characters. The first character already present in the set is the first repeat. This beats counting because we can stop early.

def first_repeating(s: str) -> str | None:
    """Return the first character that has been seen before, or None."""
    if s is None:
        raise TypeError("Input cannot be None")
    if not isinstance(s, str):
        raise TypeError("Input must be a string")

    seen = set()                     # characters encountered so far
    for ch in s:
        if ch in seen:               # O(1) average membership test
            return ch                # earliest repeat -> return immediately
        seen.add(ch)
    return None


# Time:  O(n) - stops at first repeat
# Space: O(k) - distinct characters seen
# Security relevance: early-exit dedupe detection (e.g., first replayed nonce).

assert first_repeating("abca") == "a"
assert first_repeating("abcdef") is None
assert first_repeating("") is None
assert first_repeating("xxy") == "x"

Practical use cases:

  • Replay detection: catch the first reused nonce/token in an auth stream and stop early.
  • Network capture: identify the first retransmitted or duplicate packet marker.
  • Deduplication: short-circuit when the first repeated key appears in a feed.

3. Check whether two strings are anagrams

Approach: Two strings are anagrams iff they have identical character multisets. A length check short-circuits unequal inputs; Counter equality compares multisets in O(n).

from collections import Counter


def are_anagrams(a: str, b: str) -> bool:
    """True if a and b contain the same characters with the same counts."""
    if a is None or b is None:
        raise TypeError("Inputs cannot be None")
    if not isinstance(a, str) or not isinstance(b, str):
        raise TypeError("Inputs must be strings")

    if len(a) != len(b):             # fast reject: different lengths can't match
        return False
    return Counter(a) == Counter(b)  # multiset comparison


# Time:  O(n)
# Space: O(k) distinct characters
# Security relevance: detecting reordered-but-equivalent payloads / domain look-alikes.

assert are_anagrams("listen", "silent") is True
assert are_anagrams("abc", "abd") is False
assert are_anagrams("", "") is True
assert are_anagrams("a", "aa") is False

Practical use cases:

  • Typosquatting: detect look-alike domains that are character rearrangements of a brand.
  • Malware triage: match reordered-but-equivalent payload or config strings.
  • Data validation: confirm two field sets contain the same elements regardless of order.

4. Group a list of words into anagram groups

Approach: Use a canonical key per word so all anagrams collide into the same bucket. Sorting the characters is the simplest stable key; a 26-length count tuple is O(n) per word if needed.

from collections import defaultdict


def group_anagrams(words: list[str]) -> list[list[str]]:
    """Group words that are anagrams of each other."""
    if words is None:
        raise TypeError("Input cannot be None")

    groups: dict[str, list[str]] = defaultdict(list)
    for w in words:
        if not isinstance(w, str):
            raise TypeError("All items must be strings")
        key = "".join(sorted(w))     # canonical form: sorted characters
        groups[key].append(w)        # words sharing a key are anagrams
    return list(groups.values())


# Time:  O(n * k log k) - n words, k = word length (sorting each word)
# Space: O(n * k) - all words stored in buckets
# Security relevance: clustering equivalent indicators (e.g., permuted strings).

result = group_anagrams(["eat", "tea", "tan", "ate", "nat", "bat"])
assert sorted(sorted(g) for g in result) == [["ate", "eat", "tea"], ["bat"], ["nat", "tan"]]
assert group_anagrams([]) == []

Practical use cases:

  • Threat intel clustering: bucket permuted indicators (domains/strings) under one canonical key.
  • Log normalization: group messages that are reorderings of the same tokens.
  • Dedup analytics: collapse equivalent records into groups for counting.

5. Determine whether a string is a palindrome after normalization

Approach: Normalize (lowercase, keep only alphanumerics) then use two pointers from both ends moving inward. Two pointers avoid building a reversed copy, keeping space O(1) aside from the filtered view.

def is_palindrome(s: str) -> bool:
    """True if s reads the same forwards/backwards ignoring case and non-alphanumerics."""
    if s is None:
        raise TypeError("Input cannot be None")
    if not isinstance(s, str):
        raise TypeError("Input must be a string")

    left, right = 0, len(s) - 1
    while left < right:
        # Skip characters that are not letters/digits.
        while left < right and not s[left].isalnum():
            left += 1
        while left < right and not s[right].isalnum():
            right -= 1
        if s[left].lower() != s[right].lower():
            return False             # mismatch -> not a palindrome
        left += 1
        right -= 1
    return True                      # pointers crossed -> all matched


# Time:  O(n)
# Space: O(1) - in-place two-pointer scan
# Security relevance: input canonicalization before comparison (normalize then check).

assert is_palindrome("A man, a plan, a canal: Panama") is True
assert is_palindrome("race a car") is False
assert is_palindrome("") is True
assert is_palindrome(".,") is True

Practical use cases:

  • Input canonicalization: normalize then compare to defeat formatting-based bypasses.
  • Token validation: verify symmetric or mirror-format identifiers.
  • Data cleaning: ignore punctuation and case when checking structured values.

6. Find the longest substring without repeating characters

Approach: Sliding window with a map of each character's last index. When a duplicate falls inside the window, jump left past it. Each index is visited at most twice, giving O(n).

def longest_unique_substring(s: str) -> int:
    """Length of the longest substring containing no repeated characters."""
    if s is None:
        raise TypeError("Input cannot be None")
    if not isinstance(s, str):
        raise TypeError("Input must be a string")

    last_seen: dict[str, int] = {}   # char -> most recent index
    left = 0                         # window start
    best = 0
    for right, ch in enumerate(s):
        # If ch is inside the current window, shrink from the left.
        if ch in last_seen and last_seen[ch] >= left:
            left = last_seen[ch] + 1
        last_seen[ch] = right
        best = max(best, right - left + 1)
    return best


# Time:  O(n) - each index advances left/right at most once
# Space: O(min(n, charset))
# Security relevance: bounded-window scanning over event streams (no unbounded buffering).

assert longest_unique_substring("abcabcbb") == 3
assert longest_unique_substring("bbbbb") == 1
assert longest_unique_substring("pwwkew") == 3
assert longest_unique_substring("") == 0

Practical use cases:

  • Streaming windows: bound a scan over an event stream without unbounded buffering.
  • Session analysis: find the longest run of distinct actions before a repeat.
  • Rate limiting: measure the longest unique-key span in a sliding window.

7. Find the most frequent element in an array

Approach: Count occurrences in a hash map, then take the key with the maximum count. Counter.most_common(1) does this in one call while we keep the manual logic explicit.

from collections import Counter


def most_frequent(nums: list) -> object:
    """Return the element that appears most often (ties: first to reach the max)."""
    if nums is None:
        raise TypeError("Input cannot be None")
    if len(nums) == 0:
        raise ValueError("Input must contain at least one element")

    counts = Counter(nums)           # O(n) tally
    # most_common(1) returns [(element, count)]; [0][0] is the element.
    return counts.most_common(1)[0][0]


# Time:  O(n)
# Space: O(k) distinct elements
# Security relevance: top talker detection (most frequent IP/user in a log slice).

assert most_frequent([1, 3, 3, 3, 2]) == 3
assert most_frequent(["a", "a", "b"]) == "a"
assert most_frequent([7]) == 7

Practical use cases:

  • Top-talker detection: surface the busiest IP or user in a log slice.
  • Error monitoring: find the most common failing status code.
  • Triage: identify the dominant indicator in an alert batch.

8. Find the top K frequent elements

Approach: Count frequencies, then use a heap of size K to keep the K largest. A bounded min-heap is O(n log k), better than fully sorting when k << n.

import heapq
from collections import Counter


def top_k_frequent(nums: list, k: int) -> list:
    """Return the k most frequent elements (order not guaranteed)."""
    if nums is None:
        raise TypeError("Input cannot be None")
    if not isinstance(k, int):
        raise TypeError("k must be an integer")
    if k <= 0:
        raise ValueError("k must be positive")

    counts = Counter(nums)
    if k >= len(counts):             # boundary: asking for everything
        return list(counts.keys())

    # nlargest keeps a bounded heap of size k -> O(n log k).
    return heapq.nlargest(k, counts.keys(), key=counts.get)


# Time:  O(n log k)
# Space: O(n) for the counter
# Security relevance: surfacing the noisiest k sources for triage/alerting.

assert sorted(top_k_frequent([1, 1, 1, 2, 2, 3], 2)) == [1, 2]
assert top_k_frequent([1], 1) == [1]
assert sorted(top_k_frequent([4, 4, 5, 5], 5)) == [4, 5]

Practical use cases:

  • Alert triage: surface the noisiest K sources for investigation.
  • Capacity: list the top K hottest endpoints or queries.
  • Threat hunting: rank the top K user-agents or destinations.

9. Remove duplicates from a sorted array

Approach: Two pointers in place. Because the array is sorted, duplicates are adjacent; a slow pointer marks the write position while a fast pointer scans. Returns the new logical length.

def remove_duplicates(nums: list) -> int:
    """Remove duplicates in-place from a sorted list; return new length.

    The first `length` elements of nums hold the unique values afterward.
    """
    if nums is None:
        raise TypeError("Input cannot be None")
    if len(nums) == 0:
        return 0                     # boundary: nothing to do

    write = 1                        # next position to write a unique value
    for read in range(1, len(nums)):
        if nums[read] != nums[write - 1]:   # new unique value found
            nums[write] = nums[read]
            write += 1
    return write


# Time:  O(n)
# Space: O(1) - in-place
# Security relevance: compacting sorted, deduplicated indicator lists with no extra memory.

data = [1, 1, 2, 3, 3, 3, 4]
n = remove_duplicates(data)
assert n == 4 and data[:n] == [1, 2, 3, 4]
assert remove_duplicates([]) == 0
assert remove_duplicates([5]) == 1

Practical use cases:

  • Threat feeds: compact a sorted, deduplicated indicator list in place.
  • Time series: collapse repeated sorted timestamps.
  • Allowlists: clean a sorted set of approved values with no extra memory.

10. Merge two sorted arrays

Approach: Classic merge step from merge sort. Two pointers walk both inputs, always appending the smaller front element. Leftovers are appended at the end. O(n + m) and stable.

def merge_sorted(a: list, b: list) -> list:
    """Merge two ascending-sorted lists into one ascending-sorted list."""
    if a is None or b is None:
        raise TypeError("Inputs cannot be None")

    merged = []
    i = j = 0
    # Walk both lists, taking the smaller head each step.
    while i < len(a) and j < len(b):
        if a[i] <= b[j]:             # <= keeps the merge stable
            merged.append(a[i])
            i += 1
        else:
            merged.append(b[j])
            j += 1
    # One list is exhausted; append the remaining sorted tail of the other.
    merged.extend(a[i:])
    merged.extend(b[j:])
    return merged


# Time:  O(n + m)
# Space: O(n + m) for the output
# Security relevance: merging time-sorted event streams from two sources.

assert merge_sorted([1, 3, 5], [2, 4, 6]) == [1, 2, 3, 4, 5, 6]
assert merge_sorted([], [1, 2]) == [1, 2]
assert merge_sorted([1], []) == [1]
assert merge_sorted([], []) == []

Practical use cases:

  • SIEM correlation: merge two time-sorted event streams into one timeline.
  • Log aggregation: combine pre-sorted log files.
  • Range handling: merge sorted IP or port lists before further processing.

11. Merge overlapping intervals

Approach: Sort intervals by start. Then sweep once: if the current interval overlaps the last merged interval (its start <= last end), extend the end; otherwise start a new interval. Sorting dominates the cost.

def merge_intervals(intervals: list[list[int]]) -> list[list[int]]:
    """Merge all overlapping [start, end] intervals."""
    if intervals is None:
        raise TypeError("Input cannot be None")
    if len(intervals) == 0:
        return []

    # Sort by start so overlaps are always adjacent.
    intervals = sorted(intervals, key=lambda iv: iv[0])
    merged = [list(intervals[0])]
    for start, end in intervals[1:]:
        if start > end:              # guard malformed interval
            raise ValueError(f"Invalid interval: start {start} > end {end}")
        last = merged[-1]
        if start <= last[1]:         # overlap -> extend the end
            last[1] = max(last[1], end)
        else:                        # disjoint -> push a new interval
            merged.append([start, end])
    return merged


# Time:  O(n log n) - dominated by the sort
# Space: O(n) - output (O(1) extra beyond it)
# Security relevance: collapsing overlapping time windows of activity into incidents.

assert merge_intervals([[1, 3], [2, 6], [8, 10], [15, 18]]) == [[1, 6], [8, 10], [15, 18]]
assert merge_intervals([[1, 4], [4, 5]]) == [[1, 5]]
assert merge_intervals([]) == []

Practical use cases:

  • Incident building: collapse overlapping activity windows into single incidents.
  • Firewall/ACL: merge overlapping IP or port ranges into compact rules.
  • Scheduling: consolidate overlapping maintenance or access windows.

12. Insert a new interval into a list of intervals

Approach: Assume the existing list is sorted and disjoint. Three phases: copy intervals ending before the new one, merge all that overlap the new one, then copy the rest. Single pass, O(n).

def insert_interval(intervals: list[list[int]], new: list[int]) -> list[list[int]]:
    """Insert `new` into sorted, non-overlapping `intervals`, merging as needed."""
    if intervals is None or new is None:
        raise TypeError("Inputs cannot be None")
    if new[0] > new[1]:
        raise ValueError("new interval has start > end")

    result = []
    i, n = 0, len(intervals)
    start, end = new[0], new[1]

    # Phase 1: intervals strictly before `new` (no overlap).
    while i < n and intervals[i][1] < start:
        result.append(intervals[i])
        i += 1
    # Phase 2: absorb every interval that overlaps `new`.
    while i < n and intervals[i][0] <= end:
        start = min(start, intervals[i][0])
        end = max(end, intervals[i][1])
        i += 1
    result.append([start, end])
    # Phase 3: intervals strictly after `new`.
    while i < n:
        result.append(intervals[i])
        i += 1
    return result


# Time:  O(n) - single pass over a pre-sorted list
# Space: O(n) - output
# Security relevance: adding a new observed window to an existing timeline.

assert insert_interval([[1, 3], [6, 9]], [2, 5]) == [[1, 5], [6, 9]]
assert insert_interval([[1, 2], [3, 5], [6, 7], [8, 10], [12, 16]], [4, 8]) == [[1, 2], [3, 10], [12, 16]]
assert insert_interval([], [5, 7]) == [[5, 7]]

Practical use cases:

  • Timeline updates: add a newly observed activity window and merge overlaps.
  • Rule management: insert a new allowed range into an existing rule set.
  • Booking: place a new reservation, merging adjacent slots.

13. Find whether two intervals overlap

Approach: Two closed intervals overlap iff each starts no later than the other ends: a.start <= b.end and b.start <= a.end. Constant time, no sorting.

def intervals_overlap(a: list[int], b: list[int]) -> bool:
    """True if closed intervals a=[s,e] and b=[s,e] share any point."""
    if a is None or b is None:
        raise TypeError("Inputs cannot be None")
    if a[0] > a[1] or b[0] > b[1]:
        raise ValueError("interval start must be <= end")

    # Overlap exists unless one interval is entirely before the other.
    return a[0] <= b[1] and b[0] <= a[1]


# Time:  O(1)
# Space: O(1)
# Security relevance: checking if two access/session windows intersect.

14. Given meeting intervals, determine if a person can attend all meetings

Approach: Sort by start time; if any meeting begins before the previous one ends, there is a conflict. One linear scan after sorting.

def can_attend_all(meetings: list[list[int]]) -> bool:
    """True if no two meetings overlap (person can attend every meeting)."""
    if meetings is None:
        raise TypeError("Input cannot be None")
    if len(meetings) <= 1:
        return True                  # 0 or 1 meeting never conflicts

    meetings = sorted(meetings, key=lambda m: m[0])
    for prev, curr in zip(meetings, meetings[1:]):
        if curr[0] < prev[1]:        # starts before previous ends -> conflict
            return False
    return True


# Time:  O(n log n)
# Space: O(n) for the sorted copy
# Security relevance: detecting overlapping privileged sessions for one identity.

assert can_attend_all([[0, 30], [5, 10], [15, 20]]) is False
assert can_attend_all([[7, 10], [2, 4]]) is True
assert can_attend_all([]) is True

Practical use cases:

  • Identity abuse: flag overlapping privileged sessions for one account (impossible concurrent use).
  • Resource scheduling: confirm a single resource has no conflicting bookings.
  • Licensing: verify seat usage never double-books.

15. Given meeting intervals, find the minimum number of rooms required

Approach: Separate and sort start and end times. Sweep through starts; each start needs a room unless an earlier meeting has already ended (advance the end pointer). The peak concurrency is the answer. Equivalent to a min-heap of end times.

def min_meeting_rooms(meetings: list[list[int]]) -> int:
    """Minimum number of rooms so no two overlapping meetings share a room."""
    if meetings is None:
        raise TypeError("Input cannot be None")
    if len(meetings) == 0:
        return 0

    starts = sorted(m[0] for m in meetings)
    ends = sorted(m[1] for m in meetings)

    rooms = 0
    max_rooms = 0
    e = 0                            # pointer into ends
    for s in starts:
        # If the next meeting starts after some meeting ended, free that room.
        if s >= ends[e]:
            e += 1
        else:
            rooms += 1               # need a new concurrent room
            max_rooms = max(max_rooms, rooms)
    return max_rooms


# Time:  O(n log n)
# Space: O(n)
# Security relevance: peak concurrency of sessions/connections (capacity & anomaly baseline).

assert min_meeting_rooms([[0, 30], [5, 10], [15, 20]]) == 2
assert min_meeting_rooms([[7, 10], [2, 4]]) == 1
assert min_meeting_rooms([]) == 0

Practical use cases:

  • Capacity baseline: compute peak concurrent sessions/connections for anomaly thresholds.
  • DDoS signal: track the maximum number of simultaneous connections.
  • Infra planning: size a pool to the observed peak concurrency.

16. Implement binary search

Approach: Classic halving search on a sorted array. Use a half-open invariant and compute the midpoint with low + (high - low) // 2 to avoid overflow in fixed-width languages (harmless but disciplined in Python).

def binary_search(nums: list[int], target: int) -> int:
    """Return the index of target in sorted nums, or -1 if absent."""
    if nums is None:
        raise TypeError("Input cannot be None")

    low, high = 0, len(nums) - 1
    while low <= high:               # inclusive bounds
        mid = low + (high - low) // 2     # overflow-safe midpoint
        if nums[mid] == target:
            return mid
        if nums[mid] < target:
            low = mid + 1            # discard left half
        else:
            high = mid - 1           # discard right half
    return -1                        # not found


# Time:  O(log n)
# Space: O(1)
# Security relevance: fast membership in large sorted allow/deny lists.

assert binary_search([1, 3, 5, 7, 9], 7) == 3
assert binary_search([1, 3, 5], 4) == -1
assert binary_search([], 1) == -1
assert binary_search([2], 2) == 0

Practical use cases:

  • Allow/deny lookups: fast membership in a large sorted list.
  • Threat intel: probe a sorted indicator set in O(log n).
  • Config: locate a value or boundary in sorted data.

17. Find the first and last position of a target in a sorted array

Approach: Two binary searches: one biased left to find the first index, one biased right to find the last. Each is O(log n), so the whole thing stays logarithmic even with many duplicates.

def search_range(nums: list[int], target: int) -> list[int]:
    """Return [first_index, last_index] of target, or [-1, -1] if absent."""
    if nums is None:
        raise TypeError("Input cannot be None")

    def bound(is_first: bool) -> int:
        low, high, found = 0, len(nums) - 1, -1
        while low <= high:
            mid = low + (high - low) // 2
            if nums[mid] == target:
                found = mid          # record candidate, keep searching one side
                if is_first:
                    high = mid - 1   # look further left for the first occurrence
                else:
                    low = mid + 1    # look further right for the last occurrence
            elif nums[mid] < target:
                low = mid + 1
            else:
                high = mid - 1
        return found

    return [bound(True), bound(False)]


# Time:  O(log n)
# Space: O(1)
# Security relevance: range queries over time-sorted logs (all events at timestamp T).

assert search_range([5, 7, 7, 8, 8, 10], 8) == [3, 4]
assert search_range([5, 7, 7, 8, 8, 10], 6) == [-1, -1]
assert search_range([], 0) == [-1, -1]

Practical use cases:

  • Time-sorted logs: get the full index range of all events at a timestamp.
  • Pagination: find the span of records matching a sorted key.
  • Range counts: derive the count of a value from its first/last index.

18. Find the minimum element in a rotated sorted array

Approach: Modified binary search. Compare mid with the rightmost element: if nums[mid] > nums[high], the minimum is to the right; otherwise it is at mid or to the left. Converges in O(log n) assuming distinct values.

def find_min_rotated(nums: list[int]) -> int:
    """Minimum of an ascending array rotated at an unknown pivot (distinct values)."""
    if nums is None:
        raise TypeError("Input cannot be None")
    if len(nums) == 0:
        raise ValueError("Input must contain at least one element")

    low, high = 0, len(nums) - 1
    while low < high:
        mid = low + (high - low) // 2
        if nums[mid] > nums[high]:   # min must be to the right of mid
            low = mid + 1
        else:                        # min is at mid or to its left
            high = mid
    return nums[low]                 # low == high points at the minimum


# Time:  O(log n)
# Space: O(1)
# Security relevance: efficient search over wrapped/rotated ordered data.

assert find_min_rotated([3, 4, 5, 1, 2]) == 1
assert find_min_rotated([4, 5, 6, 7, 0, 1, 2]) == 0
assert find_min_rotated([1]) == 1
assert find_min_rotated([2, 1]) == 1

Practical use cases:

  • Ring buffers: find the oldest entry or rotation point in a circular log.
  • Sharded indexes: locate the minimum in a rotated ordered partition.
  • Recovery: detect where a wrapped sequence restarts.

19. Search in a rotated sorted array

Approach: Binary search where, at each step, exactly one half is normally sorted. Detect the sorted half, check whether the target lies within it, and discard the other half accordingly.

def search_rotated(nums: list[int], target: int) -> int:
    """Index of target in a rotated sorted array (distinct values), else -1."""
    if nums is None:
        raise TypeError("Input cannot be None")

    low, high = 0, len(nums) - 1
    while low <= high:
        mid = low + (high - low) // 2
        if nums[mid] == target:
            return mid
        if nums[low] <= nums[mid]:           # left half [low..mid] is sorted
            if nums[low] <= target < nums[mid]:
                high = mid - 1               # target in sorted left half
            else:
                low = mid + 1
        else:                                 # right half [mid..high] is sorted
            if nums[mid] < target <= nums[high]:
                low = mid + 1                # target in sorted right half
            else:
                high = mid - 1
    return -1


# Time:  O(log n)
# Space: O(1)
# Security relevance: lookups in rotated/sharded ordered indexes.

assert search_rotated([4, 5, 6, 7, 0, 1, 2], 3) == -1
assert search_rotated([1], 1) == 0
assert search_rotated([], 5) == -1

Practical use cases:

  • Circular logs: look up a value in a rotated ring buffer in O(log n).
  • Sharded data: search a rotated ordered index without unrotating it.
  • Recovery: find an entry across a wrap boundary.

21. Find three numbers that sum to zero

Approach: Sort, then fix each number and use two pointers on the remainder. Sorting enables both the two-pointer sweep and easy duplicate-skipping so the result has unique triplets. O(n^2) overall.

def three_sum(nums: list[int]) -> list[list[int]]:
    """Return all unique triplets [a, b, c] with a + b + c == 0."""
    if nums is None:
        raise TypeError("Input cannot be None")

    nums = sorted(nums)              # enables two-pointer + dup skipping
    result = []
    n = len(nums)
    for i in range(n - 2):
        if nums[i] > 0:              # smallest is positive -> no zero sum possible
            break
        if i > 0 and nums[i] == nums[i - 1]:
            continue                 # skip duplicate anchor values
        left, right = i + 1, n - 1
        while left < right:
            total = nums[i] + nums[left] + nums[right]
            if total < 0:
                left += 1            # need a larger sum
            elif total > 0:
                right -= 1           # need a smaller sum
            else:
                result.append([nums[i], nums[left], nums[right]])
                left += 1
                right -= 1
                # Skip duplicates for the second/third elements.
                while left < right and nums[left] == nums[left - 1]:
                    left += 1
                while left < right and nums[right] == nums[right + 1]:
                    right -= 1
    return result


# Time:  O(n^2)
# Space: O(n) for sorting (output excluded)
# Security relevance: finding combinations of values that cancel (e.g., balancing anomalies).

22. Find the maximum subarray sum

Approach: Kadane's algorithm. Track the best sum ending at the current index: either extend the previous subarray or restart at the current element. A single pass, O(1) extra space.

def max_subarray(nums: list[int]) -> int:
    """Largest sum of any contiguous non-empty subarray (Kadane's algorithm)."""
    if nums is None:
        raise TypeError("Input cannot be None")
    if len(nums) == 0:
        raise ValueError("Input must contain at least one element")

    best = current = nums[0]
    for num in nums[1:]:
        # Either extend the running subarray or start fresh at num.
        current = max(num, current + num)

    return best

def product_except_self(nums: list[int]) -> list[int]:

def is_balanced(s: str) -> bool:
    """True if every (), [], {} is correctly opened and closed in order."""
    if s is None:

26. Evaluate a simple postfix expression

def eval_postfix(tokens: list[str]) -> int:
    """Evaluate a Reverse Polish Notation expression (integer arithmetic)."""
    if tokens is None:
        raise TypeError("Input cannot be None")

    ops = {"+", "-", "*", "/"}
    stack = []
    for tok in tokens:
        if tok in ops:
            if len(stack) < 2:
                raise ValueError("Malformed expression: not enough operands")
            b = stack.pop()
            a = stack.pop()
            if tok == "+":
                stack.append(a + b)
            elif tok == "-":
                stack.append(a - b)
            elif tok == "*":
                stack.append(a * b)
            else:
                if b == 0:
                    raise ZeroDivisionError("Division by zero in expression")
                stack.append(int(a / b))   # truncate toward zero
        else:
            stack.append(int(tok))   # operand
    if len(stack) != 1:
        raise ValueError("Malformed expression: leftover operands")
    return stack[0]

# Space: O(n)

28. Implement a queue using two stacks

Approach: An in stack receives pushes; an out stack serves pops. When out is empty, pour in into out, reversing order to achieve FIFO. Amortized O(1) per operation because each element moves at most once.

class QueueWithStacks:
    """FIFO queue backed by two LIFO stacks (amortized O(1) operations)."""

    def __init__(self):
        self._in = []                # incoming pushes
        self._out = []               # outgoing pops (reversed order)

    def push(self, x) -> None:
        self._in.append(x)           # always O(1)

    def _transfer(self) -> None:
        # Move everything from _in to _out, reversing to FIFO order.
        while self._in:
            self._out.append(self._in.pop())

    def pop(self):
        if not self._in and not self._out:
            raise IndexError("pop from empty queue")
        if not self._out:
            self._transfer()
        return self._out.pop()

    def peek(self):
        if not self._in and not self._out:
            raise IndexError("peek from empty queue")
        if not self._out:
            self._transfer()
        return self._out[-1]

    def empty(self) -> bool:
        return not self._in and not self._out


# Time:  amortized O(1) per push/pop
# Space: O(n)
# Security relevance: FIFO processing of event/work queues with simple primitives.

30. Implement an LRU cache

from collections import OrderedDict


class LRUCache:
    """Fixed-capacity cache evicting the least recently used key. O(1) get/put."""

    def __init__(self, capacity: int):
        if not isinstance(capacity, int) or capacity <= 0:
            raise ValueError("capacity must be a positive integer")
        self._capacity = capacity
        self._store: OrderedDict = OrderedDict()

    def get(self, key) -> object:
        if key not in self._store:
            return -1                # sentinel for "missing"
        self._store.move_to_end(key)  # mark as most recently used
        return self._store[key]

    def put(self, key, value) -> None:
        if key in self._store:
            self._store.move_to_end(key)   # refresh recency
        self._store[key] = value
        if len(self._store) > self._capacity:
            # Evict the least recently used item (front of the order).
            self._store.popitem(last=False)


# Time:  O(1) get and put
# Space: O(capacity)
# Security relevance: bounded caches for tokens/decisions without unbounded memory growth.

assert cache.get(3) == 3

31. Implement a min stack

class MinStack:
    """Stack that also returns the current minimum in O(1)."""

    def __init__(self):
        self._stack = []             # actual values
        self._mins = []              # running minimum at each level

    def push(self, x) -> None:
        self._stack.append(x)
        # Store the smaller of x and the previous minimum.
        current_min = x if not self._mins else min(x, self._mins[-1])

        return self._stack.pop()

            raise IndexError("top from empty stack")
        return self._stack[-1]

    def get_min(self):
        if not self._mins:
            raise IndexError("get_min from empty stack")
        return self._mins[-1]

36. Implement breadth-first search on a graph

Approach: BFS uses a FIFO queue and a visited set to explore level by level. deque.popleft() gives O(1) dequeues. Returns the visitation order.

from collections import deque


def bfs(graph: dict, start) -> list:
    """Breadth-first traversal order from `start` over an adjacency-list graph."""
    if graph is None:
        raise TypeError("graph cannot be None")
    if start not in graph:
        raise ValueError("start node not in graph")

    visited = {start}                # mark as discovered when enqueued
    queue = deque([start])
    order = []
    while queue:
        node = queue.popleft()       # FIFO -> level-order
        order.append(node)
        for neighbor in graph.get(node, []):
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return order


# Time:  O(V + E)
# Space: O(V)
# Security relevance: shortest-hop blast-radius exploration from a compromised node.

    if graph is None:

    visited = set()
    stack = [start]

        node = stack.pop()           # LIFO -> go deep first
        if node in visited:
            continue                 # skip already-expanded nodes
        visited.add(node)
        order.append(node)
        # Push neighbors; reversed() makes traversal follow listing order.
        for neighbor in reversed(graph.get(node, [])):
            if neighbor not in visited:
                stack.append(neighbor)
    return order


# Time:  O(V + E)
# Space: O(V)
# Security relevance: reachability / dependency walking through a graph of resources.

g = {"a": ["b", "c"], "b": ["d"], "c": [], "d": []}
assert dfs(g, "a") == ["a", "b", "d", "c"]
assert dfs({"x": []}, "x") == ["x"]

Practical use cases:

  • Dependency walk: traverse resource or permission graphs to completion.
  • Reachability: find all nodes reachable from a compromise point.
  • Analysis: drive topological or connectivity checks.

38. Determine whether a graph has a cycle

Approach: For a directed graph, DFS with three colors (white/gray/black). A back-edge to a gray (in-progress) node means a cycle. Coloring distinguishes "still on the recursion stack" from "fully processed."

def has_cycle_directed(graph: dict) -> bool:
    """True if the directed graph contains a cycle (DFS coloring)."""
    if graph is None:
        raise TypeError("graph cannot be None")

    WHITE, GRAY, BLACK = 0, 1, 2
    color = {node: WHITE for node in graph}

    def visit(node) -> bool:
        color[node] = GRAY           # node is on the current DFS path
        for neighbor in graph.get(node, []):
            if color.get(neighbor, WHITE) == GRAY:
                return True          # back-edge -> cycle
            if color.get(neighbor, WHITE) == WHITE and visit(neighbor):
                return True
        color[node] = BLACK          # done exploring this node
        return False

    return any(color[node] == WHITE and visit(node) for node in graph)


# Time:  O(V + E)
# Space: O(V)
# Security relevance: detecting circular trust/role dependencies that break assumptions.

assert has_cycle_directed({"a": ["b"], "b": ["c"], "c": ["a"]}) is True
assert has_cycle_directed({"a": ["b"], "b": ["c"], "c": []}) is False
assert has_cycle_directed({}) is False

Practical use cases:

  • IAM safety: detect circular role or trust dependencies (privilege loops).
  • Build systems: catch dependency cycles before scheduling.
  • Config validation: reject cyclic references.

39. Find the shortest path in an unweighted graph

Approach: BFS finds shortest paths in unweighted graphs because it explores in order of hop count. Track each node's parent to reconstruct the path; the first time we reach the target is optimal.

from collections import deque


def shortest_path(graph: dict, start, goal) -> list | None:
    """Shortest hop path from start to goal, or None if unreachable."""
    if graph is None:
        raise TypeError("graph cannot be None")
    if start not in graph:
        raise ValueError("start node not in graph")
    if start == goal:
        return [start]

    parents = {start: None}          # child -> parent for reconstruction
    queue = deque([start])
    while queue:
        node = queue.popleft()
        for neighbor in graph.get(node, []):
            if neighbor not in parents:
                parents[neighbor] = node
                if neighbor == goal:        # first arrival is shortest
                    path = [goal]
                    while parents[path[-1]] is not None:
                        path.append(parents[path[-1]])
                    return path[::-1]       # reverse to start->goal
                queue.append(neighbor)
    return None                      # goal not reachable


# Time:  O(V + E)
# Space: O(V)
# Security relevance: fewest-hop attack path between two nodes in a graph.

g = {"a": ["b", "c"], "b": ["d"], "c": ["d"], "d": ["e"], "e": []}
assert shortest_path(g, "a", "e") == ["a", "b", "d", "e"]
assert shortest_path(g, "a", "a") == ["a"]
assert shortest_path({"a": []}, "a", "z") is None

Practical use cases:

  • Attack paths: find the fewest-hop route between two nodes (lateral movement).
  • Networking: compute shortest unweighted routes.
  • Dependency: minimal chain between two resources.

40. Parse a CSV-like line with quoted fields

Approach: A small state machine handling quoted fields and escaped quotes (""). A naive split(",") breaks when commas appear inside quotes, so we scan character by character tracking whether we are inside quotes.

def parse_csv_line(line: str) -> list[str]:
    """Split one CSV line, respecting quoted fields and doubled-quote escapes."""
    if line is None:
        raise TypeError("line cannot be None")

    fields = []
    field = []
    in_quotes = False
    i = 0
    while i < len(line):
        ch = line[i]
        if in_quotes:
            if ch == '"':
                # A doubled quote inside quotes is a literal quote.
                if i + 1 < len(line) and line[i + 1] == '"':
                    field.append('"')
                    i += 1
                else:
                    in_quotes = False     # closing quote
            else:
                field.append(ch)
        else:
            if ch == '"':
                in_quotes = True          # opening quote
            elif ch == ",":
                fields.append("".join(field))   # field boundary
                field = []
            else:
                field.append(ch)
        i += 1
    if in_quotes:
        raise ValueError("Unterminated quoted field")
    fields.append("".join(field))         # final field
    return fields


# Time:  O(n)
# Space: O(n)
# Security relevance: robust log/CSV parsing resistant to delimiter-injection tricks.

assert parse_csv_line('alice,10,"city, state"') == ["alice", "10", "city, state"]
assert parse_csv_line('a,"he said ""hi""",b') == ["a", 'he said "hi"', "b"]
assert parse_csv_line("") == [""]

Practical use cases:

  • Robust ingestion: parse logs or exports resistant to delimiter-injection.
  • Data import: handle quoted fields with embedded commas or quotes.
  • ETL: safely tokenize CSV at a trust boundary.

41. Parse key-value pairs from a log line

def parse_kv(line: str) -> dict[str, str]:
    """Parse `key=value` tokens from a log line into a dict."""
    if line is None:

42. Sort log entries by timestamp

Approach: Parse timestamps to comparable datetimes and sort with a key function. ISO-8601 UTC strings also sort correctly lexicographically, but parsing is explicit and robust to mixed formats. Python's sort is stable (Timsort), preserving order of equal timestamps.

from datetime import datetime


def sort_by_timestamp(entries: list[dict]) -> list[dict]:
    """Return entries sorted ascending by their ISO-8601 'ts' field."""
    if entries is None:
        raise TypeError("entries cannot be None")

    def parse_ts(entry: dict) -> datetime:
        ts = entry.get("ts")
        if ts is None:
            raise ValueError("entry missing 'ts' field")
        # fromisoformat handles 'YYYY-MM-DDTHH:MM:SS+00:00'; normalize trailing Z.
        return datetime.fromisoformat(ts.replace("Z", "+00:00"))

    return sorted(entries, key=parse_ts)   # stable O(n log n) sort


# Time:  O(n log n)
# Space: O(n)
# Security relevance: building accurate incident timelines from unordered logs.

def find_duplicates(items: list) -> list:

    return list(duplicates)

46. Design a memory-efficient way to count frequencies

Approach: For exact counts, Counter (a hash map) is O(n) time and O(k) space in the number of distinct keys. When distinct keys are too many to store, use Count-Min Sketch: fixed-size hash tables giving approximate counts with bounded overestimate error.

import hashlib


class CountMinSketch:
    """Approximate frequency counter with fixed memory (overestimates only)."""

    def __init__(self, width: int = 1000, depth: int = 5):
        if width <= 0 or depth <= 0:
            raise ValueError("width and depth must be positive")
        self.width = width
        self.depth = depth
        # depth independent rows, each of `width` counters.
        self.table = [[0] * width for _ in range(depth)]

    def _hashes(self, key: str):
        # Derive `depth` independent bucket indices from the key.
        for i in range(self.depth):
            h = hashlib.md5(f"{i}:{key}".encode()).hexdigest()
            yield int(h, 16) % self.width

    def add(self, key: str, count: int = 1) -> None:
        for row, idx in enumerate(self._hashes(key)):
            self.table[row][idx] += count

    def estimate(self, key: str) -> int:
        # The minimum across rows reduces collision overestimation.
        return min(self.table[row][idx] for row, idx in enumerate(self._hashes(key)))


# Exact:  Counter -> O(n) time, O(k) space (k distinct keys)
# Sketch: O(depth) per update/query, O(width * depth) fixed space, slight overcount
# Security relevance: heavy-hitter detection over high-cardinality streams (IPs/domains)
#                     within a fixed memory budget.

cms = CountMinSketch(width=100, depth=4)
for _ in range(50):
    cms.add("1.2.3.4")
cms.add("9.9.9.9")
assert cms.estimate("1.2.3.4") >= 50      # may overestimate, never underestimates
assert cms.estimate("9.9.9.9") >= 1

Practical use cases:

  • Heavy hitters: track top IPs or domains over high-cardinality streams in fixed memory.
  • DDoS: approximate per-source counts at line rate.
  • Telemetry: bounded-memory frequency estimates for huge key spaces.

47. Explain time complexity for hash map lookup and collision cases

This is a conceptual question; state the model clearly.

  • Average case — O(1): A good hash function spreads keys uniformly across buckets, so lookups, inserts, and deletes touch only a short bucket and amortize to constant time.
  • Collisions: When two keys hash to the same bucket, they are chained (linked list / dynamic array) or placed by open addressing (probing). A few collisions are fine; lookup cost is proportional to the bucket's length.
  • Worst case — O(n): If every key collides into one bucket (a degenerate or adversarially crafted hash), the structure degrades to a linear scan.
  • Resizing: Maps grow when the load factor crosses a threshold and rehash all keys — an O(n) operation, but amortized O(1) across many inserts.
  • Security angle — algorithmic complexity (HashDoS): An attacker who can predict the hash function may send keys that all collide, forcing O(n) per operation and O(n^2) overall to cause a denial of service. Mitigations: randomized/seeded hashing (SipHash), per-process hash seeds, capping inputs, or switching to balanced trees (O(log n) guaranteed) for untrusted keys.
# Illustration: worst-case chaining degrades lookups to a linear scan.
# average_lookup -> O(1); adversarial_all_collisions -> O(n) per lookup.
def lookup_cost_model(num_keys: int, max_bucket_len: int) -> str:
    if max_bucket_len <= 1:
        return "O(1) average"
    if max_bucket_len >= num_keys:
        return "O(n) worst case (all collisions)"
    return "O(bucket length)"

assert lookup_cost_model(1000, 1) == "O(1) average"
assert lookup_cost_model(1000, 1000) == "O(n) worst case (all collisions)"

Practical use cases:

  • HashDoS defense: justify randomized or seeded hashing for untrusted keys.
  • Capacity planning: reason about load factor and resize cost.
  • Structure choice: pick balanced trees over maps for adversarial input.

48. Explain the tradeoff between sorting and hashing for duplicate detection

Conceptual comparison; both detect duplicates but with different profiles.

Aspect Hashing (set / dict) Sorting then scan
Time O(n) average O(n log n)
Space O(n) extra for the set O(1) if sorted in place
Order of output Unordered Naturally sorted
Streaming Yes — detect on the fly No — needs the full dataset first
Worst case O(n) per op under HashDoS O(n log n) guaranteed
Memory-bound data Needs external/approx structures External merge-sort fits disk nicely
  • Choose hashing when you want the fastest average time, can spend O(n) memory, and may need early/streaming detection.
  • Choose sorting when memory is tight, you need sorted output anyway, you want predictable worst-case behavior, or the data must be processed externally (on disk).
  • Security angle: for untrusted input, hashing risks algorithmic-complexity attacks unless the hash is randomized; sorting offers a stable O(n log n) guarantee regardless of input crafting.
# Both return the same duplicate set; the choice is about time/space/order tradeoffs.
def duplicates_hashing(items):
    seen, dups = set(), set()
    for x in items:
        (dups if x in seen else seen).add(x)
    return dups

def duplicates_sorting(items):
    s = sorted(items)               # O(n log n), O(1) extra if in place
    return {s[i] for i in range(1, len(s)) if s[i] == s[i - 1]}

assert duplicates_hashing([1, 1, 2, 3, 3]) == {1, 3}
assert duplicates_sorting([1, 1, 2, 3, 3]) == {1, 3}

Practical use cases:

  • Design decisions: choose hashing versus sorting for dedup at scale.
  • Memory limits: pick sort-on-disk when data exceeds RAM.
  • Streaming: select hashing for on-the-fly detection.
