How to Use These Notes

How to Study These Topics (from security-engineering-study-notes.md)

  • Questions are intentionally vague — ask clarifying questions first
  • Test yourself on both depth (keep going until you hit the limit) and breadth (across domains)
  • Say everything you are thinking — practice articulating your reasoning aloud
  • Always repeat the question back to buy thinking time and confirm understanding
  • State assumptions explicitly — "I am assuming logs are available for 90 days..."

Framing to Emphasize

Every design answer should demonstrate:

  1. Data protection mindset — classify data, enforce at boundaries, detect exfiltration
  2. Scale thinking — billions of events/day, millions of users, fleet-wide policy enforcement
  3. BeyondCorp awareness — device trust, context-aware access, no implicit network trust
  4. Python/automation instinct — "I would automate this check with a Python pipeline..."
  5. Least-privilege by default — every service account, role, and API key scoped minimally
  6. Auditability — every action logged, tamper-evident, immutable

Quick Rehearsal Questions (Highest-Value)

  1. "Design a DLP system that prevents employees from exfiltrating IP to personal cloud storage."
  2. "Threat-model an OAuth2 authorization server from scratch."
  3. "An EC2/GCE instance is compromised — walk me through blast radius and containment."
  4. "Design a tamper-evident audit logging pipeline for a 200-account cloud org."
  5. "Explain BeyondCorp — how does it change the security model vs VPN?"
  6. "Design an IAM governance system for 50,000 employees and 1,000 GCP projects."
  7. "How would you detect a malicious insider exfiltrating data via DNS tunneling?"
  8. "Walk me through a TLS 1.3 handshake and what breaks if cert validation is skipped."
  9. "Design secrets management for 500 microservices — no hardcoded credentials."
  10. "How would you enforce data residency requirements for PII in a multi-region GCP deployment?"
