18-Hour Security Engineering Rapid Review

Focus: broad security engineering review
Duration: 18 focused hours
Goal: Maximize recall across core topics, not read every note. Every hour ends with active recall.

Ground Rules

  • Use a 45-10-5 rhythm: 45 minutes study, 10 minutes answer aloud, 5 minutes log gaps.
  • Do not passively read for a full hour. Convert every topic into a spoken answer.
  • Keep one gap log. Any concept you cannot explain in 90 seconds goes there.
  • For each design question, use this structure: clarify, assets/data, trust boundaries, threats, controls, detection, tradeoffs.
  • If you fall behind, protect auth, tokens, DLP, cloud IAM, threat modeling, and coding.

Must-Master Answers

By the end of the 18 hours, you should be able to answer these without notes:

  1. OAuth2 authorization code + PKCE, and where people implement it badly.
  2. OIDC vs OAuth2 vs SAML, and what each token/assertion proves.
  3. JWT validation failure modes: alg:none, key confusion, expiry, audience, issuer, revocation.
  4. RBAC vs ABAC vs ReBAC, including when Zanzibar-style relationships help.
  5. DLP pipeline design: classify, inspect, decide, enforce, alert, tune, audit.
  6. TLS handshake and certificate validation, including what forward secrecy means.
  7. SSRF to cloud metadata credential theft and how to prevent it.
  8. Cloud IAM blast radius analysis for a compromised workload.
  9. STRIDE threat model for a login, API gateway, or data export flow.
  10. Incident response for suspicious auth, DNS exfiltration, or exposed secrets.
  11. A clean coding approach: clarify inputs, edge cases, complexity, implementation, tests.
  12. Reflection: a hard security tradeoff, incident ownership, disagreement, and learning loop.

Rescue Plan If Time Slips

If you lose 3-4 hours, skip deep OS/mobile/malware details and keep:

  1. Hours 2-5: auth, tokens, authorization, DLP.
  2. Hours 8-12: web/cloud/threat modeling/system design.
  3. Hours 15-17: coding and timed self-test.
  4. Hour 18: final compression.

Final 20-Minute Warmup

  • Say your topic summary once.
  • Rehearse the design answer frame once.
  • Rehearse the coding answer frame once.
  • Review only the gap log items marked "high risk".
  • Stop. Finish clear, not overloaded.
