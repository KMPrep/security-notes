Two-Week Security Engineering Study Roadmap

Focus: security engineering fundamentals and system design
Duration: 14 days · ~4–6 hours/day
Study style: Read notes → explain aloud → do practice questions → log gaps

Priority Tiers

Tier Topics
P1 — Must nail Auth/Authz, Crypto, Network Security, Web Security, Threat Modelling
P2 — Know well Cloud Security, OS/Exploitation, Detection, Incident Response, Coding
P3 — Know enough Forensics, Mobile, Malware/Reversing, Infra details

Day 13 — Coding Practice

Files: code/7-day-coding-reset-plan.md, code/day1/ through code/day7/

Work through the coding reset plan condensed into one intensive day. Security engineering work often includes coding; aim for clean, working solutions.

Session Focus
Morning (2h) Python fluency + arrays/strings/hashmaps (reset plan Days 1–2)
Midday (2h) Stacks, recursion, regex; sorting/searching, Big-O reasoning (Days 3–4)
Afternoon (2h) Security-flavored parsing: log analysis, IP parsing, CIDR containment, token validation (Days 5–7)

Must-do security coding problems:

  • Parse and count IPs from a log file; flag IPs exceeding a threshold
  • Validate a JWT structure (without library) — decode base64url, check fields
  • Check if a URL is in a blocklist using prefix/suffix matching
  • Detect anomalous login sequences (time-series array processing)
  • Implement a simple RBAC permission check
