CVSS, EPSS & the KEV Catalog

Category: Vulnerability Scoring & Threat Intelligence
Key standards / frameworks: FIRST CVSS v3.1 & v4.0, FIRST EPSS, CISA Known Exploited Vulnerabilities Catalog, CISA BOD 22-01, NVD, CWE, SSVC (CISA/CMU)

Overview

Three complementary signals dominate modern vulnerability triage. CVSS answers how bad is this flaw if exploited? It is a severity model, not a risk model. EPSS answers how likely is this flaw to be exploited in the wild soon? It is a probabilistic, empirically-trained threat signal. The CISA KEV catalog answers is this flaw being exploited right now? It is a binary, evidence-based ground-truth list.

Each signal on its own is misleading. A CVSS 9.8 with an EPSS score of 0.02% and no KEV entry is rarely the fire it appears to be, while a CVSS 7.5 that lands on KEV demands same-week action. The art is fusing all three, plus environmental context, into a defensible prioritization decision.

How It Works

CVSS v3.1 metric groups

CVSS v3.1 organizes metrics into three groups. The Base group captures intrinsic characteristics that do not change over time: Attack Vector (Network, Adjacent, Local, Physical), Attack Complexity, Privileges Required, User Interaction, Scope, and the CIA impact metrics (Confidentiality, Integrity, Availability). Base produces a 0.0 to 10.0 score and the severity band. The Temporal group adjusts for Exploit Code Maturity, Remediation Level, and Report Confidence. The Environmental group lets a defender re-weight impact by the asset's actual security requirements and modify base metrics for their environment. Most organizations publish and consume only the Base score, which is precisely the problem EPSS and KEV exist to correct.

CVSS v4.0 changes

CVSS v4.0, published by FIRST in late 2023, restructures the model to address v3.1 criticism. It renames the Temporal group to Threat and reduces it to a single Exploit Maturity metric. It splits impact into Vulnerable System (VC VI VA) and Subsequent System (SC SI SA) metrics, replacing the confusing Scope metric. It adds Supplemental metrics (Automatable, Recovery, Safety, Value Density) that are informational and do not change the number, and it introduces nomenclature like CVSS-B, CVSS-BT, and CVSS-BE to make explicit which metric groups were actually scored. A v4.0 vector string looks like CVSS:4.0/AV:N/AC:L/AT:N/PR:N/UI:N/VC:H/VI:H/VA:H/SC:N/SI:N/SA:N.

Severity bands and the risk-proxy trap

Both versions map numeric scores to bands: 0.0 None, 0.1 to 3.9 Low, 4.0 to 6.9 Medium, 7.0 to 8.9 High, 9.0 to 10.0 Critical. The central limitation is that CVSS is a measure of technical severity, not risk. It has no knowledge of whether exploit code exists, whether attackers are actually using it, whether the asset is internet-facing, or whether the vulnerable code path is even reachable. Treating the Base score as a risk score is the most common and most expensive mistake in vulnerability management.

EPSS and KEV as corrective signals

EPSS (Exploit Prediction Scoring System) outputs a probability between 0 and 1 that a given CVE will be exploited in the next 30 days, updated daily from a model trained on real exploitation telemetry, plus a percentile for relative ranking. The vast majority of CVEs score below 0.1, which is exactly why EPSS is such an effective noise filter. The CISA KEV catalog is the authoritative list of vulnerabilities with confirmed in-the-wild exploitation; under Binding Operational Directive 22-01, US federal agencies must remediate KEV entries by a set due date. Presence on KEV should override any low CVSS or EPSS score.

Key Takeaways

  • CVSS measures technical severity, not risk; the Base score is an input to prioritization, never the decision itself.
  • CVSS v4.0 replaces Scope with Vulnerable and Subsequent System impact, renames Temporal to Threat, and adds Supplemental metrics; label scores as CVSS-B, CVSS-BT, or CVSS-BE to show what was scored.
  • EPSS gives a daily-updated probability of exploitation in 30 days and is the single most effective tool for filtering out the huge tail of low-severity noise.
  • Distinguish EPSS score from EPSS percentile; they answer different questions and drive different cutoffs.
  • CISA KEV is confirmed in-the-wild exploitation and should override low CVSS or EPSS values; BOD 22-01 sets mandatory federal due dates.
  • KEV absence is not proof of safety; supplement it with additional exploit and threat-intelligence feeds.
  • The durable pattern is severity plus exploit probability plus known-exploited status plus environmental context, fused into one documented decision rule.
